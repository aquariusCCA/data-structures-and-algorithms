# 递归时间复杂度-Master theorem（跳過）

> 所属章节：[一、初识算法 / 遞歸](./README.md)
> 关键字：Master theorem、主定理、遞歸式、分治、二分查找、歸併排序、快速排序
> 建議回查情境：看到 $T(n)=aT(n/b)+f(n)$、想快速判断分治递归复杂度、想分清楚主定理何时不适用

## 本节导读

这一节说明如何用主定理估算一类常见分治递归的时间复杂度。

主定理适合处理这种问题：

> 每次把规模为 $n$ 的问题，拆成 $a$ 个规模相同的子问题，每个子问题规模都是 $n/b$，递归之外还要做 $f(n)$ 的工作。

如果递归式不长这样，例如 $T(n)=T(n-1)+n$，或快速排序最差情况下的 $T(n)=T(n-1)+T(1)+n$，就不能直接套主定理，通常要改用展开求解。

## 你會在這篇學到什麼

- 主定理的基本形式
- $a$、$b$、$f(n)$ 分别代表什么
- 为什么要比较 $f(n)$ 和 $n^{\log_b a}$
- 如何用同一套步骤判断二分查找、归并排序与快速排序的递归复杂度
- 哪些递归式不能直接用主定理

## 先讲结论

主定理的常见形式是：

$$
T(n)=aT(\frac{n}{b})+f(n)
$$

其中：

- $T(n)$：规模为 $n$ 时的运行时间
- $a$：每次递归产生的子问题个数
- $b$：子问题规模的缩小倍数，也就是子问题规模为 $n/b$
- $f(n)$：除递归调用以外，本层自己要做的工作

令：

$$
x=\log_b a
$$

如果 $f(n)$ 可以写成常见的多项式级别 $f(n)=\Theta(n^c)$，就比较 $c$ 和 $x$：

| 比较结果 | 时间复杂度 | 直觉 |
| --- | --- | --- |
| $c < x$ | $\Theta(n^x)$ | 递归展开后的子问题数量增长更快，递归部分主导 |
| $c = x$ | $\Theta(n^x \log n)$ | 每一层工作量差不多，总共有 $\log n$ 层 |
| $c > x$ | $\Theta(n^c)$ | 本层额外工作增长更快，上层工作主导 |

这里的 $n^x$ 可以先理解成「递归树展开到底之后，叶子层大约贡献多少工作量」。主定理本质上是在比较：

1. 递归分裂出来的子问题总量，也就是 $n^x$
2. 每一层除了递归之外额外做的工作，也就是 $f(n)$

> 注：完整主定理在第三种情况还会要求 regularity condition（正则条件）。本篇例子里的 $f(n)$ 都是常见多项式形式，可以用上面的简化判断。

## 使用前先确认能不能套

主定理不是所有递归式都能用。套公式前先检查：

1. 递归式要能写成 $T(n)=aT(n/b)+f(n)$。
2. $a$ 必须是固定常数，表示子问题个数。
3. $b>1$，表示每个子问题都缩小成原来的 $1/b$。
4. 子问题规模要相同，例如都是 $n/2$，而不是一个 $n/3$、一个 $2n/3$。
5. $f(n)$ 只计算「本层非递归部分」的工作量，不要把递归调用也算进去。

不能直接套的常见例子：

- $T(n)=T(n-1)+n$：子问题不是 $n/b$，而是 $n-1$。
- $T(n)=T(n/3)+T(2n/3)+n$：两个子问题规模不同。
- 快速排序最差情况 $T(n)=T(n-1)+T(1)+n$：分区极不平衡，不符合 $aT(n/b)$ 的固定等规模形式。

## 判斷流程

看到递归式时，按这个顺序做：

1. 找出 $a$：每层产生几个递归子问题。
2. 找出 $b$：每个子问题规模是 $n/b$。
3. 找出 $f(n)$：本层除了递归调用之外做了多少工作。
4. 计算 $x=\log_b a$。
5. 如果 $f(n)=\Theta(n^c)$，比较 $c$ 和 $x$。
6. 按三种情况写出复杂度。

## 只看递归式的例子

### 例 1：$T(n)=2T(n/2)+n^4$

- $a=2$
- $b=2$
- $x=\log_2 2=1$
- $f(n)=n^4$，所以 $c=4$

因为 $c>x$，所以：

$$
T(n)=\Theta(n^4)
$$

这表示递归之外的 $n^4$ 工作太大，主导了整体复杂度。

### 例 2：$T(n)=T(7n/10)+n$

这里要先把 $7n/10$ 改写成 $n/(10/7)$：

$$
T(n)=T(\frac{n}{10/7})+n
$$

- $a=1$
- $b=10/7$
- $x=\log_{10/7}1=0$
- $f(n)=n$，所以 $c=1$

因为 $c>x$，所以：

$$
T(n)=\Theta(n)
$$

### 例 3：$T(n)=16T(n/4)+n^2$

- $a=16$
- $b=4$
- $x=\log_4 16=2$
- $f(n)=n^2$，所以 $c=2$

因为 $c=x$，所以：

$$
T(n)=\Theta(n^2\log n)
$$

### 例 4：$T(n)=7T(n/3)+n^2$

- $a=7$
- $b=3$
- $x=\log_3 7 \approx 1.77$
- $f(n)=n^2$，所以 $c=2$

因为 $c>x$，所以：

$$
T(n)=\Theta(n^2)
$$

### 例 5：$T(n)=7T(n/2)+n^2$

- $a=7$
- $b=2$
- $x=\log_2 7 \approx 2.81$
- $f(n)=n^2$，所以 $c=2$

因为 $c<x$，所以：

$$
T(n)=\Theta(n^{\log_2 7})
$$

这里不是 $\Theta(n^2)$，因为递归分裂出来的子问题数量增长得更快。

### 例 6：$T(n)=2T(n/4)+\sqrt n$

- $a=2$
- $b=4$
- $x=\log_4 2=0.5$
- $f(n)=\sqrt n=n^{0.5}$，所以 $c=0.5$

因为 $c=x$，所以：

$$
T(n)=\Theta(\sqrt n\log n)
$$

## 结合代码的例子

### 例 7：二分查找

```java
int f(int[] a, int target, int i, int j) {
    if (i > j) {
        return -1;
    }
    int m = (i + j) >>> 1;
    if (target < a[m]) {
        return f(a, target, i, m - 1);
    } else if (a[m] < target) {
        return f(a, target, m + 1, j);
    } else {
        return m;
    }
}
```

二分查找每次只递归进入左半边或右半边，不会两边都递归。

- 子问题个数：$a=1$
- 子问题规模：$n/2$，所以 $b=2$
- 本层额外工作：只做比较与计算中点，是常数时间，所以 $f(n)=1=n^0$

递归式为：

$$
T(n)=T(\frac{n}{2})+1
$$

这里：

- $x=\log_2 1=0$
- $c=0$

因为 $c=x$，所以：

$$
T(n)=\Theta(\log n)
$$

直觉上，每次规模减半，最多减半 $\log n$ 次。

### 例 8：归并排序

```text
void split(B[], i, j, A[]) {
    if (j - i <= 1) {
        return;
    }

    m = (i + j) / 2;

    // 递归拆成两半
    split(A, i, m, B);
    split(A, m, j, B);

    // 合并
    merge(B, i, m, j, A);
}
```

归并排序每次拆成两个规模相同的子问题，递归结束后还要把两个有序部分合并。

- 子问题个数：$a=2$
- 子问题规模：$n/2$，所以 $b=2$
- 本层额外工作：合并两个有序区间，需要线性时间，所以 $f(n)=n$

递归式为：

$$
T(n)=2T(\frac{n}{2})+n
$$

这里：

- $x=\log_2 2=1$
- $c=1$

因为 $c=x$，所以：

$$
T(n)=\Theta(n\log n)
$$

直觉上，归并排序每一层总合并成本都是 $n$，总共有 $\log n$ 层。

### 例 9：快速排序

```text
algorithm quicksort(A, lo, hi) is
  if lo >= hi || lo < 0 then
    return

  // 分区
  p := partition(A, lo, hi)

  // 递归
  quicksort(A, lo, p - 1)
  quicksort(A, p + 1, hi)
```

快速排序能不能用主定理，要看每次分区是否足够平均。

#### 情况 1：分区很平均

如果每次都能把数组分成接近一半一半：

$$
T(n)=2T(\frac{n}{2})+n
$$

- $a=2$
- $b=2$
- $x=\log_2 2=1$
- 分区过程需要线性扫描，所以 $f(n)=n$，$c=1$

因为 $c=x$，所以：

$$
T(n)=\Theta(n\log n)
$$

#### 情况 2：分区极不平均

如果每次分区都很差，例如一边是 $0$ 个元素，另一边是 $n-1$ 个元素：

$$
T(n)=T(n-1)+T(1)+n
$$

这个递归式不能直接用主定理，因为子问题规模不是固定的 $n/b$。

这种情况要用展开求解，最终会得到快速排序最差时间复杂度：

$$
T(n)=O(n^2)
$$

## 原文容易誤解的地方

1. $x=\log_b a$ 不是凭空出现的常数，它用来衡量递归树叶子层的增长量。
2. 比较的重点不是「前者」或「后者」这两个模糊说法，而是比较 $n^x$ 与 $f(n)$ 谁增长更快。
3. 第三种情况不能无条件写成「只要 $f(n)=\Omega(n^c)$ 且 $c>x$ 就是 $\Theta(n^c)$」。本篇使用的是常见多项式版判断：$f(n)=\Theta(n^c)$。
4. 快速排序不是永远都能直接套主定理。只有在分区近似平均、能写成 $2T(n/2)+n$ 这类形式时，才适合用本篇的简化判断。

## 一句话抓核心

主定理的关键不是背公式，而是把递归式拆成 $a$、$b$、$f(n)$，再比较「递归分裂带来的增长」$n^{\log_b a}$ 和「本层额外工作」$f(n)$ 谁更大。
