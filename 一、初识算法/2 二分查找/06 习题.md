## 本节导读

这一节的重点不是引入新的算法，而是把前面学过的二分查找模板，放到具体题目中练习。

你会看到三种常见题型：

1. **查找某个值是否存在**
   → 用基础版二分查找

2. **查找目标值应该插入的位置**
   → 用 Leftmost 思路

3. **查找目标值出现的起点与终点**
   → 用 Leftmost + Rightmost 组合

所以这篇的核心目标，不是背题，而是建立下面这个对应关系：

* **找值** → 基础版
* **找插入点** → Leftmost
* **找区间左边界** → Leftmost
* **找区间右边界** → Rightmost

---

## 你會在這篇學到什麼

* LeetCode 704 是标准的基础二分查找练习题
* LeetCode 35 的本质不是“找值”，而是“找第一个大于等于 target 的位置”
* LeetCode 34 可以拆成两个问题：

  * 找 target 第一次出现的位置
  * 找 target 最后一次出现的位置

---

## 先記住這個題型對應表

| 题目类型          | 应该想到什么               |
| ------------- | -------------------- |
| 目标值存在就回传索引    | 基础版二分查找              |
| 找不到时还要知道该插在哪里 | Leftmost             |
| 有重复元素，要找最左位置  | Leftmost             |
| 有重复元素，要找最右位置  | Rightmost            |
| 有重复元素，要找完整区间  | Leftmost + Rightmost |

## E01. 二分查找 - LeetCode 704

### 题目本质

这是最标准的二分查找练习题。

给定一个有序数组和目标值 `target`：

* 如果找到，返回索引
* 如果找不到，返回 `-1`

### 这题的练习重点

这题不是难在题意，而是用来检查你是否真的掌握了基础模板：

* 区间如何收缩
* `mid` 如何计算
* 找到目标时何时返回
* 找不到时何时结束

### 结论

这题使用我们前面学过的 **基础版二分查找** 即可。

## E02. 搜索插入位置 - LeetCode 35

### 题目描述

给定一个排序数组和一个目标值：

* 如果目标值存在，返回它的索引
* 如果目标值不存在，返回它应该被插入的位置

例如：

```text
输入: nums = [1,3,5,6], target = 5
输出: 2

输入: nums = [1,3,5,6], target = 2
输出: 1

输入: nums = [1,3,5,6], target = 7
输出: 4
```

---

### 这题的关键

这题看起来像“查找目标值”，其实本质是：

> **找第一个大于等于 target 的位置**

为什么？

因为如果目标值不存在，它应该插入到“第一个比它大”的元素前面。
而如果目标值存在，那么“第一个大于等于 target 的位置”也正好就是它本身的位置。

所以这题应该想到的是：

> **Leftmost**

---

### 为什么 `return i` 就是插入位置？

看 Leftmost 模板：

```java
public static int binarySearchLeftmost(int[] a, int target) {
    int i = 0, j = a.length - 1;
    while (i <= j) {
        int m = (i + j) >>> 1;
        if (target <= a[m]) {
            j = m - 1;
        } else {
            i = m + 1;
        }
    }
    return i;
}
```

循环结束后：

* `j` 停在“最后一个小于 target 的位置”
* `i` 停在“第一个大于等于 target 的位置”

因此：

* 如果 target 存在，`i` 就是它第一次出现的位置
* 如果 target 不存在，`i` 就是它应该插入的位置

所以这题直接回传 `i` 即可。

---

### 小例子理解

以 `nums = [1,3,5,6]`，`target = 2` 为例：

* `1 < 2`
* `3 >= 2`

所以 `2` 应该插在 `1` 和 `3` 中间，也就是索引 `1`

而 Leftmost 最后回传的 `i`，正好就是 `1`

---

### 结论

这题不是单纯“找 target”，而是：

> **找 target 应该落下来的最左位置**

因此使用 **Leftmost 版二分查找** 最合适。

## E03. 搜索开始结束位置 - LeetCode 34

### 题目描述

给你一个按照非递减顺序排列的整数数组 `nums`，和一个目标值 `target`。请你找出目标值在数组中的开始位置和结束位置。

如果数组中不存在目标值 `target`，返回 `[-1, -1]`。

你必须设计并实现时间复杂度为 `O(log n)` 的算法。

例如：

```text
输入：nums = [5,7,7,8,8,10], target = 8
输出：[3,4]

输入：nums = [5,7,7,8,8,10], target = 6
输出：[-1,-1]

输入：nums = [], target = 0
输出：[-1,-1]
```

---

### 这题的本质

这题不是找“有没有 target”，而是找：

1. `target` **第一次出现的位置**
2. `target` **最后一次出现的位置**

所以它其实可以拆成两个子问题：

* 找最左边界 → `Leftmost`
* 找最右边界 → `Rightmost`

---

## 先看左边界：Leftmost

```java
public static int left(int[] a, int target) {
    int i = 0, j = a.length - 1;
    int candidate = -1;
    while (i <= j) {
        int m = (i + j) >>> 1;
        if (target < a[m]) {
            j = m - 1;
        } else if (a[m] < target) {
            i = m + 1;
        } else {
            candidate = m;
            j = m - 1;
        }
    }
    return candidate;
}
```

### 关键思路

当 `a[m] == target` 时，并不立刻结束，而是：

* 先把当前位置记录到 `candidate`
* 再继续往左边找，看前面有没有更早出现的 target

所以命中后要：

```java
j = m - 1;
```

因为我们要的是 **最左边的那个 target**

---

## 再看右边界：Rightmost

```java
public static int right(int[] a, int target) {
    int i = 0, j = a.length - 1;
    int candidate = -1;
    while (i <= j) {
        int m = (i + j) >>> 1;
        if (target < a[m]) {
            j = m - 1;
        } else if (a[m] < target) {
            i = m + 1;
        } else {
            candidate = m;
            i = m + 1;
        }
    }
    return candidate;
}
```

### 关键思路

当 `a[m] == target` 时：

* 先记录当前位置
* 再继续往右边找，看后面有没有更晚出现的 target

所以命中后要：

```java
i = m + 1;
```

因为我们要的是 **最右边的那个 target**

---

## 最终整合

```java
public static int[] searchRange(int[] nums, int target) {
    int x = left(nums, target);
    if (x == -1) {
        return new int[] {-1, -1};
    } else {
        return new int[] {x, right(nums, target)};
    }
}
```

---

## 为什么这样写就够了？

因为：

* 如果 `left(nums, target)` 都找不到，说明数组里根本没有 `target`
* 这时直接回传 `[-1, -1]`
* 如果左边界找到了，再去找右边界即可

这样整体时间复杂度仍然是：

```text
O(log n) + O(log n) = O(log n)
```

---

## 小例子推演

以 `nums = [5,7,7,8,8,10]`，`target = 8` 为例：

### 找左边界

当二分命中 `8` 时：

* 先记下当前位置
* 然后继续往左缩

最后会停在索引 `3`

### 找右边界

当二分命中 `8` 时：

* 先记下当前位置
* 然后继续往右缩

最后会停在索引 `4`

所以结果为：

```text
[3, 4]
```

## 本节总结

这三题的重点不是题目本身，而是训练你看到题目时，能立刻判断要套哪一种二分模板。

### 你应该建立的反射是：

* **704：找值是否存在**
  → 基础版二分查找

* **35：找插入位置**
  → 找第一个大于等于 target 的位置
  → Leftmost

* **34：找开始与结束位置**
  → 找左边界 + 找右边界
  → Leftmost + Rightmost
