## 第 1 题答案：基础版二分查找

### 1）查找 `38`

数组：

```java
int[] a = {7, 13, 21, 30, 38, 44, 52, 53};
```

第 1 轮：

- `i = 0`
- `j = 7`
- `m = (0 + 7) >>> 1 = 3`
- `a[m] = 30`
- `38 > 30`
- 更新：`i = m + 1 = 4`

第 2 轮：

- `i = 4`
- `j = 7`
- `m = (4 + 7) >>> 1 = 5`
- `a[m] = 44`
- `38 < 44`
- 更新：`j = m - 1 = 4`

第 3 轮：

- `i = 4`
- `j = 4`
- `m = 4`
- `a[m] = 38`
- 命中目标，返回 `4`

### 2）查找 `15`

第 1 轮：

- `i = 0`
- `j = 7`
- `m = 3`
- `a[m] = 30`
- `15 < 30`
- 更新：`j = 2`

第 2 轮：

- `i = 0`
- `j = 2`
- `m = 1`
- `a[m] = 13`
- `15 > 13`
- 更新：`i = 2`

第 3 轮：

- `i = 2`
- `j = 2`
- `m = 2`
- `a[m] = 21`
- `15 < 21`
- 更新：`j = 1`

此时：

- `i = 2`
- `j = 1`
- `i > j`

闭区间 `[i, j]` 已经为空，所以返回 `-1`。

### 3）观念说明

为什么是 `i <= j`：

- 因为基础版维护的是闭区间 `[i, j]`
- 当 `i == j` 时，还剩最后一个候选元素
- 只有当 `i > j` 时，区间才真正为空

为什么是 `m - 1 / m + 1`：

- 因为 `m` 已经比较过
- 如果不把 `m` 排除，就可能重复比较同一个位置
- 严重时还会导致区间无法缩小，甚至死循环

核心句：

> 基础版的关键不是“背模板”，而是记住它维护的是闭区间 `[i, j]`，所以 `i == j` 时也必须继续比较。

## 第 2 题答案：改动版二分查找

```java
public static int binarySearchAlternative(int[] a, int target) {
    int i = 0;
    int j = a.length;

    while (i < j) {
        int m = (i + j) >>> 1;
        if (target < a[m]) {
            j = m;
        } else if (a[m] < target) {
            i = m + 1;
        } else {
            return m;
        }
    }

    return -1;
}
```

### 为什么 `j = a.length`

- 因为改动版维护的是左闭右开区间 `[i, j)`
- 右边界 `j` 本身不是候选元素
- 要让初始区间覆盖整个数组，右边界就应该卡在最后一个元素的右边
- 所以应写成 `j = a.length`

### 为什么向左收缩时写 `j = m`

- 因为新区间应当是 `[i, m)`
- 在左闭右开区间里，右端点本来就不包含
- 所以 `j = m` 已经表示“把 `m` 排除掉，并把范围收缩到它左边”
- 不需要再写 `m - 1`

核心句：

> 改动版不是把基础版随便改几行，而是把区间模型改成了 `[i, j)`，所以循环条件和边界收缩都必须一起变。

## 第 3 题答案：平衡版二分查找

```java
public static int binarySearchBalance(int[] a, int target) {
    if (a.length == 0) {
        return -1;
    }

    int i = 0;
    int j = a.length;

    while (1 < j - i) {
        int m = (i + j) >>> 1;
        if (target < a[m]) {
            j = m;
        } else {
            i = m;
        }
    }

    return a[i] == target ? i : -1;
}
```

### 为什么是 `i = m`

- 因为平衡版循环内不直接判等返回
- 当进入 `else` 时，只能说明 `target >= a[m]`
- 此时 `m` 仍然可能就是目标值
- 所以不能跳过 `m`
- 必须把它保留在候选区间里，因此写 `i = m`

### 为什么最终判断放到循环外

- 因为平衡版的目标是先把区间缩小到只剩 1 个候选元素
- 循环内主要负责“缩区间”，不是“立即结束”
- 等只剩一个候选位置后，再统一判断是否等于 `target`

核心句：

> 平衡版的重点不是更快，而是把循环内的工作改成“持续缩范围”，最后再统一确认答案。

## 第 4 题答案：Java 标准库与插入点

数组：

```java
int[] a = {2, 5, 8};
```

结果如下：

- `Arrays.binarySearch(a, 1) = -1`
- `Arrays.binarySearch(a, 4) = -2`
- `Arrays.binarySearch(a, 6) = -3`
- `Arrays.binarySearch(a, 10) = -4`

对应插入点：

- `1` 的插入点是 `0`
- `4` 的插入点是 `1`
- `6` 的插入点是 `2`
- `10` 的插入点是 `3`

### 为什么不统一返回 `-1`

- 因为 Java 想在“查找失败”时顺便保留插入点信息
- 这样调用方不需要再做一次额外扫描
- 一次二分查找就能同时得到：
  - 找没找到
  - 如果没找到，应该插到哪里

### 标准还原写法

当 `result < 0` 时：

```java
int insertIndex = -result - 1;
```

核心句：

> Java 标准库返回负数，不是为了刁难使用者，而是为了把“失败”与“插入点”编码在同一个返回值里。

## 第 5 题答案：Leftmost / Rightmost 与题型套用

### 1）Leftmost

```java
public static int binarySearchLeftmost(int[] a, int target) {
    int i = 0;
    int j = a.length - 1;

    while (i <= j) {
        int m = (i + j) >>> 1;
        if (target <= a[m]) {
            j = m - 1;
        } else {
            i = m + 1;
        }
    }

    return i;
}
```

### 2）Rightmost

```java
public static int binarySearchRightmost(int[] a, int target) {
    int i = 0;
    int j = a.length - 1;

    while (i <= j) {
        int m = (i + j) >>> 1;
        if (target < a[m]) {
            j = m - 1;
        } else {
            i = m + 1;
        }
    }

    return i - 1;
}
```

### 3）结果

给定：

```java
int[] a = {1, 2, 4, 4, 4, 5, 6, 7};
```

结果为：

- `leftmost(4) = 2`
- `rightmost(4) = 4`
- `leftmost(3) = 2`
- `rightmost(3) = 1`
- `target = 3` 的插入位置是 `2`
- `target = 3` 的前任索引是 `1`
- `target = 4` 的后任索引是 `5`

### 4）题型对应

LeetCode 35 `searchInsert`：

- 本质是找第一个 `>= target` 的位置
- 所以应该想到 `Leftmost`

LeetCode 34 `searchRange`：

- 本质是找目标值的最左边界和最右边界
- 所以应该想到 `Leftmost + Rightmost`

核心句：

> Leftmost / Rightmost 的本质不是“处理重复元素”，而是“查找边界”，所以它们能继续延伸到插入点、前任、后任与范围查询。
