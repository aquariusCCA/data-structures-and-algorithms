# 二分查找平衡版

> 所属章节：[一、初识算法 / 二分查找](./README.md)
> 关键字：二分查找、平衡版、左闭右开、比较次数、`1 < j - i`、`Θ(log n)`
> 建議回查情境：想理解平衡版为什么减少平均比较次数时、想确认 `1 < j - i` 的含义时、比较基础版和改动版性能时

## 本节导读

这一节说明二分查找平衡版的思路。基础版在目标位于左侧和右侧时，每轮比较次数可能不同；平衡版不急着在循环内直接返回，而是持续缩小区间，直到只剩一个候选元素。

重点是理解：平衡版中 `i` 指向的仍可能是目标，因此当目标不在左侧时，`i` 不能直接跳过 `m`。

## 你會在這篇學到什麼

- 基础版左右查找比较次数不平衡的原因
- 平衡版如何通过缩小区间来减少循环内平均比较次数
- `while (1 < j - i)` 代表的候选区间含义
- 平衡版的时间复杂度与空间复杂度直觉

將循環次數計為 L 次時：

元素在最左边  ⇒  只需要比較 L 次，即 `target < a[m]` 要執行 L 次。

元素在最右边   ⇒  需要比較 $2 \times L$ 次，即需要執行 `target < a[m]`  和 `a[m] < target` 各Ｌ次。

```java
/**
 * 二分查找基础版
 *
 * @param a      待查找的升序数组
 * @param target 待查找的目标值
 * @return 找到则返回索引 找不到返回 -1
 */
public static int binarySearchBasic(int[] a, int target) {
    int i = 0, j = a.length - 1; // 设置指针和初值
    while (i <= j) { // i~j 范围内有东西
        int m = (i + j) / 2;
        if (target < a[m]) { // 目标在左边
            j = m - 1;
        } else if (a[m] < target) { // 目标在右边
            i = m + 1;
        } else { // 找到了
            return m;
        }
    }
    return -1;
}
```

在上述基础版二分查找算法中，如果待查找元素在最左侧的位置，则需要执行 L 次，即每次只需要执行 `if(target < arr[m])` 分支，而待查找元素在最右侧的位置时候，则需要执行 2L 次，即每次既需要执行 `if(target < arr[m])` 分支，又需要执行 `if(arr[m] < target)`。

元素在左侧和右侧的执行效率不一样，在左侧时查找速率更快，在右侧时查找速率要慢点，平衡版二分查找可以让左侧右侧的效率都保持一样，下面给出左右查找效率一样平衡版二分查找的实现思路：

1. 左闭右开的区间，`i` 指向的可能是目标，而 `j` 指向的不是目标
2. 不奢望循环内通过 `m` 找出目标，缩小区间直至剩 `1` 个，剩下的这个可能就是要找的（通过 `i`）
    - `1 < j - i` 的含义是，在范围内待比较的元素个数 > 1
3. 改变 `i` 边界时，它指向的可能是目标，因此不能 `m + 1`
4. 循环内的平均比较次数减少了
5. 时间复杂度 $\Theta(log(n))$

```java
/**
 * 二分查找平衡版
 *
 * @param a      待查找的升序数组
 * @param target 待查找的目标值
 * @return 找到则返回索引 找不到返回 -1
 */
public static int binarySearchBalance(int[] a, int target) {
    int i = 0, j = a.length;
    while (1 < j - i) { // 表示的是「范围内待查找的元素个数 > 1」 时，當範圍內只剩下一個元素的時候就退出循環
        int m = (i + j) >>> 1;
        if (target < a[m]) { // 目标在左边
            j = m;
        } else { // 目标在 m 或右边
            i = m;
        }
    }
    return (target == a[i]) ? i : -1;
}
```

## 常見混淆點

- **基础版 vs 平衡版**：基础版可能在循环内直接找到并返回；平衡版倾向先缩小区间，最后检查剩余候选元素。
- **`i = m + 1` vs `i = m`**：平衡版中 `m` 仍可能是目标，因此不能在目标不小于 `a[m]` 时直接跳过 `m`。
- **`1 < j - i`**：表示范围内待查找元素个数大于 1，剩下 1 个时退出循环并在循环外判断。
