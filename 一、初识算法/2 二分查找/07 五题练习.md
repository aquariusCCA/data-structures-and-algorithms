## 本页怎么使用

这一份不是把二分查找拆成很多零散小题，而是压成 **5 道综合题**。

每一题都对应这组笔记里的一个核心主题：

1. 基础版
2. 改动版
3. 平衡版
4. Java 标准库与边界理解
5. Leftmost / Rightmost 与题型套用

如果你能把这 5 题独立做完，说明这一章的主干已经比较稳了。

---

## 第 1 题：基础版二分查找

给定升序数组：

```java
int[] a = {7, 13, 21, 30, 38, 44, 52, 53};
```

请完成下面三件事：

1. 用基础版二分查找手推 `target = 38` 的查找过程，写出每一轮的 `i`、`j`、`m` 与比较结果。
2. 用基础版二分查找手推 `target = 15` 的查找过程，说明为什么最后返回 `-1`。
3. 用你自己的话说明：
   - 为什么循环条件必须写成 `i <= j`
   - 为什么边界更新必须写成 `j = m - 1` 或 `i = m + 1`

---

## 第 2 题：改动版二分查找

请你手写下面这个方法：

```java
public static int binarySearchAlternative(int[] a, int target)
```

要求：

- 使用左闭右开区间 `[i, j)`
- 找到返回索引
- 找不到返回 `-1`

写完后，再回答下面两个问题：

1. 为什么这个版本初始化要写 `j = a.length`？
2. 为什么当 `target < a[m]` 时，要写 `j = m`，而不是 `j = m - 1`？

---

## 第 3 题：平衡版二分查找

请你手写下面这个方法：

```java
public static int binarySearchBalance(int[] a, int target)
```

要求：

- 使用平衡版写法
- 补上空数组处理
- 找到返回索引
- 找不到返回 `-1`

写完后，再回答下面两个问题：

1. 为什么平衡版里向右收缩写的是 `i = m`，而不是 `i = m + 1`？
2. 为什么平衡版把“是否命中目标”的最终判断放在循环外？

---

## 第 4 题：Java 标准库与插入点

给定数组：

```java
int[] a = {2, 5, 8};
```

请直接写出下面这些表达式的返回值，并写出各自对应的插入点：

```java
Arrays.binarySearch(a, 1)
Arrays.binarySearch(a, 4)
Arrays.binarySearch(a, 6)
Arrays.binarySearch(a, 10)
```

然后再回答：

1. 为什么 Java 标准库在找不到时不统一返回 `-1`？
2. 已知 `result < 0` 时，应该用什么标准写法还原插入点？

---

## 第 5 题：Leftmost / Rightmost 与题型套用

给定数组：

```java
int[] a = {1, 2, 4, 4, 4, 5, 6, 7};
```

请完成下面问题：

1. 写出 `binarySearchLeftmost(int[] a, int target)`，要求返回第一个 `>= target` 的位置。
2. 写出 `binarySearchRightmost(int[] a, int target)`，要求返回最后一个 `<= target` 的位置。
3. 直接写出下面这些结果：
   - `leftmost(4)`
   - `rightmost(4)`
   - `leftmost(3)`
   - `rightmost(3)`
   - `target = 3` 的插入位置
   - `target = 3` 的前任索引
   - `target = 4` 的后任索引
4. 说明下面两题分别应该套什么思路：
   - LeetCode 35 `searchInsert`
   - LeetCode 34 `searchRange`
