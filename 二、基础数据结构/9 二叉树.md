# 二叉樹概述

**二叉樹（Binary Tree）** 是一種樹狀資料結構。
它的特點是：**每個節點最多只能有兩個子節點**，分別稱為：

* 左子節點（left child）
* 右子節點（right child）

例如：

```text
     1
    / \
   2   3
  /   / \
 4   5   6
```

在這棵樹中：

* `1` 是根節點
* `4`、`5`、`6` 沒有子節點，所以是葉子節點
* `2` 只有一個孩子，所以是單子節點
* `3` 有兩個孩子，所以是雙子節點

## 節點名稱

在二叉樹中，常見的節點類型如下：

* **根節點（root）**：沒有父節點的節點
* **葉子節點（leaf）**：沒有任何孩子的節點
* **單子節點**：只有一個孩子的節點
* **雙子節點**：有兩個孩子的節點

## 常見的二叉樹結構

### 1. 完全二叉樹

**完全二叉樹（Complete Binary Tree）** 的特點是：

* 除了最後一層之外，其餘每一層都必須填滿
* 最後一層的節點必須按照 **從左到右** 的順序排列

也就是說，節點不能跳空，必須先左後右地填入。

### 2. 平衡二叉樹

**平衡二叉樹（Balanced Binary Tree）** 的特點是：

* 對於每一個節點來說
* 它的左子樹與右子樹的高度差 **不超過 1**

這種結構可以避免樹過度傾斜，讓查找、插入、刪除等操作效率更穩定。

# 二叉樹的存儲方式

二叉樹通常有兩種常見的表示方法：

## 1. 用節點類別表示

這是最常見的方式，每個節點都保存：

* 自己的值 `val`
* 左子節點 `left`
* 右子節點 `right`

```java
public class TreeNode {
    public int val;
    public TreeNode left;
    public TreeNode right;

    public TreeNode(int val) {
        this.val = val;
    }

    public TreeNode(TreeNode left, int val, TreeNode right) {
        this.left = left;
        this.val = val;
        this.right = right;
    }

    @Override
    public String toString() {
        return String.valueOf(this.val);
    }
}
```

這種方式的優點是結構清楚，特別適合表示一般二叉樹。

## 2. 用數組表示

二叉樹也可以用數組來表示，例如：

```text
[7, 5, 6, 4, 2, 1, 3]
```

對應的樹結構如下：

```text
        7
      /   \
     5     6
    / \   / \
   4   2 1   3
```

### 數組與節點的位置關係

* 根節點在索引 `0`
* 某個節點若在索引 `i`

    * 左孩子在索引 `2i + 1`
    * 右孩子在索引 `2i + 2`

例如：

* 索引 `0` 的元素 `7`

    * 左孩子是索引 `1` 的 `5`
    * 右孩子是索引 `2` 的 `6`

* 索引 `1` 的元素 `5`

    * 左孩子是索引 `3` 的 `4`
    * 右孩子是索引 `4` 的 `2`

* 索引 `2` 的元素 `6`

    * 左孩子是索引 `5` 的 `1`
    * 右孩子是索引 `6` 的 `3`

### 若節點缺少孩子

如果某個位置缺少孩子，可以用 `null` 佔位：

```text
[7, 5, 6, 4, 2, null, 3]
```

這樣仍然可以維持索引對應關係。

# 二叉樹的遍歷

遍歷就是按照某種規則，把樹中的節點依序訪問一遍。
二叉樹的遍歷主要分成兩大類：

* 廣度優先遍歷（BFS）
* 深度優先遍歷（DFS）


## 廣度優先遍歷

**廣度優先遍歷（Breadth-First Search, BFS）**
又稱為 **層序遍歷**。

它的核心概念是：

> 先訪問離根節點最近的節點，再一層一層往下訪問。

例如下面這棵樹：

![image-20230216153607396](images/image-20230216153607396.png)

| 本轮开始时队列 | 本轮访问节点 |
| -------------- | ------------ |
| [1]            | 1            |
| [2, 3]         | 2            |
| [3, 4]         | 3            |
| [4, 5, 6]      | 4            |
| [5, 6]         | 5            |
| [6, 7, 8]      | 6            |
| [7, 8]         | 7            |
| [8]            | 8            |
| []             |              |


1. 初始化，將根節點加入隊列
2. 循環處理隊列中每個節點，直至隊列為空
3. 每次循環內處理節點後，將它的孩子節點（即下一層的節點）加入隊列

### BFS 的做法

通常會搭配 **隊列（Queue）** 來實作。

#### 步驟

1. 先把根節點加入隊列
2. 只要隊列不為空，就持續重複：

    * 取出隊首節點並訪問
    * 再把它的左孩子、右孩子加入隊列（如果存在）

#### 為什麼要用隊列？

因為隊列是 **先進先出（FIFO）** 的結構，能保證節點按照層級順序被處理。

**補充說明**

* 如果二叉樹是用 `TreeNode` 表示，通常用隊列做層序遍歷
* 如果二叉樹本身就是用數組表示，那麼**直接從前往後遍歷數組**，本質上就是層序遍歷

## 深度優先遍歷

**深度優先遍歷（Depth-First Search, DFS）**
是指在遍歷二叉樹時，會先沿著某一條路徑一路往下走，走到底之後，再回頭處理其他分支。

對於二叉樹來說，深度優先遍歷可以分成三種：

1. 前序遍歷（Pre-order）
2. 中序遍歷（In-order）
3. 後序遍歷（Post-order）

以下都使用這棵二叉樹來說明：

```text
        1
       / \
      2   3
     /   / \
    4   5   6
```

### 1. 前序遍歷（Pre-order）

#### 規則

對於每一棵子樹，都按照以下順序：

1. 先訪問目前節點
2. 再訪問左子樹
3. 最後訪問右子樹

也就是：

```text
根 → 左 → 右
```

**遍歷結果**

這棵樹的前序遍歷結果是：

```text
1 2 4 3 5 6
```

#### 理解過程

##### 第一步：先從根節點開始

先訪問根節點 `1`，所以先輸出：

```text
1
```

##### 第二步：處理 1 的左子樹

`1` 的左子節點是 `2`，依照前序規則，先訪問 `2`：

```text
1 2
```

接著繼續處理 `2` 的左子樹，也就是節點 `4`：

```text
1 2 4
```

`4` 沒有左子樹，也沒有右子樹，因此返回上一層。

##### 第三步：處理 2 的右子樹

`2` 的右子樹是空的，所以不做任何事，回到根節點 `1`。

##### 第四步：處理 1 的右子樹

接著處理 `1` 的右子樹，也就是節點 `3`：

```text
1 2 4 3
```

先處理 `3` 的左子樹，也就是 `5`：

```text
1 2 4 3 5
```

`5` 沒有子節點，返回。

再處理 `3` 的右子樹，也就是 `6`：

```text
1 2 4 3 5 6
```

`6` 也沒有子節點，遍歷結束。

### 2. 中序遍歷（In-order）

#### 規則

對於每一棵子樹，都按照以下順序：

1. 先訪問左子樹
2. 再訪問目前節點
3. 最後訪問右子樹

也就是：

```text
左 → 根 → 右
```

**遍歷結果**

這棵樹的中序遍歷結果是：

```text
4 2 1 5 3 6
```

#### 理解過程

##### 第一步：從根節點 1 開始

中序遍歷不會立刻輸出 `1`，而是先去處理 `1` 的左子樹。

##### 第二步：處理 1 的左子樹，也就是 2

到了 `2`，仍然不能立刻輸出，因為中序規則是先左後根再右。
所以繼續處理 `2` 的左子樹，也就是 `4`。

##### 第三步：處理 4

`4` 的左子樹是空的，所以先返回。
這時可以輸出 `4`：

```text
4
```

接著 `4` 的右子樹也是空的，再返回到 `2`。

##### 第四步：回到 2

`2` 的左子樹已經處理完了，所以現在輸出 `2`：

```text
4 2
```

`2` 的右子樹是空的，因此返回到 `1`。

##### 第五步：回到 1

`1` 的左子樹已經處理完，所以現在輸出 `1`：

```text
4 2 1
```

接著處理 `1` 的右子樹，也就是 `3`。

##### 第六步：處理 3

先處理 `3` 的左子樹，也就是 `5`。

`5` 的左子樹是空的，所以先返回；
接著輸出 `5`：

```text
4 2 1 5
```

`5` 的右子樹也是空的，因此返回到 `3`。

##### 第七步：回到 3

`3` 的左子樹已經處理完，所以輸出 `3`：

```text
4 2 1 5 3
```

接著處理 `3` 的右子樹，也就是 `6`。

##### 第八步：處理 6

`6` 的左子樹是空的，先返回；
然後輸出 `6`：

```text
4 2 1 5 3 6
```

`6` 的右子樹也是空的，整個遍歷完成。

### 3. 後序遍歷（Post-order）

#### 規則

對於每一棵子樹，都按照以下順序：

1. 先訪問左子樹
2. 再訪問右子樹
3. 最後訪問目前節點

也就是：

```text
左 → 右 → 根
```

**遍歷結果**

這棵樹的後序遍歷結果是：

```text
4 2 5 6 3 1
```

#### 理解過程

##### 第一步：從根節點 1 開始

後序遍歷不會立刻輸出 `1`，而是先處理左子樹。

##### 第二步：處理 1 的左子樹，也就是 2

到了 `2` 也不能立刻輸出，因為後序規則是左、右、根。
所以先處理 `2` 的左子樹，也就是 `4`。

##### 第三步：處理 4

`4` 的左子樹是空的，右子樹也是空的。
左右子樹都處理完後，才能輸出 `4`：

```text
4
```

然後返回到 `2`。

##### 第四步：回到 2

接著處理 `2` 的右子樹，但它是空的。
因此現在可以輸出 `2`：

```text
4 2
```

然後返回到 `1`。

##### 第五步：處理 1 的右子樹，也就是 3

到了 `3` 之後，先處理左子樹 `5`。

`5` 沒有左右子樹，因此先輸出 `5`：

```text
4 2 5
```

再回到 `3`，接著處理右子樹 `6`。

`6` 也沒有左右子樹，因此輸出 `6`：

```text
4 2 5 6
```

##### 第六步：回到 3

`3` 的左子樹和右子樹都已經處理完，所以現在輸出 `3`：

```text
4 2 5 6 3
```

##### 第七步：回到 1

最後，`1` 的左右子樹都已處理完成，因此輸出 `1`：

```text
4 2 5 6 3 1
```

遍歷結束。

### 三種 DFS 的比較

同樣一棵樹：

```text
        1
       / \
      2   3
     /   / \
    4   5   6
```

三種深度優先遍歷結果如下：

* **前序遍歷**：`1 2 4 3 5 6`
* **中序遍歷**：`4 2 1 5 3 6`
* **後序遍歷**：`4 2 5 6 3 1`

**快速記憶法**

可以直接記成下面三種順序：

* **前序**：根 → 左 → 右
* **中序**：左 → 根 → 右
* **後序**：左 → 右 → 根

### 方式一：使用遞歸的方式實現深度優先遍歷

```java
public class TreeNode {
    public int val;
    public TreeNode left;
    public TreeNode right;

    public TreeNode(int val) {
        this.val = val;
    }

    public TreeNode(TreeNode left, int val, TreeNode right) {
        this.left = left;
        this.val = val;
        this.right = right;
    }

    @Override
    public String toString() {
        return String.valueOf(this.val);
    }
}
```

```java
public class TestTraversal {
    public static void main(String[] args) {
        TreeNode root = new TreeNode(
                new TreeNode(null, 2, new TreeNode(4)),
                1,
                new TreeNode(new TreeNode(5), 3, new TreeNode(6))
        );

        preOrder(root);
        System.out.println();

        inOrder(root);
        System.out.println();

        postOrder(root);
        System.out.println();
    }

    static void preOrder(TreeNode node) {
        if (node == null) {
            return;
        }
        System.out.print(node + "\t");
        preOrder(node.left);
        preOrder(node.right);
    }

    static void inOrder(TreeNode node) {
        if (node == null) {
            return;
        }
        inOrder(node.left);
        System.out.print(node + "\t");
        inOrder(node.right);
    }

    static void postOrder(TreeNode node) {
        if (node == null) {
            return;
        }
        postOrder(node.left);
        postOrder(node.right);
        System.out.print(node + "\t");
    }
}
```

### 方式二：使用非遞歸方式實現深度優先遍歷

前面介紹過，深度優先遍歷（DFS）可以用**遞歸**完成。
但除了遞歸之外，也可以用**非遞歸**的方式實現，只是理解上會稍微難一些。

對二叉樹來說，深度優先遍歷有三種：

* 前序遍歷：`1 2 4 3 5 6`
* 中序遍歷：`4 2 1 5 3 6`
* 後序遍歷：`4 2 5 6 3 1`

例如這棵樹：

```text
    1
   / \
  2   3
 /   / \
4   5   6
```

#### 一、先觀察規律

雖然三種遍歷的**輸出順序不同**，但它們在樹中「走的路」其實很像。

以上面這棵樹為例，如果把整個走訪過程攤開來看，大致會像這樣：

```text
1 -> 2 -> 4 -> 2 -> 1 -> 3 -> 5 -> 3 -> 6 -> 3 -> 1
```

也就是說：

* 先一路往左走到底
* 走不下去時，再回到上一個節點
* 如果有右子樹，就再往右邊走
* 最後整棵樹會完整走一圈

所以三種 DFS 的差別，不是「走法完全不同」，而是：

> **同樣的走訪過程中，節點被輸出的時機不同。**

這就是理解非遞歸 DFS 的關鍵。

#### 二、為什麼需要棧（Stack）

如果只用一個指標 `curr` 往左走：

```java
TreeNode curr = root;

while (curr != null) {
    curr = curr.left;
}
```

那麼走到最左邊之後，就不知道怎麼回去了。

因此我們需要一種資料結構，幫我們記住「來時的路」。
這時最適合的就是 **棧（Stack）**。

因為棧的特性是：

* 後進先出（LIFO）
* 最後進去的節點，剛好就是目前要回退時最先取出的節點

#### 三、基本思路：一路往左，走不動就回退

假設樹如下：

```text
    1
   / \
  2   3
 /   / \
4   5   6
```

一開始從 `1` 出發：

1. 經過 `1`，把 `1` 壓入棧
2. 繼續往左走到 `2`，把 `2` 壓入棧
3. 再往左走到 `4`，把 `4` 壓入棧

此時棧的內容如下：

```text
| 4 |
| 2 |
| 1 |
-----
```

接著發現 `4` 已經沒有左孩子了，說明走到頭了。
這時就要開始回退，也就是把棧頂元素彈出來，回到上一個節點。

所以整體流程可以概括成：

* **往左走時**：一路壓棧
* **走不下去時**：彈棧回退
* **回到某個節點後**：再考慮是否要走它的右子樹

#### 四、前序遍歷（非遞歸）

##### 規則

前序遍歷的順序是：

```text
根 → 左 → 右
```

因此，節點應該在**第一次遇到它時就立刻輸出**。

##### 實作程式碼

```java
LinkedList<TreeNode> stack = new LinkedList<>();
TreeNode curr = root;

while (!stack.isEmpty() || curr != null) {
    if (curr != null) {
        System.out.println(curr.val); // 前序：先處理自己
        stack.push(curr);             // 記住回來的路
        curr = curr.left;             // 繼續往左走
    } else {
        TreeNode node = stack.pop();  // 回退
        curr = node.right;            // 改走右子樹
    }
}
```

##### 理解方式

前序遍歷最重要的一點就是：

> **節點在第一次到達時就輸出。**

所以程式裡，輸出時機放在：

```java
if (curr != null) {
    System.out.println(curr.val);
}
```

也就是剛走到這個節點時。

#### 五、中序遍歷（非遞歸）

##### 規則

中序遍歷的順序是：

```text
左 → 根 → 右
```

因此，節點應該在：

* 左子樹處理完之後
* 右子樹處理之前

才輸出。

##### 實作程式碼

```java
LinkedList<TreeNode> stack = new LinkedList<>();
TreeNode curr = root;

while (!stack.isEmpty() || curr != null) {
    if (curr != null) {
        stack.push(curr);     // 記住回來的路
        curr = curr.left;     // 先一路往左
    } else {
        TreeNode node = stack.pop();  // 左子樹處理完，回到父節點
        System.out.println(node.val); // 中序：此時才輸出
        curr = node.right;            // 再處理右子樹
    }
}
```

##### 理解方式

中序遍歷最重要的一點就是：

> **一定要先走完左子樹，回來後才輸出當前節點。**

所以和前序不同，中序不是在第一次遇到節點時輸出，而是在：

```java
TreeNode node = stack.pop();
System.out.println(node.val);
```

這個回退時刻才輸出。

#### 六、後序遍歷（非遞歸）

##### 規則

後序遍歷的順序是：

```text
左 → 右 → 根
```

也就是說，只有在：

* 左子樹處理完
* 右子樹也處理完

之後，才能輸出目前節點。

這就是後序遍歷最難的地方。

#### 七、為什麼後序遍歷比較難

以前序和中序來說，某個節點彈棧之後，通常就不需要再回來了。
但是後序不同。

看下面這棵樹：

```text
      1
    /   \
   2     3
  / \   / \
 4  7  5   6
```

假設目前走到這裡：

```text
1 -> 2 -> 4 -> 2
```

棧內容大概是：

```text
| 2 |
| 1 |
-----
```

此時如果直接把 `2` 彈出來，就有問題了。
因為 `2` 的右子樹 `7` 還沒處理，之後從 `7` 回來時，就找不到 `2` 了。

所以後序遍歷不能像前序、中序那樣，看到機會就直接彈棧。
它必須先確認：

> **右子樹是不是已經處理完了。**

只有確認完成後，才能真正彈棧並輸出。

#### 八、如何判斷右子樹是否處理完成

做法是：

* 額外用一個變數 `pop`，記錄「最近一次彈棧的節點」
* 每次查看棧頂元素 `peek`
* 如果：

    * `peek.right == null`，表示沒有右子樹
    * 或 `peek.right == pop`，表示右子樹剛剛處理完

那就可以安全地彈棧並輸出

#### 九、後序遍歷的非遞歸程式碼

```java
LinkedList<TreeNode> stack = new LinkedList<>();
TreeNode curr = root; // 當前節點
TreeNode pop = null;  // 最近一次彈棧的節點

while (!stack.isEmpty() || curr != null) {
    if (curr != null) {
        stack.push(curr);   // 記住回來的路
        curr = curr.left;   // 先往左走
    } else {
        TreeNode peek = stack.peek(); // 查看棧頂，但先不急著彈出

        if (peek.right == null || peek.right == pop) {
            pop = stack.pop();              // 可以安全彈棧
            System.out.println(pop.val);    // 後序：左右都完成後才輸出
        } else {
            curr = peek.right;              // 右子樹還沒處理，先去右邊
        }
    }
}
```

#### 十、後序遍歷的核心理解

這段程式最重要的地方在這裡：

```java
if (peek.right == null || peek.right == pop)
```

它的意思是：

* 如果右子樹不存在，當然可以處理目前節點
* 如果右子樹存在，但剛剛已經處理完了，也可以處理目前節點
* 否則就還不能彈棧，必須先去處理右子樹

所以後序遍歷其實就是：

> **每個節點都要等左右子樹都完成後，才能真正輸出。**

#### 十一、可以把三種遍歷理解成同一套骨架

其實三種非遞歸 DFS 的流程骨架幾乎一樣：

1. 能往左就一直往左
2. 走不下去就回頭看棧頂
3. 判斷現在該不該輸出
4. 如果右子樹還沒處理，就去右邊
5. 如果可以結束這個節點，就彈棧

也就是說：

> **走法幾乎一樣，差別只在於節點何時被處理。**

##### 統一觀念版程式骨架

下面這段程式，可以視為三種遍歷共同的基礎框架：

```java
LinkedList<TreeNode> stack = new LinkedList<>();
TreeNode curr = root;
TreeNode pop = null;

while (!stack.isEmpty() || curr != null) {
    if (curr != null) {
        stack.push(curr);
        curr = curr.left;
    } else {
        TreeNode peek = stack.peek();

        if (peek.right == null || peek.right == pop) {
            pop = stack.pop();
        } else {
            curr = peek.right;
        }
    }
}
```

這段程式先把「走訪路徑」建立起來。
之後只要根據不同遍歷方式，把「輸出的時機」放到正確的位置，就可以得到前序、中序、後序。

# LeetCode 101 - 對稱二叉樹

> [101. 对称二叉树](https://leetcode.cn/problems/symmetric-tree/description/)

題目要我們判斷：**一棵二叉樹是否左右對稱**。

例如下面這棵樹就是對稱的：

```text
                1
            /         \
           2           2
         /  \         /  \
        3     4      4     3
       / \   / \    / \   / \
      5   6 7   8   8 7  6   5
```

## 一、什麼叫做對稱？

所謂對稱，不是單純看左右子樹「一模一樣」，而是要看它們是否互為鏡像。

也就是說：

* 左子樹的左孩子，要對應右子樹的右孩子
* 左子樹的右孩子，要對應右子樹的左孩子

所以比較方式不是：

* `left.left` 和 `right.left`
* `left.right` 和 `right.right`

而是：

* `left.left` 和 `right.right`
* `left.right` 和 `right.left`

這一點非常重要。

## 二、檢查的思路

以根節點 `1` 為例，根節點本身不用比，真正要比較的是它的左右子樹。

檢查流程如下：

1. 先比較根節點左子樹和右子樹的根節點值是否相同
2. 再比較：

    * 左邊節點的左孩子 vs 右邊節點的右孩子
    * 左邊節點的右孩子 vs 右邊節點的左孩子
3. 一路往下遞歸比較
4. 只要有任意一組不符合對稱條件，就不是對稱二叉樹

## 三、遞歸函數的設計

我們可以設計一個函數：

```java
private boolean check(TreeNode left, TreeNode right) {

}
```

這個函數的意思是：

> 檢查 `left` 和 `right` 這兩個節點所代表的子樹，是否互為鏡像。

## 四、先處理遞歸的終止條件

### 情況 1：左右節點都為 null

如果兩邊都沒有節點，代表它們是對稱的，返回 `true`。

```text
    1
```

```java
if (left == null && right == null) {
    return true;
}
```

### 情況 2：其中一邊為 null，另一邊不為 null

如果只有一邊有節點，另一邊沒有，這一定不對稱，返回 `false`。

例如：

```text
    1
   /
  2
```

或

```text
    1
      \
       2
```

```java
private boolean check(TreeNode left, TreeNode right) {
    if (left == null && right == null) {
        return true;
    }
    // 能執行到這裡說明 left 和 right 不能同時為 null
    // 在這種情況下 left != null 或者 right != null 就顯得多餘了
    // 所以我們可以把它去掉
    if (left != null && right == null || left == null && right != null) {
        return false;
    }
}
```

這裡可以直接寫成下面這種簡化形式。

```java
if (left == null || right == null) {
    return false;
}
```

因為前面已經處理過「兩邊都為 `null`」的情況了，能走到這裡，表示不可能同時都為 `null`，所以只要有一邊是 `null`，就代表另一邊不是 `null`。

## 五、再處理節點值不同的情況

如果左右兩個節點都存在，但值不一樣，那也不對稱。

```java
if (left.val != right.val) {
    return false;
}
```

這裡要特別注意一件事：

即使 `left.val == right.val`，也**不能直接返回 `true`**，因為它們下面的子樹未必對稱。

例如下面這棵樹就不是對稱的：

```text
    1
   / \
  2   2
 /     \
3
```

雖然左右兩個 `2` 的值相同，但它們的子節點結構不同，所以整棵樹仍然不是對稱的。

因此，值相等只是「其中一個條件」，還必須繼續往下檢查子樹。

## 六、遞歸檢查下一層

當前兩個節點值相同後，接下來要繼續比較它們的孩子節點是否鏡像對稱：

* `left.left` 要和 `right.right` 比
* `left.right` 要和 `right.left` 比

所以遞歸寫法如下：

```java
return check(left.left, right.right) && check(left.right, right.left);
```

只有這兩組都成立，當前這兩棵子樹才算對稱。

## 七、完整程式碼

```java
class Solution {
    public boolean isSymmetric(TreeNode root) {
        if (root == null) {
            return true;
        }
        return check(root.left, root.right);
    }

    private boolean check(TreeNode left, TreeNode right) {
        // 1. 兩邊都為 null，對稱
        if (left == null && right == null) {
            return true;
        }

        // 2. 一邊為 null，一邊不為 null，不對稱
        if (left == null || right == null) {
            return false;
        }

        // 3. 節點值不同，不對稱
        if (left.val != right.val) {
            return false;
        }

        // 4. 遞歸比較外側和內側
        return check(left.left, right.right) && check(left.right, right.left);
    }
}
```

# LeetCode 104 - 二叉樹的最大深度

> [104. 二叉樹的最大深度](https://leetcode.cn/problems/maximum-depth-of-binary-tree/description/)

## 題目理解

這題要我們求出一棵二叉樹的**最大深度**。

所謂最大深度，可以理解成：

* 從**根節點**出發
* 一直到**最遠的葉子節點**
* 這條路徑上總共有多少個節點

也就是說，**LeetCode 這題的深度是用「節點數」來算的**。
如果你平常習慣用「邊數」來理解，那麼這題的答案就會比邊數 **多 1**。

## 先看幾個例子

### 例子 1

```text
深度 2
    1
   / \
  2   3
```

從根節點 `1` 走到最遠的葉子節點，最多會經過 `2` 個節點，所以深度是 `2`。

### 例子 2

```text
深度 3
    1
   / \
  2   3
       \
        4
```

最長路徑是 `1 -> 3 -> 4`，一共經過 `3` 個節點，所以深度是 `3`。

### 例子 3

```text
深度 1
    1
```

只有一個根節點，深度就是 `1`。

## 方式一：遞迴

這題最直觀的解法，就是使用**遞迴**。

### 核心想法

如果我們想知道某個節點的最大深度，其實只需要知道兩件事：

1. 它的**左子樹最大深度**
2. 它的**右子樹最大深度**

然後取兩者中比較大的那一個，再加上目前這個節點自己。

也就是：

```text
當前節點的最大深度 = max(左子樹深度, 右子樹深度) + 1
```

### 用例子理解

例如下面這棵樹：

```text
深度 3
    1
   / \
  2   3
       \
        4
```

如果我們想知道根節點 `1` 的最大深度，就可以拆成：

* 先求 `1` 的左子樹深度
* 再求 `1` 的右子樹深度
* 最後取較大的那一個，再加 `1`

那如果我們想知道節點 `3` 的最大深度呢？
做法也完全一樣：

* 先求 `3` 的左子樹深度
* 再求 `3` 的右子樹深度
* 取最大值後加 `1`

你會發現，**大問題可以一直拆成相同形式的小問題**，這就是遞迴很適合這題的原因。

### 遞迴什麼時候停止？

遞迴一定要有終止條件。

這題最基本的終止條件是：

* 如果當前節點是 `null`
* 那它的深度就是 `0`

為什麼可以這樣定義？

因為空節點不存在，自然沒有深度，所以回傳 `0` 是最合理的。

### 程式碼寫法

```java
public int maxDepth(TreeNode node) {
    // 如果節點為 null，深度就是 0
    if (node == null) {
        return 0;
    }

    int d1 = maxDepth(node.left);
    int d2 = maxDepth(node.right);

    // 左右子樹取較大值，再加上當前節點自己
    return Math.max(d1, d2) + 1;
}
```

#### 為什麼這樣寫是對的？

假設目前傳入的是葉子節點 `4`：

```text
    4
```

那麼：

* `node.left` 是 `null`
* `node.right` 也是 `null`

所以程式會變成：

```java
int d1 = maxDepth(node.left);   // 0
int d2 = maxDepth(node.right);  // 0
return Math.max(d1, d2) + 1;    // 1
```

結果就是 `1`，代表葉子節點的深度是 `1`，這完全符合題意。

接著再一路往上回推，就能算出整棵樹的最大深度。

#### 一個容易多寫的判斷

有些人一開始會這樣寫：

```java
if (node.left == null && node.right == null) {
    return 1;
}
```

這段的意思是：

* 如果目前節點是葉子節點
* 那就直接回傳 `1`

這樣寫**沒有錯**，但其實**可以省略**。

因為即使你不特別判斷葉子節點，程式也會自然算出正確答案：

```java
int d1 = maxDepth(node.left);   // 左子樹是 null，回傳 0
int d2 = maxDepth(node.right);  // 右子樹是 null，回傳 0
return Math.max(d1, d2) + 1;    // 1
```

所以這段葉子節點判斷不是必要的。

#### 更精簡的遞迴版本

```java
public int maxDepth(TreeNode node) {
    // 空節點的深度為 0
    if (node == null) {
        return 0;
    }

    int leftDepth = maxDepth(node.left);
    int rightDepth = maxDepth(node.right);

    return Math.max(leftDepth, rightDepth) + 1;
}
```

## 方式二：用非遞迴後序遍歷求最大深度

除了遞迴以外，這題也可以用**非遞迴的後序遍歷**來解。

這種做法的核心觀念是：

> 在遍歷過程中，**棧裡目前有多少個節點**，就代表「從根節點走到當前位置」的深度。

因此，我們只要在遍歷過程中持續記錄棧的大小，並更新最大值，最後就能得到整棵樹的最大深度。

### 先看範例

```text
              1
           /     \
          2        3
         / \        \
        4   5        6
           /
          7
```

這棵樹的最大深度其實是 `4`，因為最長路徑是：

```text
1 -> 2 -> 5 -> 7
```

### 為什麼後序遍歷可以求最大深度？

我們先回憶一下，**非遞迴後序遍歷**通常會搭配一個**棧**來做。

遍歷時，我們會一路往左走，並把沿途經過的節點都壓入棧中。

例如一開始從 `1 -> 2 -> 4` 一路往左走到底，棧會變成：

```text
| 4 |
| 2 |
| 1 |
-----
```

此時棧中一共有 `3` 個節點，代表目前從根節點 `1` 走到節點 `4` 的深度就是 `3`。

### 棧的大小，其實就是當前路徑深度

接著遍歷繼續進行。

當我們走到另一條更深的路徑 `1 -> 2 -> 5 -> 7` 時，棧會變成：

```text
| 7 |
| 5 |
| 2 |
| 1 |
-----
```

這時候棧中共有 `4` 個節點，表示目前這條路徑的深度是 `4`。

因此，我們只要：

* 每次節點入棧時
* 取得目前棧的大小
* 用它更新最大深度

就能在整個遍歷過程中找出最深的那條路徑。

### 整體思路

這個方法可以分成兩個重點：

#### 1. 用非遞迴方式完成後序遍歷

後序遍歷順序是：

```text
左子樹 -> 右子樹 -> 根節點
```

因為不能用遞迴，所以要自己用棧模擬整個過程。

#### 2. 在入棧時更新最大深度

每當一個節點被壓入棧中，代表目前路徑又往下走了一層。
這時候：

* `stack.size()` 就是目前深度
* 如果比之前記錄的最大深度還大，就更新答案

### 程式碼

```java
public int maxDepth(TreeNode root) {
    TreeNode curr = root;
    TreeNode pop = null;
    LinkedList<TreeNode> stack = new LinkedList<>();
    int maxDepth = 0;

    while (!stack.isEmpty() || curr != null) {
        if (curr != null) {
            stack.push(curr);

            int size = stack.size();
            if (size > maxDepth) {
                maxDepth = size;
            }

            curr = curr.left;
        } else {
            TreeNode peek = stack.peek();

            if (peek.right == null || peek.right == pop) {
                pop = stack.pop();
            } else {
                curr = peek.right;
            }
        }
    }
    return maxDepth;
}
```

### 程式碼解釋

#### 變數說明

```java
TreeNode curr = root;
```

* `curr` 表示目前正在處理的節點。

```java
TreeNode pop = null;
```

* `pop` 用來記錄**最近一次彈出的節點**。
* 它的作用是幫助我們判斷：某個節點的右子樹是不是已經處理過了。

```java
LinkedList<TreeNode> stack = new LinkedList<>();
```

* `stack` 用來模擬遞迴過程中的呼叫堆疊。

```java
int maxDepth = 0;
```

* 用來記錄目前為止找到的最大深度。

#### while 迴圈在做什麼？

```java
while (!stack.isEmpty() || curr != null)
```

只要還有節點沒處理，就繼續遍歷。

情況有兩種：

* `curr != null`：表示目前還能繼續往下走
* `stack` 不為空：表示雖然走到底了，但還有回頭處理的節點

#### 情況一：一路往左走

```java
if (curr != null) {
    stack.push(curr);

    int size = stack.size();
    if (size > maxDepth) {
        maxDepth = size;
    }

    curr = curr.left;
}
```

這段邏輯表示：

1. 先把當前節點壓入棧中
2. 棧大小就是目前路徑深度
3. 用它更新 `maxDepth`
4. 再繼續往左子樹走

這就是「一路往左深入」的過程。

#### 情況二：左邊走到底，準備回頭

```java
else {
    TreeNode peek = stack.peek();

    if (peek.right == null || peek.right == pop) {
        pop = stack.pop();
    } else {
        curr = peek.right;
    }
}
```

這段是非遞迴後序遍歷最關鍵的部分。

##### peek = stack.peek()

先看看棧頂節點是誰，但先不急著彈出。

#### 什麼時候可以彈出棧頂節點？

```java
if (peek.right == null || peek.right == pop)
```

有兩種情況可以把棧頂彈出：

##### 情況 1：沒有右子樹

如果 `peek.right == null`，表示：

* 左子樹已經處理完
* 右子樹也不存在

那這個節點就可以正式處理完畢，直接彈出。

##### 情況 2：右子樹已經處理過

如果 `peek.right == pop`，表示：

* 這個節點的右子樹剛剛才處理完
* 左右子樹都完成了

所以這個節點也可以彈出。


##### 否則就去處理右子樹

```java
else {
    curr = peek.right;
}
```

如果右子樹存在，而且還沒處理過，那就先轉去處理右子樹。

這樣才能符合後序遍歷的順序：

```text
左 -> 右 -> 根
```

### 為什麼這樣就能求出最大深度？

因為在整個遍歷過程中：

* 棧中保存的，永遠都是「從根到目前位置」這條路徑上的節點
* 所以棧的大小，正好就是當前路徑深度
* 我們只要持續更新最大值，最後得到的就是整棵樹的最大深度

換句話說，這題雖然表面上是在做後序遍歷，但本質上是在利用：

> **棧保存當前路徑** 這件事，順便統計最大深度。

## 方式三：層序遍歷

這題也可以用**層序遍歷（BFS）**來解，而且這種方式非常直觀。

### 核心觀念

二叉樹的**最大深度**，其實就等於這棵樹的**層數**。

也就是說：

* 如果一棵樹總共有 `4` 層
* 那它的最大深度就是 `4`

例如下面這棵樹：

```text
              1
           /     \
          2        3
         / \        \
        4   5        6
           /
          7
```

它一共有 4 層：

* 第 1 層：`1`
* 第 2 層：`2、3`
* 第 3 層：`4、5、6`
* 第 4 層：`7`

所以這棵樹的最大深度就是 `4`。

### 為什麼層序遍歷適合這題？

層序遍歷的順序就是：

```text
一層一層，由上往下遍歷
```

例如上面的樹，層序遍歷結果會是：

```text
1   2   3   4   5   6   7
```

但如果只是單純印出這樣的結果，我們雖然知道節點的拜訪順序，卻**不知道哪些節點屬於同一層**。

所以，這題的關鍵不是只做層序遍歷，而是要想辦法：

> **把每一層清楚地區分開來**

只要能區分層與層，我們就能知道總共有幾層，也就知道最大深度了。

### 先看最基本的層序遍歷

層序遍歷通常會搭配**佇列（Queue）**來實作。

```java
public int maxDepth(TreeNode root) {
    if (root == null) {
        return 0;
    }

    Queue<TreeNode> queue = new LinkedList<>();
    queue.offer(root);

    while (!queue.isEmpty()) {
        TreeNode poll = queue.poll();
        System.out.print(poll.val + "\t");

        if (poll.left != null) {
            queue.offer(poll.left);
        }
        if (poll.right != null) {
            queue.offer(poll.right);
        }
    }
    return 0;
}
```

輸出結果會是：

```text
1   2   3   4   5   6   7
```

這樣雖然完成了層序遍歷，但還是沒辦法直接知道：

* 第一層有哪些節點
* 第二層有哪些節點
* 第三層有哪些節點

因此，我們還需要再進一步處理。

### 做法一：自己記錄每層有幾個節點

我們可以先定義一個變數 `c1`，用來表示「當前這一層有幾個節點」。

例如：

* 第 1 層一開始只有根節點，所以 `c1 = 1`
* 接著在處理這一層時，順便統計下一層有幾個節點，記成 `c2`
* 一層處理完後，再把 `c2` 指派給 `c1`

這樣下一輪就知道要處理幾個節點了。

```java
public int maxDepth(TreeNode root) {
    if (root == null) {
        return 0;
    }

    Queue<TreeNode> queue = new LinkedList<>();
    queue.offer(root);

    int c1 = 1; // 當前層節點數，第一層只有根節點

    while (!queue.isEmpty()) {
        int c2 = 0; // 下一層節點數

        for (int i = 0; i < c1; i++) {
            TreeNode poll = queue.poll();
            System.out.print(poll.val + "\t");

            if (poll.left != null) {
                queue.offer(poll.left);
                c2++;
            }
            if (poll.right != null) {
                queue.offer(poll.right);
                c2++;
            }
        }

        c1 = c2; // 更新為下一層的節點數
        System.out.println();
    }

    return 0;
}
```

這樣輸出時就能清楚看出每一層：

```text
1
2   3
4   5   6
7
```

### 做法二：直接利用 queue.size()

其實上面的 `c1` 可以不用自己維護，因為佇列本身就提供了 `size()` 方法。

在每一輪 `while` 開始時：

* `queue.size()` 就表示「目前這一層有幾個節點」

所以我們可以直接用它來取代 `c1`，讓程式更簡潔。

```java
public int maxDepth(TreeNode root) {
    if (root == null) {
        return 0;
    }

    Queue<TreeNode> queue = new LinkedList<>();
    queue.offer(root);

    while (!queue.isEmpty()) {
        int size = queue.size(); // 目前這一層的節點數

        for (int i = 0; i < size; i++) {
            TreeNode poll = queue.poll();
            System.out.print(poll.val + "\t");

            if (poll.left != null) {
                queue.offer(poll.left);
            }
            if (poll.right != null) {
                queue.offer(poll.right);
            }
        }

        System.out.println();
    }

    return 0;
}
```

這樣就能在遍歷時自然地把每一層切分開來。

### 套用到這題：統計層數

既然我們已經能夠一層一層地遍歷，那麼這題就很簡單了。

我們只要再加一個變數 `depth`：

* 每處理完一層
* 就把 `depth` 加一

最後 `depth` 的值，就是整棵樹的最大深度。

### 最終程式碼

```java
public int maxDepth(TreeNode root) {
    if (root == null) {
        return 0;
    }

    Queue<TreeNode> queue = new LinkedList<>();
    queue.offer(root);

    int depth = 0; // 記錄層數，也就是最大深度

    while (!queue.isEmpty()) {
        int size = queue.size(); // 目前這一層的節點數

        for (int i = 0; i < size; i++) {
            TreeNode poll = queue.poll();

            if (poll.left != null) {
                queue.offer(poll.left);
            }
            if (poll.right != null) {
                queue.offer(poll.right);
            }
        }

        depth++; // 每處理完一層，深度加一
    }

    return depth;
}
```

#### 程式碼思路整理

這段程式的邏輯可以簡單整理成下面幾步：

1. 先把根節點放進佇列
2. 每次進入 `while` 時，先記下目前這一層有幾個節點
3. 用 `for` 迴圈把這一層的節點全部處理完
4. 處理過程中，把下一層的子節點放進佇列
5. 一層處理完後，`depth++`
6. 直到佇列為空，表示整棵樹都遍歷完畢

### 為什麼這樣是對的？

因為層序遍歷本來就是**一層一層往下走**：

* 第一次 `while`：處理第 1 層
* 第二次 `while`：處理第 2 層
* 第三次 `while`：處理第 3 層
* …

所以每完整跑完一次 `while`，就代表處理完一層。
既然最大深度就是層數，那麼每處理完一層就把 `depth` 加一，最後得到的自然就是答案。

# LeetCode 111 - 二叉樹的最小深度

> [111. 二叉树的最小深度](https://leetcode.cn/problems/minimum-depth-of-binary-tree/description/)

## 題目說明

給定一棵二叉樹，請找出它的**最小深度**。

這裡的最小深度，指的是：

> **從根節點到最近葉子節點的最短路徑上的節點數量。**

注意，**葉子節點**必須是「左右子節點都為 `null`」的節點。

## 遞迴解法（後序遍歷）

### 為什麼不能直接照抄「最大深度」的寫法？

很多人在做這題時，會很自然地想到最大深度的寫法：

```java
public int minDepth(TreeNode node) {
    if (node == null) {
        return 0;
    }
    int d1 = minDepth(node.left);
    int d2 = minDepth(node.right);
    return Math.min(d1, d2) + 1;
}
```

看起來很合理：
左子樹深度算出來、右子樹深度算出來，再取比較小的那個加一。

但這樣其實是**錯的**。

### 錯誤發生在哪裡？

來看下面這棵樹：

```text
深度 2
    1
   /
  2
```

對應測試：

```java
@Test
public void test() {
    TreeNode root = new TreeNode(
        new TreeNode(2),
        1,
        null
    );
    assertEquals(2, new LeetCode111_1().minDepth(root));
}
```

正確答案應該是 `2`，但上面的寫法會得到 `1`。

### 原因分析

當根節點 `1` 呼叫 `minDepth` 時：

* 左子樹深度 `d1 = 1`
* 右子樹深度 `d2 = 0`，因為右子樹是空的

此時執行：

```java
Math.min(d1, d2) + 1
```

也就是：

```java
Math.min(1, 0) + 1 = 1
```

這樣就錯了。

### 核心觀念：空子樹不能拿來當作最小值比較

問題的關鍵在於：

> **最小深度一定要走到葉子節點。**

如果某一邊子樹是空的，代表那一邊根本**不是一條能走到葉子節點的路**，因此不能直接拿 `0` 來和另一邊比較最小值。

也就是說：

* 如果左子樹是空的，就只能走右子樹
* 如果右子樹是空的，就只能走左子樹
* 只有當左右子樹都存在時，才能取 `Math.min(...)`

### 正確寫法

```java
public int minDepth(TreeNode node) {
    if (node == null) {
        return 0;
    }

    int d1 = minDepth(node.left);
    int d2 = minDepth(node.right);

    // 左子樹為空，只能走右子樹
    if (d1 == 0) {
        return d2 + 1;
    }

    // 右子樹為空，只能走左子樹
    if (d2 == 0) {
        return d1 + 1;
    }

    // 左右子樹都存在，取較小值
    return Math.min(d1, d2) + 1;
}
```

### 遞迴解法思路整理

這個做法本質上是 **後序遍歷**，因為要先知道左子樹和右子樹的最小深度，才能回頭計算目前節點的答案。

可以分成三種情況：

#### 1. 當前節點為 null

```java
if (node == null) {
    return 0;
}
```

空樹深度就是 `0`。

#### 2. 某一邊子樹為空

例如：

```text
    1
   /
  2
```

右子樹為空時，不能取 `min(1, 0)`，因為右邊根本不是有效路徑。
這時只能回傳：

```java
左子樹深度 + 1
```

#### 3. 左右子樹都存在

這時才真正能取左右子樹中較小的那一邊：

```java
return Math.min(d1, d2) + 1;
```

## 層序遍歷解法

除了遞迴以外，這題還有一個很好理解，而且通常更直觀的做法：**層序遍歷（BFS）**。

### 關鍵觀念

> **在層序遍歷中，第一個遇到的葉子節點，它所在的層數就是最小深度。**

因為 BFS 是一層一層往下找，所以最先找到的葉子節點，一定是距離根節點最近的。

### 舉例說明

```text
              1
           /     \
          2       3
         / \
        4   5
           /
          7
```

葉子節點有：

* `4`
* `3`
* `7`

雖然 `4` 也是葉子節點，但在層序遍歷中，會先遇到 `3`，而 `3` 所在層數是 `2`，所以最小深度就是 `2`。

### 層序遍歷程式碼

```java
public int minDepth(TreeNode root) {
    if (root == null) {
        return 0;
    }

    Queue<TreeNode> queue = new LinkedList<>();
    queue.offer(root);

    int depth = 0;

    while (!queue.isEmpty()) {
        int size = queue.size();
        depth++;

        for (int i = 0; i < size; i++) {
            TreeNode node = queue.poll();

            // 遇到第一個葉子節點，直接回傳目前層數
            if (node.left == null && node.right == null) {
                return depth;
            }

            if (node.left != null) {
                queue.offer(node.left);
            }
            if (node.right != null) {
                queue.offer(node.right);
            }
        }
    }

    return depth;
}
```

### 為什麼 BFS 遇到第一個葉子節點就可以直接返回？

因為 BFS 的順序是：

* 先走第 1 層
* 再走第 2 層
* 再走第 3 層
* ...

所以當你第一次遇到葉子節點時，表示這個葉子節點一定是**離根最近的葉子節點**，也就代表找到了最小深度。

這也是這題用 BFS 很適合的原因。

# LeetCode 226 - 翻轉二叉樹

> [226. 翻转二叉树](https://leetcode.cn/problems/invert-binary-tree/description/)

## 題目在做什麼？

所謂的**翻轉二叉樹**，就是把樹中每一個節點的：

* 左子樹和右子樹互相交換

也就是說，對每個節點來看：

* 原本的左孩子變成右孩子
* 原本的右孩子變成左孩子

## 先用例子理解

原本的二叉樹如下：

```text
              1
           /     \
          2       3
         / \
        4   5
           / \
          7   8
```

### 第一步：翻轉節點 1

把 `1` 的左子樹和右子樹交換：

```text
              1
           /     \
          3       2
                 / \
                4   5
                   / \
                  7   8
```

### 第二步：翻轉節點 3

節點 `3` 沒有左右孩子，所以不用處理。

### 第三步：翻轉節點 2

把 `2` 的左孩子 `4` 和右孩子 `5` 交換：

```text
              1
           /     \
          3       2
                 / \
                5   4
               / \
              7   8
```

### 第四步：翻轉節點 5

把 `5` 的左孩子 `7` 和右孩子 `8` 交換：

```text
              1
           /     \
          3       2
                 / \
                5   4
               / \
              8   7
```

### 第五步：翻轉節點 4、7、8

這三個節點都沒有左右孩子，所以不用再交換。

### 最終結果

```text
              1
           /     \
          3       2
                 / \
                5   4
               / \
              8   7
```

## 解題思路

這題的核心其實很簡單：

1. 先交換目前節點的左右孩子
2. 再去處理它的左子樹
3. 再去處理它的右子樹

因為**每個節點都做同樣的事情**，所以非常適合用**遞迴**來解。

## Java 程式碼

```java
public TreeNode invertTree(TreeNode root) {
    fn(root);
    return root;
}

private static void fn(TreeNode node) {
    if (node == null) {
        return;
    }

    TreeNode temp = node.left;
    node.left = node.right;
    node.right = temp;

    fn(node.left);
    fn(node.right);
}
```

## 程式碼說明

### invertTree

```java
public TreeNode invertTree(TreeNode root) {
    fn(root);
    return root;
}
```

這個方法是題目要求的入口。

* 傳入根節點 `root`
* 呼叫 `fn(root)` 去翻轉整棵樹
* 最後回傳翻轉後的根節點

### fn(node)

```java
private static void fn(TreeNode node) {
    if (node == null) {
        return;
    }

    TreeNode temp = node.left;
    node.left = node.right;
    node.right = temp;

    fn(node.left);
    fn(node.right);
}
```

這個遞迴方法在做三件事：

#### 1. 終止條件

```java
if (node == null) {
    return;
}
```

如果目前節點是 `null`，代表到底了，就直接結束。

#### 2. 交換左右孩子

```java
TreeNode temp = node.left;
node.left = node.right;
node.right = temp;
```

這裡先用 `temp` 暫存左孩子，再完成左右交換。

#### 3. 遞迴處理左右子樹

```java
fn(node.left);
fn(node.right);
```

交換完目前節點後，繼續翻轉它的左右子樹。

## 為什麼可以用遞迴？

因為這題有很明顯的「子問題結構」：

* 翻轉整棵樹
  = 先翻轉根節點

    * 再翻轉左子樹
    * 再翻轉右子樹

而左子樹、右子樹本身又都是一棵二叉樹，所以可以用相同邏輯一直往下處理。

這就是遞迴最適合的地方。

# 根據後綴表達式構造表達式樹

在學這個主題之前，先回憶一下什麼是**中綴表達式**與**後綴表達式**。

* 中綴表達式：`(2 - 1) * 3`
* 後綴表達式：`2 1 - 3 *`

而這個後綴表達式，其實可以對應成一棵**表達式樹**：

```text
    *
   / \
  -   3
 / \
2   1
```

## 一、什麼是表達式樹？

表達式樹本質上是一棵二叉樹：

* **葉子節點**：操作數，例如 `2`、`1`、`3`
* **非葉子節點**：運算符，例如 `-`、`*`

以上面的例子來看：

* `-` 負責計算 `2 - 1`
* `*` 再把 `(2 - 1)` 和 `3` 相乘

所以整棵樹就代表 `(2 - 1) * 3`。

## 二、為什麼要把後綴表達式轉成樹？

如果只是把表達式存成字串，例如：

```text
(2 - 1) * 3
```

雖然人類容易看懂，但對程式來說，字串不方便後續處理。

如果轉成樹狀結構，就能更方便做這些事：

* 計算結果
* 轉回中綴 / 前綴 / 後綴表達式
* 做語法分析
* 進一步優化表達式

所以在很多編譯器或運算系統中，表達式通常都不是直接存成字串，而是存成樹。

## 三、核心思路：使用棧

這題的解法，和「計算後綴表達式」很像，核心工具都是 **棧（stack）**。

### 前提條件

要能順利構造表達式樹，必須先假設：

1. 後綴表達式是**合法的**
2. 運算符都是**二元運算符**

    * 也就是每個運算符都只需要 **兩個操作數**
    * 例如 `+`、`-`、`*`、`/`

## 四、構造流程

我們從左到右遍歷後綴表達式：

```text
2 1 - 3 *
```

### 1. 遇到操作數：直接入棧

先讀到 `2`，它是數字，所以直接入棧：

```text
|   |
| 2 |
-----
```

再讀到 `1`，也是數字，繼續入棧：

```text
|   |
| 1 |
| 2 |
-----
```

#### 為什麼操作數要先入棧？

因為在後綴表達式中，**運算符會出現在操作數後面**。
也就是說，當父節點（運算符）還沒出現時，我們必須先把這些操作數暫存起來，等之後遇到運算符時，再拿出來建立父子關係。

### 2. 遇到運算符：從棧中取出兩個節點

接下來讀到 `-`，代表要建立一個新的父節點。

這時要特別注意：

* **第一次彈出的節點，是右子樹**
* **第二次彈出的節點，是左子樹**

所以：

* 先彈出 `1` → 它是 `-` 的右孩子
* 再彈出 `2` → 它是 `-` 的左孩子

建立後的樹如下：

```text
    -
   / \
  2   1
```

也就是：

```text
-.left = 2
-.right = 1
```

然後再把這個新建好的 `-` 節點重新入棧：

```text
|   |
| - |
-----
```

#### 為什麼運算符節點還要再入棧？

因為這個 `-` 節點雖然已經有左右孩子了，但它還可能繼續和其他節點組成更大的子樹。

也就是說，它接下來還可能成為別人的左子樹或右子樹，所以必須先放回棧中等待後續處理。

### 3. 繼續遍歷

接下來讀到 `3`，它是數字，直接入棧：

```text
|   |
| 3 |
| - |
-----
```

再來讀到 `*`，表示又要建立新的父節點。

依照規則：

* 先彈出 `3` → 它是 `*` 的右孩子
* 再彈出 `-` → 它是 `*` 的左孩子

建立後：

```text
    *
   / \
  -   3
```

其中 `-` 節點本身又有左右孩子，所以整棵樹就是：

```text
    *
   / \
  -   3
 / \
2   1
```

再把 `*` 入棧：

```text
|   |
| * |
-----
```

## 五、最後結果

當所有 token 都遍歷完之後，棧中只會剩下一個元素。

這個元素就是**整棵表達式樹的根節點**，也就是：

```text
    *
   / \
  -   3
 / \
2   1
```

## 六、整體規律總結

整個過程可以濃縮成兩條規則：

### 規則 1：遇到操作數

建立節點，直接入棧。

### 規則 2：遇到運算符

從棧中彈出兩個節點：

* 第一個彈出的是**右孩子**
* 第二個彈出的是**左孩子**

再建立新的運算符節點，接上左右孩子後重新入棧。

## 七、Java 程式碼實作

```java
public TreeNode constructExpressionTree(String[] tokens) {
    LinkedList<TreeNode> stack = new LinkedList<>();

    for (String token : tokens) {
        switch (token) {
            case "+", "-", "*", "/" -> {
                TreeNode right = stack.pop();
                TreeNode left = stack.pop();

                TreeNode root = new TreeNode(token);
                root.left = left;
                root.right = right;

                stack.push(root);
            }
            default -> {
                stack.push(new TreeNode(token));
            }
        }
    }

    return stack.peek();
}
```

## 八、怎麼驗證構造出來的樹是正確的？

有一個很好用的方法：

> **對這棵表達式樹做後序遍歷，結果應該剛好就是原本的後綴表達式。**

例如這棵樹：

```text
    *
   / \
  -   3
 / \
2   1
```

對它做後序遍歷：

1. 先遍歷左子樹 → `2 1 -`
2. 再遍歷右子樹 → `3`
3. 最後訪問根節點 → `*`

得到結果：

```text
2 1 - 3 *
```

這正好就是原本的後綴表達式。

所以只要後序遍歷的結果與原始後綴表達式一致，就能說明構造出的表達式樹是正確的。

# LeetCode 105 - 從前序與中序遍歷序列構造二叉樹

> [105. 从前序与中序遍历序列构造二叉树](https://leetcode.cn/problems/construct-binary-tree-from-preorder-and-inorder-traversal/description/?envType=study-plan-v2&envId=top-100-liked)

## 一、題目範例

```text
前序遍歷 preorder = [1, 2, 4, 3, 6, 7]
中序遍歷 inorder  = [4, 2, 1, 6, 3, 7]
```

根據這兩組序列，我們要還原出這棵二叉樹：

```text
              1
           /     \
          2       3
         /       / \
        4       6   7
```

## 二、解題關鍵

要解這題，必須先掌握兩種遍歷方式的特性。

### 1. 前序遍歷

順序是：

```text
根節點 -> 左子樹 -> 右子樹
```

所以：

* 前序遍歷的**第一個元素一定是根節點**

### 2. 中序遍歷

順序是：

```text
左子樹 -> 根節點 -> 右子樹
```

所以：

* 在中序遍歷中，**根節點左邊的元素都屬於左子樹**
* **根節點右邊的元素都屬於右子樹**

## 三、先看整體怎麼切

### 第一步：找根節點

前序遍歷是：

```text
[1, 2, 4, 3, 6, 7]
```

因為前序遍歷第一個值一定是根節點，所以：

```text
根節點 = 1
```

### 第二步：用中序遍歷切出左右子樹

中序遍歷是：

```text
[4, 2, 1, 6, 3, 7]
```

找到根節點 `1` 的位置後：

* 左邊 `[4, 2]` 是左子樹
* 右邊 `[6, 3, 7]` 是右子樹

所以目前可以知道：

```text
        1
       / \
   [4,2] [6,3,7]
```

但這時候還只是知道「哪些值屬於左子樹、哪些值屬於右子樹」，還不知道左右子樹內部的結構。

## 四、再看前序遍歷怎麼對應左右子樹

前序遍歷扣掉根節點 `1` 後，剩下：

```text
[2, 4, 3, 6, 7]
```

左子樹在中序遍歷中有 2 個元素 `[4, 2]`，所以前序遍歷中接下來的 2 個元素就是左子樹的前序遍歷：

* 左子樹前序：`[2, 4]`
* 右子樹前序：`[3, 6, 7]`

因此可以拆成兩個子問題：

```text
左子樹：
preorder = [2, 4]
inorder  = [4, 2]

右子樹：
preorder = [3, 6, 7]
inorder  = [6, 3, 7]
```

## 五、為什麼這題適合用遞迴？

因為你會發現：

* 原問題：根據一組 preorder 和 inorder 建樹
* 子問題：根據左子樹的 preorder 和 inorder 建樹
* 子問題：根據右子樹的 preorder 和 inorder 建樹

**每個子問題的形式都跟原問題一模一樣**，只是資料範圍變小而已。

這就是典型的遞迴。

## 六、用遞迴一步一步理解

### 處理左子樹

```text
preorder = [2, 4]
inorder  = [4, 2]
```

* 前序第一個值 `2` 是根節點
* 在中序中找到 `2`

```text
[4, 2]
 ^
 左邊是左子樹
```

所以：

* 左子樹中序：`[4]`
* 右子樹中序：`[]`

再根據左子樹大小去切前序：

* 左子樹前序：`[4]`
* 右子樹前序：`[]`

因此 `2` 的結構就是：

```text
    2
   /
  4
```

### 處理右子樹

```text
preorder = [3, 6, 7]
inorder  = [6, 3, 7]
```

* 前序第一個值 `3` 是根節點
* 在中序中找到 `3`

```text
[6, 3, 7]
 左  根  右
```

所以：

* 左子樹中序：`[6]`
* 右子樹中序：`[7]`

再根據左子樹大小切前序：

* 左子樹前序：`[6]`
* 右子樹前序：`[7]`

因此 `3` 的結構就是：

```text
    3
   / \
  6   7
```

### 最後組合起來

```text
              1
           /     \
          2       3
         /       / \
        4       6   7
```

## 七、解題流程總結

每次遞迴都做這幾件事：

### 1. 前序遍歷的第一個值就是根節點

### 2. 在中序遍歷中找到根節點位置

### 3. 根據這個位置切出左子樹與右子樹

### 4. 再依照左子樹的節點數量，去前序遍歷中切出對應範圍

### 5. 遞迴建立左子樹與右子樹

## 八、程式碼

```java
class Solution {
    public TreeNode buildTree(int[] preorder, int[] inorder) {
        // 遞迴終止條件
        if (preorder.length == 0 || inorder.length == 0) {
            return null;
        }

        // 前序遍歷的第一個元素就是根節點
        int rootValue = preorder[0];
        TreeNode root = new TreeNode(rootValue);

        // 在中序遍歷中找到根節點的位置
        for (int i = 0; i < inorder.length; i++) {
            if (inorder[i] == rootValue) {
                // 切出中序遍歷的左子樹與右子樹
                int[] leftInorder = Arrays.copyOfRange(inorder, 0, i);
                int[] rightInorder = Arrays.copyOfRange(inorder, i + 1, inorder.length);

                // 根據左子樹的長度，切出前序遍歷的左子樹與右子樹
                int[] leftPreorder = Arrays.copyOfRange(preorder, 1, 1 + leftInorder.length);
                int[] rightPreorder = Arrays.copyOfRange(preorder, 1 + leftInorder.length, preorder.length);

                // 遞迴建立左右子樹
                root.left = buildTree(leftPreorder, leftInorder);
                root.right = buildTree(rightPreorder, rightInorder);

                break;
            }
        }

        return root;
    }
}
```

# LeetCode 106 - 從中序與後序遍歷序列構造二叉樹

> [106. 从中序与后序遍历序列构造二叉树](https://leetcode.cn/problems/construct-binary-tree-from-inorder-and-postorder-traversal/description/)

## 一、題目範例

```text
中序遍歷 inorder   = [4, 2, 1, 6, 3, 7]
後序遍歷 postorder = [4, 2, 6, 7, 3, 1]
```

根據這兩組序列，我們要還原出這棵二叉樹：

```text
              1
           /     \
          2       3

        4       6   7
```

## 二、解題關鍵

這題和 **LeetCode 105（前序 + 中序建樹）** 很像，只是這次換成了 **中序 + 後序**。

因為左子樹在中序遍歷中有 2 個元素 `[4, 2]`，所以後序遍歷前 2 個元素就是左子樹：

* 子問題：根據左子樹的 inorder 和 postorder 建樹

## 六、解題流程總結

每次遞迴時都做同樣幾件事：

### 1. 後序遍歷最後一個元素就是根節點

### 2. 在中序遍歷中找到根節點位置

### 3. 根據這個位置切出左子樹與右子樹

### 4. 根據左子樹的節點數量，切出對應的後序遍歷

### 5. 遞迴建立左子樹與右子樹

## 七、程式碼思路

```java
class Solution {
    public TreeNode buildTree(int[] inorder, int[] postorder) {
        // 遞迴終止條件
        if (inorder.length == 0 || postorder.length == 0) {
            return null;
        }

        // 後序遍歷的最後一個元素就是根節點
        int rootValue = postorder[postorder.length - 1];
        TreeNode root = new TreeNode(rootValue);

        // 在中序遍歷中找到根節點的位置
        for (int i = 0; i < inorder.length; i++) {
            if (inorder[i] == rootValue) {
                // 切出中序遍歷的左右子樹
                int[] inLeft = Arrays.copyOfRange(inorder, 0, i);
                int[] inRight = Arrays.copyOfRange(inorder, i + 1, inorder.length);

                // 根據左子樹長度，切出後序遍歷的左右子樹
                int[] postLeft = Arrays.copyOfRange(postorder, 0, i);
                int[] postRight = Arrays.copyOfRange(postorder, i, postorder.length - 1);

                // 遞迴建立左右子樹
                root.left = buildTree(inLeft, postLeft);
                root.right = buildTree(inRight, postRight);

                break;
            }
        }

        return root;
    }
}
```

## 八、為什麼 postLeft 和 postRight 要這樣切？

這裡是很多人最容易卡住的地方。

假設：

```text
inorder   = [4, 2, 1, 6, 3, 7]
postorder = [4, 2, 6, 7, 3, 1]
```

根節點是 `1`，在中序遍歷中的位置是 `2`。

代表：

* 左子樹有 `2` 個節點
* 右子樹有 `3` 個節點

所以在後序遍歷中：

```text
[4, 2, 6, 7, 3, 1]
```

去掉最後的根節點 `1` 後：

```text
[4, 2, 6, 7, 3]
```

因為左子樹有 2 個節點，所以：

* 左子樹後序：`[4, 2]`
* 右子樹後序：`[6, 7, 3]`

也就是：

```java
int[] postLeft = Arrays.copyOfRange(postorder, 0, i);
int[] postRight = Arrays.copyOfRange(postorder, i, postorder.length - 1);
```

其中 `i` 就是根節點在中序遍歷中的位置，也剛好等於左子樹的節點數量。
