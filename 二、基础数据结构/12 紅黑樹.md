# 紅黑樹概述

## 一、什麼是紅黑樹？

紅黑樹（Red-Black Tree）是一種**自平衡二元搜尋樹**（Self-Balancing Binary Search Tree）。

它和 AVL 樹一樣，都能在查詢、插入、刪除時維持不錯的效率，但兩者「判斷平衡」的方式不同：

### AVL 樹

AVL 樹要求：

* 每個節點的左右子樹高度差不能超過 1

只要超過 1，就要旋轉調整。

### 紅黑樹

紅黑樹不是直接看高度差，而是依靠一套**顏色規則**來維持近似平衡。

## 二、紅黑樹相比 AVL 樹的優勢

紅黑樹最大的優勢在於：

* **插入、刪除時調整次數通常比 AVL 少**
* 因此在實務上，**修改操作的效率常常比 AVL 更好**

也就是說：

* AVL：查詢更嚴格平衡
* 紅黑樹：插入刪除更靈活、調整成本較低

## 三、紅黑樹的五大性質

只要一棵樹符合下面五條規則，它就是一棵合法的紅黑樹。

### 性質 1：每個節點不是紅色就是黑色

紅黑樹的節點只有兩種顏色：

* 紅色（RED）
* 黑色（BLACK）

所以在設計節點類別時，通常會多一個 `color` 屬性。

### 性質 2：null 節點視為黑色

如果某個子節點不存在，也就是 `null`，在紅黑樹中要把它當成**黑色節點**。

這點很重要，因為後面在判斷「黑色節點數量是否一致」時，`null` 也要算進去。

### 性質 3：紅色節點不能相鄰

也就是說：

* 紅色節點的父節點不能是紅色
* 紅色節點的子節點也不能是紅色

換句話說，**不可以出現連續兩個紅色節點**

這是判斷紅黑樹是否失衡的重要依據之一。

### 性質 4：根節點必須是黑色

整棵樹的最上方根節點一定要是黑色。

如果在插入或調整過程中，根節點暫時變成紅色，最後也要改回黑色。

### 性質 5：從根到任意葉子的每條路徑，黑色節點數量都相同

這條規則是紅黑樹平衡的核心之一。

注意這裡說的是：

* 從根節點出發
* 到每個葉子（實際上含 `null` 葉子）
* 每條路徑經過的**黑色節點數必須一樣多**

這個數量通常稱為 **黑高（black height）**。

## 四、如何快速判斷一棵樹是不是平衡的紅黑樹？

實務上判斷時，最常優先看這三條：

* 性質 3：有沒有紅紅相鄰
* 性質 4：根是不是黑色
* 性質 5：每條路徑黑色節點數是否一致

如果這三條都滿足，通常就已經很接近正確答案了。

## 五、幾個常見判斷例子

### 例 1：紅色節點相鄰，不平衡

```text
      6
     / \
    2   8
   /
  1

紅色：1、2、8
黑色：6
```

這棵樹不是合法紅黑樹，因為：

* `2` 是紅色
* `1` 也是紅色
* 出現紅色相鄰

違反性質 3。

### 例 2：黑色節點數不同，不平衡

```text
          6
         /  \
        2    8
       / \  / \ 
      1  3  7  9

紅色：1、3
黑色：2、6、7、8、9
```

* 路徑 1：6 → 2 → 1
  * 經過的黑色節點：`6、2`
  * 黑色節點數：**2**

* 路徑 2：6 → 2 → 3
  * 經過的黑色節點：`6、2`
  * 黑色節點數：**2**

* 路徑 3：6 → 8 → 7
  * 經過的黑色節點：`6、8、7`
  * 黑色節點數：**3**

* 路徑 4：6 → 8 → 9
  * 經過的黑色節點：`6、8、9`
  * 黑色節點數：**3**


> 左側路徑與右側路徑經過的黑色節點數不同，因此違反性質 5，不平衡。

### 例 3：這是一棵合法紅黑樹

```text
         6
        / \
       2   8
      /
     1

紅色：1
黑色：2、6、8
```

* 路徑 1：6 → 2 → 1 → null
  * 經過的黑色節點：`6、2`
  * 黑色節點數：**2**

* 路徑 2：6 → 2 → null
  * 經過的黑色節點：`6、2`
  * 黑色節點數：**2**

* 路徑 3：6 → 8 → null
  * 經過的黑色節點：`6、8`
  * 黑色節點數：**2**

* 路徑 4：6 → 8 → null
  * 經過的黑色節點：`6、8`
  * 黑色節點數：**2**

這棵樹是平衡的，因為：

* 沒有紅色相鄰
* 根節點 `6` 是黑色
* 每條路徑黑色節點數一致

## 六、為什麼判斷時一定要把 null 算進去？

這是初學紅黑樹最容易忽略的地方。

例如下面這棵樹：

```text
         6
        / \
       2   8
      / 
     1  

紅色：2
黑色：1、6、8
```

很多人第一眼會覺得平衡，因為：

* 根是黑色
* 沒有紅色相鄰
* 看起來左右黑節點數差不多

但其實**不平衡**。

因為紅黑樹在計算路徑時，必須把 `null` 也視為黑色節點。

補上 `null` 後，其實會變成：

```text
            6
        /      \
       2        8
      / \      / \
     1  null null null
    / \
 null null

紅色：2
黑色：1、6、8
```

這時候你會發現：

* 路徑 1：6 → 8 → null
  * 經過的黑色節點：`6、8、null`
  * 黑色節點數：**3**

* 路徑 2：6 → 8 → null
  * 經過的黑色節點：`6、8、null`
  * 黑色節點數：**3**

* 路徑 3：6 → 2 → 1 → null
  * 經過的黑色節點：`6、1、null`
  * 黑色節點數：**3**

* 路徑 4：6 → 2 → 1 → null
  * 經過的黑色節點：`6、1、null`
  * 黑色節點數：**3**

* 路徑 5：6 → 2 → null
  * 經過的黑色節點：`6、null`
  * 黑色節點數：**2**

所以不平衡。

### 一個很好記的經驗

#### 紅色葉子可以單獨出現

例如只有左紅孩子、沒有右兄弟，可能仍然合法。

#### 黑色葉子若單獨出現，通常有問題

因為會破壞不同路徑上的黑高一致性。

所以可以記：

> **黑色葉子通常要成對看，紅色葉子可以單邊存在。**

## 七、紅黑樹節點設計

紅黑樹節點和一般二元搜尋樹節點很像，但多了兩個重要屬性：

1. `parent`：父節點
2. `color`：顏色

而且新插入的節點，通常預設是**紅色**。

```java
public class RedBlackTree {
    enum Color {
        RED, BLACK
    }

    private Node root;

    private class Node {
        int key;
        Object value;
        Node left;
        Node right;
        Node parent;   // 父節點
        Color color = Color.RED; // 新節點預設紅色
    }
}
```

## 八、常用工具方法

### 1. 判斷自己是不是左孩子

```java
boolean isLeftChild() {
    return parent != null && parent.left == this;
}
```

意思是：

* 先確認自己有父節點
* 再看父節點的左孩子是不是自己

如果是，就代表目前節點是左孩子。

### 2. 找叔叔節點 uncle()

叔叔節點的意思是：

* 父節點的兄弟節點

判斷方式：

* 沒有父節點 → 沒有叔叔
* 沒有祖父節點 → 沒有叔叔
* 如果父節點是祖父的左孩子，叔叔就是祖父的右孩子
* 反之亦然

```java
Node uncle() {
    if (parent == null || parent.parent == null) {
        return null;
    }
    if (parent.isLeftChild()) {
        return parent.parent.right;
    } else {
        return parent.parent.left;
    }
}
```

### 3. 找兄弟節點 sibling()

兄弟節點的意思是：

* 和自己同一個父節點的另一個孩子

```java
Node sibling() {
    if (parent == null) {
        return null;
    }
    if (this.isLeftChild()) {
        return parent.right;
    } else {
        return parent.left;
    }
}
```

### 4. 判斷節點是不是紅色

因為有可能傳進來的是 `null`，所以不能直接 `node.color`。

```java
boolean isRed(Node node) {
    return node != null && node.color == Color.RED;
}
```

### 5. 判斷節點是不是黑色

因為 `null` 在紅黑樹中視為黑色，所以：

```java
boolean isBlack(Node node) {
    return node == null || node.color == Color.BLACK;
}
```

也可以寫成：

```java
boolean isBlack(Node node) {
    return !isRed(node);
}
```

## 九、紅黑樹旋轉的核心觀念

紅黑樹和 AVL 樹一樣，也會用：

* 左旋（left rotate）
* 右旋（right rotate）

來調整樹的結構。

### 和 AVL 旋轉相比，多了兩個麻煩點

#### 1. 要維護 parent

因為節點多了 `parent` 屬性，所以旋轉後不只左右孩子要調整，父節點指向也要一起更新。

#### 2. 旋轉完後，要直接把新根和更上層接起來

以前可能旋轉函式只回傳新的子樹根節點，外部再接回去。

但這裡通常會直接在旋轉方法內把整個父子關係接好。

## 十、右旋的理解

假設對節點 `x` 做右旋：

* `y = x.left`
* `b = y.right`

旋轉前：

```text
        x
       / \
      y   ...
     / \
   ...  b
```

旋轉後：

```text
        y
       / \
     ...  x
         / \
        b  ...
```

### 變化重點

* `y` 上升成為新的子樹根
* `x` 下沉成為 `y` 的右孩子
* `b` 改掛到 `x` 的左邊


### 右旋時要維護哪些關係？

假設：

* `x`：原本要旋轉的節點
* `y = x.left`
* `b = y.right`

那麼右旋時需要處理：

#### 1. 左右子樹連接

* `y.right = x`
* `x.left = b`

#### 2. 父節點指向

* 如果 `b != null`，則 `b.parent = x`
* `y.parent = x.parent`
* `x.parent = y`

#### 3. 更上層節點要接回 y

如果 `x` 原本不是整棵樹根節點，還要讓 `x` 的父節點改指向 `y`。

如果 `x` 原本就是根節點，則旋轉後：

* `root = y`

### 右旋的直觀例子

例如下面這棵樹：

```text
             8
           /   \
          5    10
         / \   / \
        3   6 9   11
       / \   \
      2   4   7
     /
    1

紅色：1、3、7、8、9、11
黑色：2、4、5、6、10

8 是要旋轉的節點
5 是要上升的新根
6 是中間子樹，要重新換父節點
```

如果我們對節點 `8` 做右旋：

* `x = 8`
* `y = 5`
* `b = 6`

因為：

* `8.left = 5`
* `5.right = 6`

所以右旋後會變成：

```text
             5
           /   \
          3     8
         / \   /  \
        2   4 6    10
       /       \   / \
      1         7  9  11

紅色：1、3、7、8、9、11
黑色：2、4、5、6、10
```

#### 這個過程發生了什麼？

* `5` 上升，成為新的子樹根
* `8` 下沉，成為 `5` 的右孩子
* 原本 `5` 的右子樹 `6`，改掛到 `8` 的左邊
* `3`、`2`、`4` 原本就在 `5` 的左側，所以不需要搬動
* `10`、`9`、`11` 原本就在 `8` 的右側，所以也不需要搬動

### 右旋程式碼

```java
private void rightRotate(Node x) {
    Node y = x.left;
    Node b = y.right;
    Node p = x.parent;

    // 1. b 掛到 x 左邊
    x.left = b;
    if (b != null) {
        b.parent = x;
    }

    // 2. x 掛到 y 右邊
    y.right = x;
    x.parent = y;

    // 3. y 接到原本 x 的父節點
    y.parent = p;

    // 4. 原本 x 的父節點改連到 y
    if (p == null) {
        root = y;
    } else if (p.left == x) {
        p.left = y;
    } else {
        p.right = y;
    }
}
```

你可以記成四步：

#### 第一步：找三個角色

* `x`：原根
* `y`：左孩子（要上升）
* `b`：`y` 的右子樹（中間要搬家）

#### 第二步：把 b 移給 x.left

因為旋轉後 `x` 要下沉，所以 `b` 會變成 `x` 的左子樹。

#### 第三步：讓 y 成為新根

* `y.right = x`

#### 第四步：接回上層

* 如果 `x` 原本是根，更新 `root`
* 否則讓 `x` 的父節點改連到 `y`

## 十一、左旋的理解

假設對節點 `x` 做左旋：

* `y = x.right`
* `b = y.left`

旋轉前：

```text
        x
       / \
     ...  y
         / \
        b  ...
```

旋轉後：

```text
        y
       / \
      x  ...
     / \
   ...  b
```

### 變化重點

* `y` 上升成為新的子樹根
* `x` 下沉成為 `y` 的左孩子
* `b` 改掛到 `x` 的右邊

左旋和右旋是完全對稱的：

* 右旋：左孩子上升、原節點下沉到右邊
* 左旋：右孩子上升、原節點下沉到左邊

### 左旋時要維護哪些關係？

假設：

* `x`：原本要旋轉的節點
* `y = x.right`
* `b = y.left`

那麼左旋時需要處理：

#### 1. 左右子樹連接

* `x.right = b`
* `y.left = x`

#### 2. 父節點指向

* 如果 `b != null`，則 `b.parent = x`
* `y.parent = x.parent`
* `x.parent = y`

#### 3. 更上層節點要接回 y

如果 `x` 原本不是整棵樹的根節點，還要讓 `x` 的父節點改指向 `y`。

如果 `x` 原本就是根節點，則旋轉後：

* `root = y`

### 左旋的直觀例子

例如下面這棵樹：

```text
             8
           /   \
          5    10
              /  \
             9   11
                  \
                  12

紅色：5、8、12
黑色：9、10、11

8 是要旋轉的節點
10 是要上升的新根
9 是中間子樹，要重新換父節點
```

如果我們對節點 `8` 做左旋：

* `x = 8`
* `y = 10`
* `b = 9`

因為：

* `8.right = 10`
* `10.left = 9`

所以左旋後會變成：

```text
             10
            /  \
           8    11
          / \     \
         5   9     12

紅色：5、8、12
黑色：9、10、11
```

#### 這個過程發生了什麼？

* `10` 上升，成為新的子樹根
* `8` 下沉，成為 `10` 的左孩子
* 原本 `10` 的左子樹 `9`，改掛到 `8` 的右邊
* `11` 和 `12` 原本就在 `10` 的右側，所以不需要搬動

### 左旋程式碼

```java
private void leftRotate(Node x) {
    Node y = x.right;
    Node b = y.left;
    Node p = x.parent;

    // 1. b 掛到 x 右邊
    x.right = b;
    if (b != null) {
        b.parent = x;
    }

    // 2. x 掛到 y 左邊
    y.left = x;
    x.parent = y;

    // 3. y 接到原本 x 的父節點
    y.parent = p;

    // 4. 原本 x 的父節點改連到 y
    if (p == null) {
        root = y;
    } else if (p.left == x) {
        p.left = y;
    } else {
        p.right = y;
    }
}
```

你可以把它拆成四步來看。

#### 第一步：找三個角色

* `x`：原根
* `y`：右孩子（要上升）
* `b`：`y` 的左子樹（中間要搬家）

```java
Node y = x.right;
Node b = y.left;
Node p = x.parent;
```

#### 第二步：把 b 移給 x.right

因為旋轉後 `x` 要下沉，所以 `b` 會變成 `x` 的右子樹。

```java
x.right = b;
if (b != null) {
    b.parent = x;
}
```

這一步要注意：

* `b` 有可能是 `null`
* 所以更新 `b.parent` 前一定要先判空

#### 第三步：讓 y 成為新根

把 `x` 掛到 `y` 的左邊：

```java
y.left = x;
x.parent = y;
```

這樣就完成了「右孩子上升、原節點下沉」的核心動作。

#### 第四步：接回上層

旋轉後，新的子樹根變成 `y`，所以要把 `y` 接回原本 `x` 的父節點 `p`。

```java
y.parent = p;

if (p == null) {
    root = y;
} else if (p.left == x) {
    p.left = y;
} else {
    p.right = y;
}
```

意思是：

* 如果 `x` 原本就是根節點，更新整棵樹的 `root`
* 如果 `x` 原本是父節點的左孩子，就讓 `p.left = y`
* 否則讓 `p.right = y`
