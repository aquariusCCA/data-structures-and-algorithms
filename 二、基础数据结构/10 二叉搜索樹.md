# 二叉搜索樹（Binary Search Tree, BST）概述

在前面，我們已經學過一些基本的資料結構。
不過，如果我們想要在這些資料結構中**快速查找某個元素**，效率通常不算高。

這時候，很多人會想到我們之前學過的 **二分查找**。

## 為什麼不能只用二分查找？

二分查找的確可以大幅提升查詢效率，但它有一個重要前提：

> **必須建立在「已排序的陣列」上。**

問題是，若原本的資料沒有排序，我們就得先花時間排序。
而排序本身是有成本的，有時候會變成：

* 還沒開始查找
* 就先花了不少時間在排序上

這樣一來，整體效率未必划算。

## 有沒有更好的折衷方法？

有。
我們可以改用另一種資料結構：**二叉搜索樹**。

二叉搜索樹的好處是：

* 可以提升查找效率
* 在插入元素的過程中，就自然維持一定的大小順序

也就是說，它不像陣列那樣需要先整體排序，而是在加入資料時，就把元素放到合適的位置。

## 什麼是二叉搜索樹？

下面這棵樹就是一棵二叉搜索樹，也叫做 **二叉排序樹**。

```text
              4
           /     \
          2       7
         / \      / \
        1   3    6   8
                /
               5 
```

## 二叉搜索樹的兩個核心特徵

要成為二叉搜索樹，必須滿足以下條件：

### 1. 每個節點都有一個 key，用來比較大小

* `key` 可以理解成節點的值
* `key` **不能重複**

上圖中的 `1、2、3、4、5、6、7、8`，就是各節點的 key。

### 2. 任意節點都要符合大小規則

對於每一個節點來說：

* **左子樹所有節點的 key，都比它小**
* **右子樹所有節點的 key，都比它大**

這是二叉搜索樹最重要的性質。

### 例子：為什麼 6 的右孩子不能亂放？

假設我們想幫節點 `6` 加上一個右孩子，那麼這個值必須同時滿足：

* 比 `6` 大
* 又要比 `7` 小

所以像 `6.5` 就是合理的。

```text
              4
           /     \
          2       7
         / \      / \
        1   3    6   8
                / \
               5   6.5
```

這樣整棵樹依然符合二叉搜索樹的規則。

## 二叉搜索樹為什麼查找比較快？

假設我們要查找 `8`。

### 查找過程

1. 從根節點 `4` 開始
   因為 `8 > 4`，所以往右子樹走

2. 到節點 `7`
   因為 `8 > 7`，所以繼續往右子樹走

3. 到節點 `8`
   找到了目標

總共只比較了 **3 次**。

## 查找效率和什麼有關？

二叉搜索樹的查找效率，通常和 **樹的高度** 有關。

樹越矮、越平衡，查找通常越快。
因為每次比較後，都可以排除一大半不可能的範圍。

所以在理想情況下，二叉搜索樹的查找效率會很好。

## 但它也有缺點：可能會退化

如果樹長得很不平均，例如下面這樣：

```text
                5
               /
              4
             /
            3
           /
          2
         /
        1 
```

這棵樹幾乎已經不像樹了，而是很像一條鏈。

這種情況代表：

* 左右子樹高度差很多
* 樹**不平衡**

### 不平衡時，查找效率會變差

例如要查找 `1`：

1. 先比 `5`
2. 再比 `4`
3. 再比 `3`
4. 再比 `2`
5. 最後比 `1`

總共要比較 **5 次**。

這時候查找效率就退化成和線性查找差不多，時間複雜度會回到：

$$
O(n)
$$

# 二叉搜索樹 get 方法實現

前面我們已經了解了二叉搜索樹（BST）的基本概念，接下來來看它最常見的一個操作：**查找節點**，也就是 `get` 方法。

`get` 方法的作用很簡單：

> 根據給定的 `key`，找到對應節點的 `value`。

如果找不到，就回傳 `null`。

## 一、先定義節點類別

在二叉搜索樹中，每一個節點都需要保存：

* `key`：用來比較大小
* `value`：實際儲存的資料
* `left`：左子節點
* `right`：右子節點

這裡把節點設計成 `BSTTree1` 的靜態內部類：

```java
public class BSTTree1 {
    static class BSTNode {
        int key;
        Object value;
        BSTNode left;
        BSTNode right;

        public BSTNode(int key) {
            this.key = key;
        }

        public BSTNode(int key, Object value) {
            this.key = key;
            this.value = value;
        }

        public BSTNode(int key, Object value, BSTNode left, BSTNode right) {
            this.key = key;
            this.value = value;
            this.left = left;
            this.right = right;
        }
    }
}
```

## 二、二叉搜索樹至少要有根節點

在 BST 類別中，最基本的屬性就是：

```java
BSTNode root;  // 根節點
```

整棵樹的查找、插入、刪除，通常都是從根節點開始。

## 三、get 方法的查找思路

查找時，從根節點開始，把目前節點的 `key` 和目標 `key` 做比較，分成 3 種情況：

### 1. 目標 key 比目前節點小

代表要找的節點應該在**左子樹**，所以往左走。

### 2. 目標 key 比目前節點大

代表要找的節點應該在**右子樹**，所以往右走。

### 3. 兩者相等

表示已經找到目標節點，直接回傳它的 `value`。

這個判斷規則不只適用於根節點，也適用於後續每一個子節點。
因此，這是一個很適合用**遞迴**實作的問題。

## 四、遞迴版本的 get 方法

### 1. 對外公開的方法

`get` 是給外部呼叫的，所以通常會從 `root` 開始查找：

```java
public Object get(int key) {
    return doGet(root, key);
}
```

### 2. 真正執行查找的方法

```java
private Object doGet(BSTNode node, int key) {
    if (node == null) {
        return null; // 沒有找到
    }
    if (key < node.key) {
        return doGet(node.left, key); // 從左子樹繼續查找
    } else if (key > node.key) {
        return doGet(node.right, key); // 從右子樹繼續查找
    } else {
        return node.value; // 找到了，回傳值
    }
}
```

### 如何理解這段遞迴程式？

這段程式其實就是在重複做同一件事：

1. 先看目前節點是不是 `null`

    * 如果是，表示這條路已經走到底了，還是沒找到，所以回傳 `null`

2. 如果目前節點不為 `null`

    * `key < node.key`：往左找
    * `key > node.key`：往右找
    * `key == node.key`：找到答案

也就是說，每次遞迴都在縮小查找範圍。

### 測試資料範例

下面建立了一棵二叉搜索樹：

```java
class TestBSTTree1 {
    public BSTTree1 createTree() {
        BSTTree1.BSTNode n1 = new BSTTree1.BSTNode(1, "张无忌");
        BSTTree1.BSTNode n3 = new BSTTree1.BSTNode(3, "宋青书");
        BSTTree1.BSTNode n2 = new BSTTree1.BSTNode(2, "周芷若", n1, n3);

        BSTTree1.BSTNode n5 = new BSTTree1.BSTNode(5, "说不得");
        BSTTree1.BSTNode n7 = new BSTTree1.BSTNode(7, "殷离");
        BSTTree1.BSTNode n6 = new BSTTree1.BSTNode(6, "赵敏", n5, n7);
        BSTTree1.BSTNode root = new BSTTree1.BSTNode(4, "小昭", n2, n6);

        BSTTree1 tree = new BSTTree1();
        tree.root = root;
        return tree;
    }

    @Test
    void get() {
        BSTTree1 tree = createTree();
        assertEquals("张无忌", tree.get(1));
        assertEquals("周芷若", tree.get(2));
        assertEquals("宋青书", tree.get(3));
        assertEquals("小昭", tree.get(4));
        assertEquals("说不得", tree.get(5));
        assertEquals("赵敏", tree.get(6));
        assertEquals("殷离", tree.get(7));
        assertNull(tree.get(8));
    }
}
```

這棵樹的結構可以理解成：

```text
              4(小昭)
           /           \
     2(周芷若)         6(赵敏)
      /     \          /     \
1(张无忌) 3(宋青书) 5(说不得) 7(殷离)
```

### 查找過程示例

例如執行：

```java
tree.get(5)
```

查找過程如下：

1. 從根節點 `4` 開始

    * `5 > 4`，往右子樹走

2. 到節點 `6`

    * `5 < 6`，往左子樹走

3. 到節點 `5`

    * `5 == 5`，找到，回傳 `"说不得"`

所以 BST 查找並不是一個一個慢慢比，而是每次比較後，直接排除一半方向。

## 五、非遞迴版本的 get 方法

剛剛是用遞迴實現，現在來看**非遞迴**寫法。

思路其實完全一樣，只是把「函式自己呼叫自己」改成「用 while 迴圈不斷往下走」。

```java
public Object get(int key) {
    BSTNode current = root; // 從根節點開始
    while (current != null) {
        if (key < current.key) {
            current = current.left; // 從左子樹繼續查找
        } else if (key > current.key) {
            current = current.right; // 從右子樹繼續查找
        } else {
            return current.value; // 找到了，回傳值
        }
    }
    return null; // 沒有找到
}
```

### 如何理解非遞迴版本？

可以把 `current` 想成「目前正在比較的節點」。

每次迴圈做的事就是：

* 比大小
* 決定往左還是往右
* 更新 `current`
* 持續往下找

如果最後 `current` 變成 `null`，就表示走到底了還是沒找到。

## 六、什麼是尾遞迴？

原文提到一個概念：**尾遞迴**。

尾遞迴的意思是：

> 一個函式在最後一步呼叫自己，而且呼叫完之後不再做其他事情。

例如：

```java
return doGet(node.left, key);
```

或：

```java
return doGet(node.right, key);
```

這就是尾遞迴，因為遞迴呼叫後，沒有額外運算，直接把結果回傳。

也因為這樣，這類程式通常很容易改寫成非遞迴版本。

## 七、完整理解：get 方法的核心本質

BST 的 `get` 方法之所以高效，是因為它利用了二叉搜索樹的性質：

* 左邊都比較小
* 右邊都比較大

所以每次比較之後，都能立刻決定下一步方向，不需要像普通查找那樣把所有節點都掃過一次。

這就是 BST 查找快的原因。

# 二叉搜索樹的泛型版本實現

前面在實作二叉搜索樹時，我們把節點的 `key` 設計成 `int` 類型。

這樣做的原因很簡單：

> 二叉搜索樹中的節點必須能夠比較大小，才能決定往左走還是往右走。

`int` 本身就能直接比較大小，所以一開始用它來當 `key` 很合理。

不過，問題來了：

* 如果我想用 `String` 當 key 呢？
* 如果我想用 `Integer`、`Long`、`LocalDate` 當 key 呢？

也就是說，我們希望二叉搜索樹的 `key` 不只支援 `int`，而是能支援更多可比較的型別。

這時候，就會想到使用 **泛型**。

## 一、先把 key 改成泛型

最直覺的想法，就是先把原本的 `int key` 改成 `T key`：

```java
public class BSTTree1<T> {
    static class BSTNode<T> {
        T key;
        Object value;
        BSTNode<T> left;
        BSTNode<T> right;

        public BSTNode(T key) {
            this.key = key;
        }

        public BSTNode(T key, Object value) {
            this.key = key;
            this.value = value;
        }

        public BSTNode(T key, Object value, BSTNode<T> left, BSTNode<T> right) {
            this.key = key;
            this.value = value;
            this.left = left;
            this.right = right;
        }
    }
}
```

表面上看起來，好像已經完成泛型化了。
但其實這樣還不夠。

## 二、為什麼直接寫 T 還不行？

因為二叉搜索樹的核心前提是：

> `key` 必須能比較大小。

但如果你只是隨便寫一個泛型 `T`，Java 並不能保證這個 `T` 一定可以比較大小。

例如：

* `Integer` 可以比較
* `String` 可以比較
* 但某些自訂類別如果沒有提供比較規則，就不能直接比較

所以，單純寫成 `T` 並不能滿足 BST 的需求。

## 三、限制泛型必須可以比較大小

在 Java 中，如果一個類型想要具備「可比較大小」的能力，通常要實作 `Comparable` 介面。

因此，我們要對泛型加上限制：

```java
T extends Comparable<T>
```

這表示：

> `T` 必須是一個可以和同型別物件比較大小的類型。

改完之後，程式就會變成這樣：

```java
public class BSTTree2<T extends Comparable<T>> {
    static class BSTNode<T extends Comparable<T>> {
        T key;
        Object value;
        BSTNode<T> left;
        BSTNode<T> right;

        public BSTNode(T key) {
            this.key = key;
        }

        public BSTNode(T key, Object value) {
            this.key = key;
            this.value = value;
        }

        public BSTNode(T key, Object value, BSTNode<T> left, BSTNode<T> right) {
            this.key = key;
            this.value = value;
            this.left = left;
            this.right = right;
        }
    }
}
```


## 四、加入根節點

一棵二叉搜索樹，最基本一定要有根節點：

```java
BSTNode<T> root;
```

完整來說：

```java
public class BSTTree2<T extends Comparable<T>> {
    BSTNode<T> root;

    static class BSTNode<T extends Comparable<T>> {
        T key;
        Object value;
        BSTNode<T> left;
        BSTNode<T> right;

        public BSTNode(T key) {
            this.key = key;
        }

        public BSTNode(T key, Object value) {
            this.key = key;
            this.value = value;
        }

        public BSTNode(T key, Object value, BSTNode<T> left, BSTNode<T> right) {
            this.key = key;
            this.value = value;
            this.left = left;
            this.right = right;
        }
    }
}
```

## 五、泛型版本的 get 方法

接著來看查找方法 `get`。

思路和前面 `int` 版本完全一樣，只是原本用 `<`、`>` 比較整數，現在要改用：

```java
key.compareTo(cur.key)
```

來比較大小。

### 非遞迴寫法

```java
public Object get(T key) {
    BSTNode<T> cur = root;
    while (cur != null) {
        int result = key.compareTo(cur.key);
        if (result == 0) {
            return cur.value;
        } else if (result < 0) {
            cur = cur.left;
        } else {
            cur = cur.right;
        }
    }
    return null;
}
```

## 六、如何理解 compareTo 的結果？

`compareTo` 會回傳一個整數，代表比較結果：

* `result == 0`：兩個 key 相等
* `result < 0`：目前查找的 key 比節點 key 小
* `result > 0`：目前查找的 key 比節點 key 大

因此在 BST 中就可以這樣判斷：

* 小於 0，往左子樹走
* 大於 0，往右子樹走
* 等於 0，表示找到節點

## 七、範例：使用 String 作為 key

這裡我們不再用數字當 key，而是改用字串 `"a"`、`"b"`、`"c"` ... 來建立一棵樹。

概念上仍然是同一棵 BST，只是比較規則改成字串的字典順序。

```java
class TestBSTTree2 {
    public BSTTree2<String> createTree() {
        /*
                     d
                   /   \
                  b     f
                 / \   / \
                a   c e   g
         */
        BSTTree2.BSTNode<String> n1 = new BSTTree2.BSTNode<>("a", "张无忌");
        BSTTree2.BSTNode<String> n3 = new BSTTree2.BSTNode<>("c", "宋青书");
        BSTTree2.BSTNode<String> n2 = new BSTTree2.BSTNode<>("b", "周芷若", n1, n3);

        BSTTree2.BSTNode<String> n5 = new BSTTree2.BSTNode<>("e", "说不得");
        BSTTree2.BSTNode<String> n7 = new BSTTree2.BSTNode<>("g", "殷离");
        BSTTree2.BSTNode<String> n6 = new BSTTree2.BSTNode<>("f", "赵敏", n5, n7);
        BSTTree2.BSTNode<String> root = new BSTTree2.BSTNode<>("d", "小昭", n2, n6);

        BSTTree2<String> tree = new BSTTree2<>();
        tree.root = root;
        return tree;
    }

    @Test
    void get() {
        BSTTree2<String> tree = createTree();
        assertEquals("张无忌", tree.get("a"));
        assertEquals("周芷若", tree.get("b"));
        assertEquals("宋青书", tree.get("c"));
        assertEquals("小昭", tree.get("d"));
        assertEquals("说不得", tree.get("e"));
        assertEquals("赵敏", tree.get("f"));
        assertEquals("殷离", tree.get("g"));
        assertNull(tree.get("h"));
    }
}
```

### 這個例子說明了什麼？

它說明了一件事：

> 只要 `key` 所屬的型別實作了 `Comparable`，BST 就能正常工作。

也就是說，BST 並不在乎 `key` 是不是數字，
它只在乎這個 `key` 能不能比較大小。

所以：

* `Integer` 可以
* `String` 可以
* 自訂類別也可以，只要你實作 `Comparable`

## 八、目前還有一個可以再改進的地方

上面的版本雖然把 `key` 改成泛型了，但 `value` 仍然是：

```java
Object value;
```

這代表樹中存放的值雖然可以是任何型別，但每次取出時都只能當成 `Object` 看待，不夠精確，也不夠安全。

所以我們可以進一步把 `value` 也改成泛型。

### 同時對 key 和 value 使用泛型

改進後的版本如下：

```java
public class BSTTree3<K extends Comparable<K>, V> {
    static class BSTNode<K extends Comparable<K>, V> {
        K key;
        V value;
        BSTNode<K, V> left;
        BSTNode<K, V> right;

        public BSTNode(K key) {
            this.key = key;
        }

        public BSTNode(K key, V value) {
            this.key = key;
            this.value = value;
        }

        public BSTNode(K key, V value, BSTNode<K, V> left, BSTNode<K, V> right) {
            this.key = key;
            this.value = value;
            this.left = left;
            this.right = right;
        }
    }
}
```

這樣的設計更完整：

* `K`：表示 key 的型別，必須可比較
* `V`：表示 value 的型別，可以自由指定

例如：

* `BSTTree3<Integer, String>`
* `BSTTree3<String, User>`
* `BSTTree3<LocalDate, Order>`

都可以成立。

## 九、為什麼 `key` 要限制，`value` 不需要？

這是因為在二叉搜索樹中：

* `key` 要負責決定節點放哪裡，所以必須能比較大小
* `value` 只是附加資料，不參與大小比較

因此：

* `key` 必須 `extends Comparable<K>`
* `value` 不需要這個限制

# 二叉搜索樹 min 方法實現

接下來我們來看二叉搜索樹中的 `min` 方法，也就是：

> **找出樹中最小的 key。**

## 一、為什麼最小值一定在最左邊？

先看下面這棵二叉搜索樹：

```text
              4
           /     \
          2       7
         / \      / \
        1   3    6   8
                /
               5
```

根據二叉搜索樹的規則：

* 左子樹的 key 都比目前節點小
* 右子樹的 key 都比目前節點大

因此，**越往左走，key 就越小**。
所以要找最小 key，只要從根節點開始，一直往左走，直到不能再走為止。

例如這棵樹的查找過程：

1. 從 `4` 開始，往左走到 `2`
2. 從 `2` 再往左走到 `1`
3. `1` 已經沒有左子節點了

所以最小 key 就是 `1`。

## 二、核心思路

`min` 方法的邏輯其實很簡單：

* 從根節點開始
* 不斷往左子節點走
* 當某個節點已經沒有左子節點時
* 這個節點就是最小節點

換句話說，判斷「有沒有走到頭」的方法就是：

> 看目前節點的 `left` 是否為 `null`

## 三、遞迴版本實現

### 對外方法

```java
public Object min() {
    return doMin(root);
}
```

### 真正執行查找的方法

```java
public Object doMin(BSTNode node) {
    // 特殊情況：樹是空的
    if (node == null) {
        return null;
    }
    if (node.left == null) { // 已經走到最左邊
        return node.key;
    }
    return doMin(node.left); // 繼續向左找
}
```

### 如何理解這段遞迴？

這段程式每次只做一件事：

1. 先判斷目前節點是不是 `null`

    * 如果是，代表整棵樹是空的，回傳 `null`

2. 如果目前節點不為 `null`

    * 檢查它有沒有左子節點
    * 如果沒有，代表已經到最左邊，直接回傳目前節點的 key
    * 如果有，就繼續往左遞迴查找

你可以把它理解成一句話：

> **一直往左找，直到左邊沒有節點為止。**

## 四、非遞迴版本實現

除了遞迴，也可以用迴圈來完成。

```java
public Object min() {
    if (root == null) {
        return null;
    }
    BSTNode current = root;
    while (current.left != null) { // 還沒走到最左邊
        current = current.left;

```

### 如何理解非遞迴版本？

可以把 `current` 想成「目前所在的節點」。

整個流程就是：

1. 從 `root` 開始
2. 只要左邊還有節點，就繼續往左走
3. 當左邊沒有節點時，停下來
4. 此時 `current` 就是最小節點

這和遞迴版本的邏輯完全相同，只是寫法不同。

# 二叉搜索樹 put 方法實現

接下來我們來看二叉搜索樹中的 `put` 方法，也就是：

> **向二叉搜索樹中新增節點；如果 key 已存在，則更新該節點的 value。**

## 一、put 方法要處理哪兩種情況？

`put` 方法的核心只有兩種情況：

### 1. key 已存在

如果樹中已經有這個 key，就不需要新增節點，只要把原本節點的 `value` 更新掉即可。

### 2. key 不存在

如果整棵樹都找不到這個 key，就要建立一個新節點，並把它插入到正確的位置。

## 二、先看一棵範例樹

```text
              4
           /     \
          2       7
         / \      / \
        1   3    6   8
                /
               5
```

### 情況一：key 已存在，更新 value

假設現在要新增：

```java
key = 1
```

查找過程如下：

1. 先和根節點 `4` 比較
   因為 `1 < 4`，所以往左子樹走

2. 再和節點 `2` 比較
   因為 `1 < 2`，所以繼續往左子樹走

3. 最後找到節點 `1`

這表示：

* `1` 這個 key 已經存在
* 不需要建立新節點
* 只要更新它的 `value` 即可

### 情況二：key 不存在，新增節點

假設現在要新增：

```java
key = 9
```

查找過程如下：

1. 和 `4` 比較
   因為 `9 > 4`，往右走

2. 和 `7` 比較
   因為 `9 > 7`，繼續往右走

3. 和 `8` 比較
   因為 `9 > 8`，再往右走

4. 發現 `8` 的右子節點是 `null`

這代表什麼？

這代表我們已經走到頭了，整棵樹都沒有找到 `9`，所以要建立一個新的節點，並把它放到 `8` 的右子樹位置。

插入後的結果如下：

```text
              4
           /     \
          2       7
         / \      / \
        1   3    6   8
                /     \
               5       9
```

## 三、put 的基本查找思路

其實 `put` 的前半段，和 `get` 很像：

* 從根節點開始
* 比較 `key` 和目前節點的大小
* 小於就往左走
* 大於就往右走
* 等於就更新 value

先看基本骨架：

```java
public void put(int key, Object value) {
    BSTNode current = root; // 從根節點開始
    while (current != null) {
        if (key < current.key) {
            current = current.left; // 繼續往左子樹走
        } else if (key > current.key) {
            current = current.right; // 繼續往右子樹走
        } else {
            current.value = value; // key 已存在，更新 value
            return;
        }
    }
    // key 不存在，建立新節點並插入
    BSTNode newNode = new BSTNode(key, value);
}
```

## 四、為什麼這樣還不夠？

上面的程式雖然能找到：

* key 是否存在
* 或者最後走到 `null`

但有一個問題：

> **如果要插入新節點，我們必須知道它要接在誰的下面。**

例如插入 `9` 時，最後雖然知道應該插入，但還必須知道：

* `9` 的父節點是 `8`

因為只有找到父節點，才能把新節點掛到父節點的左邊或右邊。

## 五、如何記住父節點？

觀察插入 `9` 時的移動路徑：

```text
4 -> 7 -> 8 -> null
```

當 `current` 最後變成 `null` 前，
它其實曾經停在 `8`。

所以我們只要在每次移動 `current` 前，先把目前節點記錄下來，就能保留父節點資訊。

這樣就可以多加一個變數：

```java
BSTNode parent = null;
```

### 加入 parent 後的寫法

```java
public void put(int key, Object value) {
    BSTNode current = root; // 從根節點開始
    BSTNode parent = null;  // 用來記錄父節點

    while (current != null) {
        parent = current; // 先記住目前節點

        if (key < current.key) {
            current = current.left; // 繼續往左走
        } else if (key > current.key) {
            current = current.right; // 繼續往右走
        } else {
            current.value = value; // key 已存在，直接更新
            return;
        }
    }

    // key 不存在，建立新節點
    BSTNode newNode = new BSTNode(key, value);
}
```

這時候：

* `current == null` 代表找不到 key
* `parent` 就是新節點要接上的父節點


## 六、找到父節點後，怎麼插入新節點？

接下來只要判斷：

* 新節點的 key 比父節點小，掛到左邊
* 新節點的 key 比父節點大，掛到右邊

另外還有一個特殊情況：

* 如果 `parent == null`
* 表示原本整棵樹是空的
* 這時新節點就是第一個節點，要直接設成 `root`

完整程式如下：

```java
public void put(int key, Object value) {
    BSTNode current = root; // 從根節點開始
    BSTNode parent = null;  // 記錄父節點

    while (current != null) {
        parent = current;

        if (key < current.key) {
            current = current.left;
        } else if (key > current.key) {
            current = current.right;
        } else {
            current.value = value; // key 已存在，更新 value
            return;
        }
    }

    // key 不存在，建立新節點
    BSTNode newNode = new BSTNode(key, value);

    if (parent == null) {
        root = newNode; // 原本是空樹
    } else if (key < parent.key) {
        parent.left = newNode; // 插入到父節點左邊
    } else {
        parent.right = newNode; // 插入到父節點右邊
    }
}
```

# 二叉搜索樹查找節點的前任與後任

接下來我們來看一個常見問題：

> **在二叉搜索樹中，如何找到某個節點的前任（predecessor）與後任（successor）？**

先看下面這棵二叉搜索樹：

```text
              4
           /     \
          2       7
         / \      / \
        1   3    6   8
                /
               5
```

## 一、什麼是前任與後任？

如果把二叉搜索樹中的所有 `key` 由小到大排好，會得到：

```text
1, 2, 3, 4, 5, 6, 7, 8
```

假設現在看的是 `4`：

* `4` 的前任，就是排序後排在它前面的那個值，也就是 `3`
* `4` 的後任，就是排序後排在它後面的那個值，也就是 `5`

所以你可以先把這兩個概念記成：

* **前任**：比當前節點小，而且最接近它的那個節點
* **後任**：比當前節點大，而且最接近它的那個節點

## 二、為什麼會先想到中序遍歷？

因為二叉搜索樹有一個很重要的特性：

> **對 BST 做中序遍歷，得到的結果就是升序序列。**

中序遍歷的順序是：

> **左子樹 → 根節點 → 右子樹**

所以對上面那棵樹做中序遍歷，結果就是：

```text
1, 2, 3, 4, 5, 6, 7, 8
```

既然中序遍歷後可以得到升序結果，那麼前任與後任看起來就很好找了。
例如找到 `4` 之後：

* 前一個是 `3`
* 後一個是 `5`

## 三、但實際上不會用中序遍歷來找

雖然中序遍歷可以解題，但它不是高效做法。

原因是：

* 中序遍歷通常要把整棵樹走一遍
* 時間複雜度是 `O(n)`

如果樹很大，效率就不好。
例如你只是想找 `8` 的前任，最壞情況下可能仍然要遍歷整棵樹。這也是原筆記後面開始整理規律、改用 BST 性質直接查找的原因。

## 四、前任（predecessor）的規則

找某個節點的前任時，有兩種情況。

### 情況 1：節點有左子樹

如果某個節點有左子樹，那麼：

> **它的前任，就是左子樹中的最大值。**

為什麼？

因為：

* 左子樹所有節點都比當前節點小
* 而前任要找的是「比它小，但又最接近它」的那個值
* 所以只要在左子樹裡挑最大的即可

例如：

* `4` 的左子樹是 `1, 2, 3`
* 其中最大的就是 `3`
* 所以 `4` 的前任是 `3`

再例如：

* `6` 的左子樹只有 `5`
* 所以 `6` 的前任就是 `5`

### 情況 2：節點沒有左子樹

如果節點沒有左子樹，那就不能從左子樹找了，這時要改看祖先。

規則是：

> **離它最近、而且是「自左而來」的祖先，就是它的前任。**

這句話比較抽象，我把它拆開講。

#### 什麼叫「自左而來」？

假設你在查找某個 key 時，過程中有一步是：

```text
某祖先 -> 向右走
```

這就代表：

* 你要找的節點在這個祖先的右邊
* 換句話說，這個祖先比目標節點小
* 所以它才有資格成為前任候選人

例如查找 `8` 的路徑：

```text
4 -> 7 -> 8
```

其中：

* `4 -> 7` 是向右走，所以 `4` 是自左而來的祖先
* `7 -> 8` 也是向右走，所以 `7` 也是自左而來的祖先

而離 `8` 最近的是 `7`，所以：

> `8` 的前任是 `7`

再例如查找 `1`：

```text
4 -> 2 -> 1
```

整個過程都在向左走，表示根本沒有「自左而來」的祖先，所以：

> `1` 的前任是 `null`

## 五、前任方法的實作思路

### 第一步：先找到目標節點

這一段和 `get` 很像：

* `key < p.key`，往左走
* `key > p.key`，往右走
* `key == p.key`，找到節點

只是這次在查找過程中，我們還要順便記錄：

> 最近一個「自左而來」的祖先

也就是說，只要發生：

```java
p = p.right;
```

就表示目前這個祖先比目標節點小，可以先暫存起來。

### 第二步：根據兩種情況決定答案

找到節點後：

* 如果它有左子樹，前任 = 左子樹最大值
* 如果它沒有左子樹，前任 = 最近的自左而來祖先

整理後的程式如下：

```java
public Object predecessor(int key) {
    BSTNode p = root;
    BSTNode ancestorFromLeft = null;

    while (p != null) {
        if (key < p.key) {
            p = p.left;
        } else if (p.key < key) {
            ancestorFromLeft = p; // 記錄最近的自左而來祖先
            p = p.right;
        } else {
            break;
        }
    }

    // 沒找到節點
    if (p == null) {
        return null;
    }

    // 情況1：有左子樹，前任是左子樹最大值
    if (p.left != null) {
        return max(p.left);
    }

    // 情況2：沒有左子樹，前任是最近的自左而來祖先
    return ancestorFromLeft != null ? ancestorFromLeft.key : null;
}

public Object max(BSTNode node) {
    if (node == null) {
        return null;
    }
    BSTNode current = node;
    while (current.right != null) {
        current = current.right;
    }
    return current.key;
}
```

## 六、後任（successor）的規則

後任和前任剛好是鏡像關係，也分成兩種情況。

### 情況 1：節點有右子樹

如果某個節點有右子樹，那麼：

> **它的後任，就是右子樹中的最小值。**

因為：

* 右子樹所有節點都比當前節點大
* 後任要找的是「比它大，但又最接近它」的那個值
* 所以只要在右子樹裡挑最小的即可

例如：

* `2` 的右子樹是 `3`

* 所以 `2` 的後任是 `3`

* `7` 的右子樹是 `8`

* 所以 `7` 的後任是 `8`

### 情況 2：節點沒有右子樹

如果節點沒有右子樹，就要改看祖先。

規則是：

> **離它最近、而且是「自右而來」的祖先，就是它的後任。**

#### 什麼叫「自右而來」？

如果你在查找過程中有一步是：

```text
某祖先 -> 向左走
```

這表示：

* 你要找的節點在這個祖先的左邊
* 換句話說，這個祖先比目標節點大
* 所以它可以成為後任候選人

例如查找 `1` 的路徑：

```text
5 -> 2 -> 1
```

其中：

* `5 -> 2` 是向左走，所以 `5` 是自右而來祖先
* `2 -> 1` 也是向左走，所以 `2` 也是自右而來祖先

離 `1` 最近的是 `2`，所以：

> `1` 的後任是 `2`

再例如查找 `8`：

整路都在向右走，所以沒有自右而來的祖先，因此：

> `8` 的後任是 `null`

## 七、後任方法的實作思路

和前任完全對稱。

查找目標節點時：

* 如果 `key < p.key`，表示要往左走
  這時目前節點可能是「自右而來」祖先，要記錄下來
* 如果 `key > p.key`，往右走
* 找到節點後再分情況判斷

整理後的程式如下：

```java
public Object successor(int key) {
    BSTNode p = root;
    BSTNode ancestorFromRight = null;

    while (p != null) {
        if (key < p.key) {
            ancestorFromRight = p; // 記錄最近的自右而來祖先
            p = p.left;
        } else if (p.key < key) {
            p = p.right;
        } else {
            break;
        }
    }

    // 沒找到節點
    if (p == null) {
        return null;
    }

    // 情況1：有右子樹，後任是右子樹最小值
    if (p.right != null) {
        return min(p.right);
    }

    // 情況2：沒有右子樹，後任是最近的自右而來祖先
    return ancestorFromRight != null ? ancestorFromRight.key : null;
}

private Object min(BSTNode node) {
    if (node == null) {
        return null;
    }
    BSTNode p = node;
    while (p.left != null) {
        p = p.left;
    }
    return p.key;
}
```

# 二叉搜索樹 delete 方法實現

接下來我們來學習二叉搜索樹的刪除操作。

刪除比 `get`、`put` 更複雜，因為刪掉一個節點後，還要想辦法讓整棵樹仍然保持二叉搜索樹的性質。

## 一、刪除操作的 4 種情況

刪除某個節點時，可以分成 4 種情況：

### 1. 被刪除節點沒有左孩子

這時直接把它的**右孩子**接到它原本的位置。

### 2. 被刪除節點沒有右孩子

這時直接把它的**左孩子**接到它原本的位置。

### 3. 被刪除節點左右孩子都沒有

也就是葉子節點。
其實這種情況已經被前兩種涵蓋了：

* 沒有左孩子，那就把右孩子 `null` 接上去
* 沒有右孩子，那就把左孩子 `null` 接上去

所以不需要額外分開寫。

### 4. 被刪除節點左右孩子都有

這是最複雜的一種情況。
通常會選擇用它的**後繼節點**來取代它。

## 二、先看前兩種簡單情況

### 情況 1：沒有左孩子

例如下面這棵樹，要刪除節點 `4`：

```text
         3
           \
            4
              \
               7
              /  
             6
```

節點 `4` 沒有左孩子，所以只要把 `4` 的右孩子 `7` 接到 `4` 原本的位置即可。

刪除後變成：

```text
         3
           \
            7
           /  
          6
```

也就是說：

> **沒有左孩子時，直接用右孩子補上。**

### 情況 2：沒有右孩子

例如下面這棵樹，要刪除節點 `4`：

```text
           6
         /  
        4
      /
     2
    / \
   1   3
```

節點 `4` 沒有右孩子，所以只要把 `4` 的左孩子 `2` 接到 `4` 原本的位置即可。

刪除後變成：

```text
           6
         /  
        2
       / \
      1   3
```

也就是說：

> **沒有右孩子時，直接用左孩子補上。**

## 三、刪除前，先找到要刪掉的節點與它的父節點

和 `get` 很像，刪除時也要先從根節點開始查找目標 `key`。

不過這次除了找到目標節點 `p`，還要順便記住它的父節點 `parent`。

因為之後要把別的節點接到 `p` 原本的位置時，必須知道：

* `p` 是 `parent` 的左孩子
* 還是右孩子

基本查找骨架如下：

```java
public Object delete(int key) {
    BSTNode p = root;
    BSTNode parent = null;

    while (p != null) {
        if (key < p.key) {
            parent = p;
            p = p.left;
        } else if (key > p.key) {
            parent = p;
            p = p.right;
        } else {
            break;
        }
    }

    if (p == null) {
        return null; // 沒找到
    }

    return p.value;
}
```

## 四、什麼是「托孤」？

意思其實就是：

> **把某個節點的孩子，接到它父節點原本指向它的位置上。**

例如刪掉 `4`，把 `7` 接上去：

```text
         3
           \
            4
              \
               7
```

刪除 `4` 後，要讓 `3.right = 7`。

這種「把 child 接到 deleted 原本位置」的操作，會一直重複出現，
所以很適合抽成一個方法。

### shift 方法

```java
private void shift(BSTNode parent, BSTNode deleted, BSTNode child) {
    if (parent == null) {           // deleted 是根節點
        root = child;
    } else if (parent.left == deleted) {
        parent.left = child;
    } else {
        parent.right = child;
    }
}
```

這個方法的意思是：

* 如果刪掉的是根節點，就直接改 `root`
* 如果刪掉的是父節點的左孩子，就讓 `parent.left = child`
* 如果刪掉的是父節點的右孩子，就讓 `parent.right = child`

## 五、先完成情況 1、2、3

因為情況 3（葉子節點）其實已包含在前兩種裡，所以真正程式可以寫成：

```java
public Object delete(int key) {
    BSTNode p = root;
    BSTNode parent = null;

    while (p != null) {
        if (key < p.key) {
            parent = p;
            p = p.left;
        } else if (key > p.key) {
            parent = p;
            p = p.right;
        } else {
            break;
        }
    }

    if (p == null) {
        return null;
    }

    if (p.left == null) {           // 沒有左孩子（包含葉子節點）
        shift(parent, p, p.right);
    } else if (p.right == null) {   // 沒有右孩子
        shift(parent, p, p.left);
    } else {
        // 左右孩子都有，後面再處理
    }

    return p.value;
}
```

你可以這樣記：

* `left == null` → 用 `right` 補
* `right == null` → 用 `left` 補

## 六、最難的情況：左右孩子都有

現在來看最複雜的情況。

例如下面這棵樹，要刪除節點 `4`：

```text
         7
       /  
      4
    /   \ 
   2     5
  / \     \
 1   3     6
```

節點 `4` 同時有左子樹和右子樹。
這時不能隨便拿某個孩子上來頂替，否則可能會破壞 BST 的大小規則。

正確做法通常是：

> **用被刪除節點的後繼節點來替代它。**

### 為什麼選後繼節點？

後繼節點就是：

> **右子樹中的最小值**

因為它滿足兩件事：

1. 它比被刪除節點大
2. 它又是所有比被刪除節點大的節點中最小的那個

所以拿它來替代，最不會破壞 BST 的結構。

### 如何找後繼節點？

如果某節點有右子樹，那它的後繼節點一定是：

> **右子樹一路往左走到底的那個節點**

所以刪除 `p` 時，後繼節點可以這樣找：

```java
BSTNode successor = p.right;
while (successor.left != null) {
    successor = successor.left;
}
```

但這還不夠，因為之後還要處理後繼節點原本的位置，
所以還要再記錄它的父節點 `successorParent`：

```java
BSTNode successor = p.right;
BSTNode successorParent = p;

while (successor.left != null) {
    successorParent = successor;
    successor = successor.left;
}
```

### 後繼節點有兩種情況

這裡是整個刪除最容易卡住的地方。

#### 情況 A：後繼節點和被刪除節點相鄰

例如：

```text
         7
       /  
      4
    /   \ 
   2     5
  / \     \
 1   3     6
```

刪除 `4` 時：

* `4.right` 是 `5`
* `5` 本身就是右子樹最小值
* 所以後繼節點就是 `5`

這時 `successorParent == p`，表示它們相鄰。

這種情況比較簡單，因為 `5` 原本就在 `4` 的右邊，
不需要先把 `5` 從更深的位置拆出來。

#### 情況 B：後繼節點和被刪除節點不相鄰

例如：

```text
         4
       /   \ 
      2     8
           /  \
          7    9
         / 
        5  
         \
          6
```

刪除 `4` 時：

* 右子樹是 `8`
* 一路往左找到 `7`
* 再往左找到 `5`

所以 `5` 是 `4` 的後繼節點。

但 `5` 不是直接接在 `4` 右邊，它更深。
如果直接把 `5` 拉上來，`5` 原本的右孩子 `6` 就會丟失。

因此這種情況必須先做一步：

> **先把 `5` 的右孩子 `6` 接回 `5` 原本父節點 `7` 的左邊**

也就是：

```text
7.left = 6
```

之後 `5` 才能放心頂上去。

##### 為什麼只需要處理 successor.right？

因為後繼節點是「右子樹中的最小值」。

既然它是最小值，那它不可能還有左孩子。
否則左邊那個節點會比它更小，那它就不是最小值了。

所以在「不相鄰」情況下，處理後繼節點原位置時，只需要：

```java
shift(successorParent, successor, successor.right);
```

不需要考慮 `successor.left`。

### 完整刪除思路

當 `p.left != null && p.right != null` 時：

#### 第一步：找到後繼節點 successor

也就是右子樹最小值。

#### 第二步：如果不相鄰，先處理後繼節點原本的位置

把 `successor.right` 接到 `successorParent` 原本指向 `successor` 的地方。

#### 第三步：讓 successor 取代 p

也就是把 `successor` 接到 `p` 原本的位置。

#### 第四步：補上 successor 的左右孩子

因為 `successor` 上來後，要接管 `p` 原本的左右子樹。

## 七、完整程式碼

整理後的 `delete` 方法如下：

```java
public Object delete(int key) {
    BSTNode p = root;
    BSTNode parent = null;

    // 1. 找到待刪除節點 p 以及其父節點 parent
    while (p != null) {
        if (key < p.key) {
            parent = p;
            p = p.left;
        } else if (key > p.key) {
            parent = p;
            p = p.right;
        } else {
            break;
        }
    }

    if (p == null) {
        return null;
    }

    // 2. 沒有左孩子
    if (p.left == null) {
        shift(parent, p, p.right);
    }
    // 3. 沒有右孩子
    else if (p.right == null) {
        shift(parent, p, p.left);
    }
    // 4. 左右孩子都有
    else {
        BSTNode successor = p.right;
        BSTNode successorParent = p;

        // 找右子樹最小值
        while (successor.left != null) {
            successorParent = successor;
            successor = successor.left;
        }

        // 後繼節點和被刪除節點不相鄰
        if (successorParent != p) {
            shift(successorParent, successor, successor.right);
            successor.right = p.right;
        }

        // 後繼節點頂替被刪除節點
        shift(parent, p, successor);
        successor.left = p.left;
    }

    return p.value;
}
```

搭配前面的 `shift` 方法：

```java
private void shift(BSTNode parent, BSTNode deleted, BSTNode child) {
    if (parent == null) {
        root = child;
    } else if (parent.left == deleted) {
        parent.left = child;
    } else {
        parent.right = child;
    }
}
```

## 八、為什麼 successor.left = p.left 一定要做？

因為後繼節點上來後，要接手被刪除節點原本的位置。

例如原本是：

```text
      4
    /   \
   2     ...
```

如果 `4` 被 `5` 取代，那 `5` 必須接住 `4` 原本的左子樹 `2`，否則左半邊整棵子樹就斷掉了。

所以：

```java
successor.left = p.left;
```

一定要做。

## 九、為什麼 successor.right = p.right 只在不相鄰時才做？

這點很關鍵。

如果 `successorParent == p`，代表後繼節點就是 `p.right`，也就是它們相鄰。

例如：

```text
      4
    /   \
   2     5
          \
           6
```

這時 `successor` 本來就是 `p.right`，
它原本的 `right` 指向就已經正確，不需要重新指定。

只有在不相鄰時，`successor` 是從更深處拉上來的，
它原本的 `right` 不是 `p.right`，所以才需要：

```java
successor.right = p.right;
```

## 十、為什麼不能隨便選一個節點來頂替？

例如下面這棵樹刪除 `4`：

```text
         4
       /   \ 
      2     8
           /  \
          7    9
         / 
        5  
         \
          6
```

如果你拿 `6` 來替代 `4`：

```text
         6
       /   \ 
      2     8
           /  \
          7    9
         / 
        5
```

就會出問題，因為 `5` 比 `6` 小，卻跑到 `6` 的右子樹中，違反 BST 規則。

所以不能隨便挑節點。
必須挑：

* 後繼節點（右子樹最小值）
* 或前任節點（左子樹最大值）

這兩者才安全。

## 十一、能不能用前任節點替代？

可以。

除了用後繼節點，也可以用：

> **前任節點 = 左子樹中的最大值**

來取代被刪除節點。

本質上是對稱的，只是實作時你要改成：

* 往左子樹找最大值
* 處理前任節點原本的位置
* 再讓前任節點頂上來

不過實務上很多教材會優先示範「用後繼節點替代」，因為思路比較常見。

# 二叉搜索樹使用遞歸實現 delete 方法

前面我們已經看過非遞歸版的刪除，接下來來看 **遞歸版的 `delete` 方法**。

遞歸版通常會拆成兩個方法：

```java
public Object delete(int key) { }

private BSTNode doDelete(BSTNode node, int key) { }
```

其中：

* `delete(int key)`：給外部呼叫
* `doDelete(BSTNode node, int key)`：真正負責遞歸刪除

這裡的 `node` 表示：

> **這一次刪除，要從哪一個節點開始處理。**

## 一、為什麼 doDelete 一定要有返回值？

這是遞歸版刪除最核心的地方。

原因是：

> **刪除一個節點後，它原本的位置可能要由別的節點接替。**

例如下面這棵樹：

```text 
         4
        / \
       2   6
      / \   \
     1  3    7
```

如果要刪除 `6`，刪掉之後應該讓 `7` 補上 `6` 原本的位置：

```text 
         4
        / \
       2   7
      / \   
     1  3
```

也就是說，刪除 `6` 之後，`doDelete(6, 6)` 應該回傳 `7`。
這樣它的父節點 `4` 才能做這件事：

```java
node.right = doDelete(node.right, key);
```

也就是把 `4.right` 改成 `7`。

## 二、遞歸版刪除的本質

你可以把 `doDelete(node, key)` 理解成一句話：

> **在以 `node` 為根的子樹中刪除 `key`，並回傳「刪除後新的子樹根節點」。**

這句話非常重要。

因為遞歸時，每一層都不是單純「找到了就刪」，而是：

* 子節點刪完之後
* 父節點要把自己的 `left` 或 `right`
* 重新指向「刪除後的新子樹根」

所以才會有這種寫法：

```java
node.left = doDelete(node.left, key);
node.right = doDelete(node.right, key);
```

## 三、先完成查找骨架

遞歸版刪除，前半段和 `get` 很像。

### 1. 如果目前節點是 null

表示沒找到，直接回傳 `null`

### 2. 如果 key 比目前節點小

代表要去左子樹刪除

### 3. 如果 key 比目前節點大

代表要去右子樹刪除

### 4. 整理後的骨架如下：

```java
private BSTNode doDelete(BSTNode node, int key) {
    if (node == null) {
        return null;
    }

    if (key < node.key) {
        node.left = doDelete(node.left, key);
        return node;
    }

    if (key > node.key) {
        node.right = doDelete(node.right, key);
        return node;
    }

    // 走到這裡，表示找到要刪除的節點
    return null;
}
```

這裡的重點是：

* 去左邊刪完後，要把結果重新接回 `node.left`
* 去右邊刪完後，要把結果重新接回 `node.right`

否則父子關係就接不起來了。

## 四、找到目標節點後，刪除仍然分 4 種情況

和非遞歸版一樣，遞歸版找到節點後，也要分成 4 種情況處理：

### 1. 沒有左孩子

直接回傳右孩子

### 2. 沒有右孩子

直接回傳左孩子

### 3. 沒有孩子

其實就是前兩種的特例，本質上是回傳 `null`

### 4. 左右孩子都有

要找後繼節點來替代它

## 五、前兩種簡單情況

### 情況 1：沒有左孩子

例如刪除 `6`：

```text
         4
        / \
       2   6
      /     \
     1       7
```

因為 `6.left == null`，所以直接回傳 `6.right`，也就是 `7`。

```java
if (node.left == null) {
    return node.right;
}
```

### 情況 2：沒有右孩子

如果某個節點沒有右孩子，就讓左孩子補上：

```java
if (node.right == null) {
    return node.left;
}
```

這兩種情況加進去後，程式會變成：

```java 
private BSTNode doDelete(BSTNode node, int key) {
    if (node == null) {
        return null;
    }

    if (key < node.key) {
        node.left = doDelete(node.left, key);
        return node;
    }

    if (key > node.key) {
        node.right = doDelete(node.right, key);
        return node;
    }

    // 找到要刪除的節點
    if (node.left == null) {
        return node.right;
    }

    if (node.right == null) {
        return node.left;
    }

    return null;
}
```

## 六、最複雜的情況：左右孩子都有

當被刪除節點同時有左子樹和右子樹時，不能直接隨便拿一個孩子頂上去，否則可能破壞 BST 的結構。

這時通常會選擇：

> **用後繼節點（右子樹最小值）來取代被刪除節點。**

### 先看相鄰的情況

先看一段原始思路：

```java
BSTNode successor = node.right;
while (successor.left != null) {
    successor = successor.left;
}
successor.left = node.left;

return successor;
```

這段程式只適合一種比較簡單的情況：

> **被刪除節點和後繼節點相鄰**

也就是說，後繼節點剛好就是 `node.right`，中間沒有更深的左孩子鏈。

### 為什麼只考慮相鄰情況不夠？

看下面這棵樹：

```text
         4
       /   \ 
      2     8
           /  \
          7    9
         / 
        5  
         \
          6
```

如果要刪除 `4`，它的後繼節點是 `5`。

此時 `5` 並不是直接接在 `4.right` 上，而是藏在更深的位置：

* `4.right = 8`
* `8.left = 7`
* `7.left = 5`

如果你只是直接把 `5` 拿上來，然後：

```java
successor.left = node.left;
return successor;
```

那原本右子樹裡的 `7、8、9` 這些節點就有可能接丟，整棵樹會壞掉。

### 不相鄰時，真正的關鍵做法

關鍵想法是：

> **先把後繼節點從原本的位置刪掉，再讓它來頂替被刪除節點。**

也就是：

1. 先找到後繼節點 `successor`
2. 在 `node.right` 這棵子樹中，把 `successor.key` 刪掉
3. 這樣後繼節點原本的位置就會被妥善處理
4. 接著再把 `successor` 拿來當新的根

原筆記中最重要的一行就是：

```java
successor.right = doDelete(node.right, successor.key);
```

它的意思是：

> 在原本的右子樹中，把後繼節點刪掉，並把刪完後的結果接回 `successor.right`。

## 七、完整的遞歸刪除核心寫法

整理後如下：

```java 
private BSTNode doDelete(BSTNode node, int key) {
    if (node == null) {
        return null;
    }

    if (key < node.key) {
        node.left = doDelete(node.left, key);
        return node;
    }

    if (key > node.key) {
        node.right = doDelete(node.right, key);
        return node;
    }

    // 找到要刪除的節點
    if (node.left == null) {
        return node.right;
    }

    if (node.right == null) {
        return node.left;
    }

    // 左右孩子都有：找後繼節點
    BSTNode successor = node.right;
    while (successor.left != null) {
        successor = successor.left;
    }

    // 先把後繼節點從右子樹中刪掉，再重建 successor 的左右子樹
    successor.right = doDelete(node.right, successor.key);
    successor.left = node.left;

    return successor;
}
```

## 八、為什麼 root = doDelete(root, key) 不能省略？

外部方法通常會這樣寫：

```java id="v6h9zo"
public Object delete(int key) {
    root = doDelete(root, key);
}
```

這裡最重要的是：

```java
root = doDelete(root, key);
```

這一步不能少。

原因是：

> **如果被刪除的節點剛好就是根節點，那刪除後整棵樹的根可能會改變。**

例如：

```text
         4
       /   \ 
      2     8
           /  \
          7    9
         / 
        5  
         \
          6
```

如果刪除的是根節點 `4`，最後頂上來的會是後繼節點 `5`。

所以刪除後新的根節點不再是 `4`，而是 `5`。
如果你不把 `doDelete` 的返回結果重新賦值給 `root`，那整棵樹的入口就錯了。

## 九、delete 和 doDelete 的返回值不同

這裡很容易搞混。

### doDelete 的返回值

`doDelete` 回傳的是：

> **刪除後新的子樹根節點**

它是給遞歸過程用的，用來重建樹的結構。

### delete 的返回值

`delete` 通常想回傳的是：

> **被刪除節點原本的 value**

也就是給外部使用者看的結果。

### 如何同時做到「重建樹」與「回傳被刪除值」？

因為 `doDelete` 已經拿來回傳「新的子樹根」，
所以不能再直接用它回傳被刪除節點的值。

原筆記的做法是額外傳入一個集合，用來保存被刪除節點的 value：

```java
public Object delete(int key) {
    ArrayList<Object> result = new ArrayList<>();
    root = doDelete(root, key, result);
    return result.isEmpty() ? null : result.get(0);
}
```

對應的輔助方法變成：

```java
private BSTNode doDelete(BSTNode node, int key, ArrayList<Object> result) {
    if (node == null) {
        return null;
    }

    if (key < node.key) {
        node.left = doDelete(node.left, key, result);
        return node;
    }

    if (key > node.key) {
        node.right = doDelete(node.right, key, result);
        return node;
    }

    // 找到要刪除的節點，先保存它的 value
    result.add(node.value);

    if (node.left == null) {
        return node.right;
    }

    if (node.right == null) {
        return node.left;
    }

    BSTNode successor = node.right;
    while (successor.left != null) {
        successor = successor.left;
    }

    // 這裡不能沿用 result，因為刪除後繼節點只是結構調整，
    // 我們不想把後繼節點的 value 也記進去
    successor.right = doDelete(node.right, successor.key, new ArrayList<>());
    successor.left = node.left;

    return successor;
}
```

這樣就能同時做到兩件事：

1. 正確重建刪除後的樹
2. 把最初那個真正被刪除目標節點的 value 回傳出去 

# 二叉搜索樹的範圍查詢方法

除了單點查找 `get(key)` 以外，二叉搜索樹也很適合做**範圍查詢**。

例如我們可能想查：

* 所有 **小於某個 key** 的值
* 所有 **大於某個 key** 的值
* 所有 **介於兩個 key 之間** 的值

對應的方法如下：

```java
// 找 < key 的所有 value
public List<Object> less(int key) { 
   ArrayList<Object> result = new ArrayList<>();
   return result;
}

// 找 > key 的所有 value
public List<Object> greater(int key) {
   ArrayList<Object> result = new ArrayList<>();
   return result; 
}

// 找 >= key1 且 <= key2 的所有值
public List<Object> between(int key1, int key2) {
   ArrayList<Object> result = new ArrayList<>();
   return result;
}
```

## 一、這三個方法的核心思想

這幾個方法的實作基礎，其實都來自 BST 的一個重要性質：

> **對二叉搜索樹做中序遍歷，會得到升序結果。**

例如下面這棵樹：

```text
          4
        /   \ 
       2     6
      / \   / \
     1   3 5   7
```

如果做中序遍歷，得到的順序就是：

```text
1, 2, 3, 4, 5, 6, 7
```

因此，範圍查詢的核心思路就是：

> **在中序遍歷的過程中，加入條件判斷，只收集符合範圍的節點。**

## 二、less(key)：找出所有小於 key 的 value

### 思路

因為中序遍歷得到的是升序序列，所以：

* 只要目前訪問到的節點 `node.key < key`
* 就把它的 `value` 加入結果

而且由於遍歷結果是遞增的：

* 一旦碰到第一個 `node.key >= key`
* 後面的節點只會更大
* 因此可以直接停止遍歷

這就是 `less` 方法可以提前結束的原因。

### 實作

```java
public List<Object> less(int key) {
    ArrayList<Object> result = new ArrayList<>();
    BSTNode p = root;
    LinkedList<BSTNode> stack = new LinkedList<>();

    while (p != null || !stack.isEmpty()) {
        if (p != null) {
            stack.push(p);
            p = p.left;
        } else {
            BSTNode pop = stack.pop();

            // 比較的是 key，加入結果的是 value
            if (pop.key < key) {
                result.add(pop.value);
            } else {
                // 中序遍歷是升序，後面只會更大，可以直接停止
                break;
            }

            p = pop.right;
        }
    }
    return result;
}
```

### 如何理解這段程式？

這裡使用的是**非遞迴中序遍歷**：

1. 一路往左走，把沿路節點放進 stack
2. 左邊走到底後，彈出一個節點來訪問
3. 訪問完後，再去處理它的右子樹

而 `less(key)` 的關鍵就在這一段：

```java
if (pop.key < key) {
    result.add(pop.value);
} else {
    break;
}
```

因為遍歷是升序的，所以一旦碰到不小於 `key` 的節點，就可以直接結束。

## 三、greater(key)：找出所有大於 key 的 value

### 先看基本思路

最直覺的方式，也是沿用中序遍歷：

* 如果 `node.key > key`，就加入結果
* 否則略過

寫法如下：

```java
public List<Object> greater(int key) {
    ArrayList<Object> result = new ArrayList<>();
    BSTNode p = root;
    LinkedList<BSTNode> stack = new LinkedList<>();

    while (p != null || !stack.isEmpty()) {
        if (p != null) {
            stack.push(p);
            p = p.left;
        } else {
            BSTNode pop = stack.pop();

            if (pop.key > key) {
                result.add(pop.value);
            }

            p = pop.right;
        }
    }
    return result;
}
```

### 這種寫法的缺點

這個版本雖然正確，但效率不夠好。

因為中序遍歷的順序是：

```text
1, 2, 3, 4, 5, 6, 7
```

假設我們要找 `> 6` 的值，那麼：

* 前面的 `1, 2, 3, 4, 5, 6` 都得先走過
* 直到遇到 `7` 才真正加入結果

也就是說：

> **即使答案很少，仍然可能要遍歷很多不需要的節點。**

而且這種從小到大的遍歷方式，不能像 `less` 一樣提早停止。

### greater(key) 的優化思路

既然中序遍歷是從小到大：

```text
1, 2, 3, 4, 5, 6, 7
```

那我們可不可以反過來，從大到小遍歷呢？

答案是可以，這就是：

> **反向中序遍歷**

它的順序是：

> **右子樹 → 根節點 → 左子樹**

對上面的樹做反向中序遍歷，結果就是：

```text
7, 6, 5, 4, 3, 2, 1
```

這時如果我們要找 `> 6` 的值：

* 先看到 `7`，符合條件，加入結果
* 再看到 `6`，已經不大於 `6`
* 因為後面只會更小，所以可以直接停止

這樣就比普通中序遍歷更有效率。

### 優化後的 greater(key) 實作

```java
public List<Object> greater(int key) {
    ArrayList<Object> result = new ArrayList<>();
    BSTNode p = root;
    LinkedList<BSTNode> stack = new LinkedList<>();

    while (p != null || !stack.isEmpty()) {
        if (p != null) {
            stack.push(p);
            p = p.right; // 先往右走
        } else {
            BSTNode pop = stack.pop();

            if (pop.key > key) {
                result.add(pop.value);
            } else {
                // 反向中序遍歷是降序，後面只會更小，可以停止
                break;
            }

            p = pop.left;
        }
    }
    return result;
}
```

#### 這段程式和普通中序遍歷差在哪？

普通中序遍歷是：

* 左
* 根
* 右

反向中序遍歷則是：

* 右
* 根
* 左

所以差別只在兩個地方：

##### 普通中序

```java
p = p.left;
...
p = pop.right;
```

##### 反向中序

```java
p = p.right;
...
p = pop.left;
```

這樣就能把遍歷順序從升序改成降序。

## 四、between(key1, key2)：找出介於兩者之間的 value

### 思路

這個方法一樣可以用中序遍歷。

因為中序遍歷結果是升序，所以：

* 如果 `node.key` 落在 `[key1, key2]` 之間，就加入結果
* 如果目前 `node.key > key2`，那後面的節點一定更大，可以直接停止

### 實作

```java
public List<Object> between(int key1, int key2) {
    ArrayList<Object> result = new ArrayList<>();
    BSTNode p = root;
    LinkedList<BSTNode> stack = new LinkedList<>();

    while (p != null || !stack.isEmpty()) {
        if (p != null) {
            stack.push(p);
            p = p.left;
        } else {
            BSTNode pop = stack.pop();

            if (pop.key >= key1 && pop.key <= key2) {
                result.add(pop.value);
            } else if (pop.key > key2) {
                // 中序遍歷是升序，後面只會更大，可以停止
                break;
            }

            p = pop.right;
        }
    }
    return result;
}
```

### 如何理解這段程式？

`between` 和 `less` 很像，都是利用升序特性做提前停止。

例如要找 `[3, 5]`：

* 遇到 `1`、`2` 時，雖然不加入結果，但還不能停
* 遇到 `3`、`4`、`5` 時，加入結果
* 一旦遇到 `6`，因為 `6 > 5`，就可以直接停止

# LeetCode 450 - 删除二叉搜索树中的节点

> [450. 删除二叉搜索树中的节点](https://leetcode.cn/problems/delete-node-in-a-bst/description/)


```java
public TreeNode deleteNode(TreeNode node, int key) {
    // 1. 沒找到目標節點
    if (node == null) {
        return null;
    }

    // 2. 目標比目前節點小，往左子樹刪
    if (key < node.val) {
        node.left = deleteNode(node.left, key);
        return node;
    }

    // 3. 目標比目前節點大，往右子樹刪
    if (key > node.val) {
        node.right = deleteNode(node.right, key);
        return node;
    }

    // 4. 找到要刪除的節點後，開始分類處理

    // 情況 1：只有右孩子，直接讓右孩子補上
    if (node.left == null) {
        return node.right;
    }

    // 情況 2：只有左孩子，直接讓左孩子補上
    if (node.right == null) {
        return node.left;
    }

    // 情況 3：左右孩子都存在
    // 找右子樹中的最小節點（後繼節點）
    TreeNode s = node.right;
    while (s.left != null) {
        s = s.left;
    }

    // 把後繼節點從原本右子樹中刪掉，
    // 並把刪除後的新右子樹接到 s.right
    s.right = deleteNode(node.right, s.val);

    // 把原本節點的左子樹接到 s.left
    s.left = node.left;

    // 回傳 s，讓它取代原本被刪除的節點
    return s;
}
```

- 在 LeetCode 中使用的 TreeNode 類相當於我們之前筆記中的 BSTNode 類：
   - TreeNode: val、left、right
   - BSTNode: key、value、left、right

- 它的 TreeNode 沒有 key，比較用的是 TreeNode.val 屬性與待刪除 key 進行比較

# LeetCode 701 - 二叉搜尋樹中的插入操作

> [701. 二叉搜索树中的插入操作](https://leetcode.cn/problems/insert-into-a-binary-search-tree/description/)

## 一、這題本質上就是我們之前做過的 put

其實這題和我們之前實作二叉搜尋樹時的 `put` 很像。

當時 `put` 通常分成兩種情況：

1. **key 已存在**：更新原本節點的 value
2. **key 不存在**：找到適當位置，插入新節點

但是 LeetCode 701 把問題簡化了。

題目已經明確告訴我們：

> `val` 一定與樹中現有節點不同

這代表：

* 不會出現重複值
* 不需要考慮更新
* 只要專心處理「插入新節點」即可

## 二、BST 插入的核心想法

二叉搜尋樹的規則是：

* 左子樹的值都比目前節點小
* 右子樹的值都比目前節點大

所以當我們要插入一個新值時，只要不停比較：

* 比目前節點小，就往左邊找
* 比目前節點大，就往右邊找
* 直到找到空位置，就把新節點放進去

## 三、例子一：插入 1

原本的樹：

```text
         5
        / \
       2   6
        \
         3
```

現在要插入 `1`。

### 第一步：從根節點 5 開始比較

* `1 < 5`
* 所以 `1` 應該放在 `5` 的左子樹

但 `5` 的左邊不是空的，而是節點 `2`，所以不能直接插。

要繼續往左子樹找。

### 第二步：來到節點 2

* `1 < 2`
* 所以 `1` 應該放在 `2` 的左邊

這時發現：

* `2.left == null`

表示找到空位了，所以可以把 `1` 插在這裡。

插入後變成：

```text
         5
        / \
       2   6
      / \
     1   3
```

## 四、例子二：插入 7

原本的樹：

```text
         5
        / \
       2   6
      / \
     1   3
```

現在要插入 `7`。

### 第一步：從根節點 5 開始比較

* `7 > 5`
* 所以要往右子樹找

來到 `6`

### 第二步：比較 6

* `7 > 6`
* 所以 `7` 應該插到 `6` 的右邊

這時發現：

* `6.right == null`

表示找到空位，可以直接插入。

插入後：

```text
         5
        / \
       2   6
      / \   \
     1   3   7
```

## 五、遞迴解法的思路

遞迴版本的想法很自然：

### 如果目前節點是空的

表示我們已經找到應該插入的位置，直接建立新節點回傳。

```java
if (node == null) {
    return new TreeNode(val);
}
```

### 如果 val 比目前節點小

表示新值應該插入左子樹，所以遞迴處理左邊。

```java
if (val < node.val) {
    node.left = insertIntoBST(node.left, val);
}
```

注意這裡不是只呼叫遞迴而已，還要：

```java
node.left = ...
```

因為遞迴回傳的是「插入完成後的左子樹根節點」，要重新接回目前節點。

### 如果 val 比目前節點大

表示新值應該插入右子樹，所以遞迴處理右邊。

```java
else if (node.val < val) {
    node.right = insertIntoBST(node.right, val);
}
```

同樣也要把回傳結果接回 `node.right`。

### 最後回傳目前節點

```java
return node;
```

因為對父節點來說，目前這棵子樹插入完成後，根節點仍然是 `node`。

## 六、整理後的程式碼

下面是加上清楚註解後，比較適合讀者閱讀的版本：

```java
public TreeNode insertIntoBST(TreeNode node, int val) {
    // 1. 找到空位置，建立新節點並回傳
    if (node == null) {
        return new TreeNode(val);
    }

    // 2. 如果插入值比較小，往左子樹插入
    if (val < node.val) {
        node.left = insertIntoBST(node.left, val);
    }
    // 3. 如果插入值比較大，往右子樹插入
    else if (node.val < val) {
        node.right = insertIntoBST(node.right, val);
    }

    // 4. 回傳目前節點，讓上一層遞迴把子樹重新接回去
    return node;
}
```

## 七、為什麼一定要寫 node.left = ... 或 node.right = ...？

這是很多人在學遞迴插入時最容易卡住的地方。

例如這段：

```java
node.left = insertIntoBST(node.left, val);
```

它的意思不是單純「去左子樹插入」而已。

更完整地說，它是在做：

1. 先去左子樹遞迴插入
2. 插完後，取得插入完成後的左子樹根節點
3. 再把這個結果接回 `node.left`

### 用插入 1 來看

原本：

```text
         5
        / \
       2   6
        \
         3
```

執行流程大致是：

#### 在節點 5

因為 `1 < 5`，所以做：

```java
node.left = insertIntoBST(node.left, 1);
```

也就是：

```java
5.left = insertIntoBST(2, 1);
```

#### 在節點 2

因為 `1 < 2`，所以做：

```java
2.left = insertIntoBST(null, 1);
```

#### 在 null

找到空位，建立新節點：

```java
return new TreeNode(1);
```

所以回到上一層後：

```java
2.left = 1
```

此時 `2` 這棵子樹已經變成：

```text
    2
   / \
  1   3
```

再回到節點 `5` 時：

```java
5.left = 2
```

### 這裡為什麼會有「多餘的賦值」？

當遞迴一路返回時，會出現一些看起來有點多餘的重新賦值。

例如插入 `1` 之後：

* `2.left = 1` 是必要的
* 但回到上一層時的 `5.left = 2` 看起來像是多做了一次

因為本來 `5.left` 就已經是 `2`

這確實是遞迴寫法的一個特點：

* 它為了維持統一的遞迴結構
* 每一層都會把「處理後的子樹」重新接回去
* 有些賦值在邏輯上是必要流程
* 但實際上可能沒有改變節點引用

所以可以說：

> 遞迴的優點是寫法簡潔、結構清楚；缺點是可能會有一些額外但無害的賦值操作。

## 八、這題為什麼不用考慮相等的情況？

因為題目已經保證：

> `val` 一定與樹中現有節點不同

所以：

* 不會出現 `val == node.val`
* 不需要更新節點
* 只會往左或往右繼續找位置

因此程式中只需要處理：

* `val < node.val`
* `val > node.val`

即可。

# LeetCode 98 - 驗證二叉搜尋樹

> [98. 验证二叉搜索树](https://leetcode.cn/problems/validate-binary-search-tree/description/)

## 一、什麼是有效的二叉搜尋樹？

一棵有效的二叉搜尋樹，必須同時滿足以下三個條件：

1. **左子樹所有節點的值都必須嚴格小於目前節點**
2. **右子樹所有節點的值都必須嚴格大於目前節點**
3. **左右子樹本身也都必須是二叉搜尋樹**

這裡要特別注意兩個關鍵字：

* **所有節點**
* **嚴格小於 / 嚴格大於**

也就是說，不只是比父節點而已，而是要符合整棵子樹的範圍限制。

## 二、先看幾個例子

### 範例 1：有效的 BST

```text
       4
      / \ 
     2   6
    / \
   1   3
```

這棵樹是合法的，因為：

* `2 < 4`，`6 > 4`
* `1 < 2`，`3 > 2`
* 而且整體都符合 BST 的規則

### 範例 2：看起來局部正確，但整體不合法

```text
       5
      / \ 
     4   6
        / \
       3   7
```

很多人一開始會誤以為這棵樹是合法的，因為：

* `3 < 6`
* `7 > 6`

看起來節點 `6` 的左右孩子都沒問題。

但實際上這棵樹 **不是 BST**，原因是：

* `3` 雖然比 `6` 小
* 但它位於 `5` 的右子樹中
* 既然在 `5` 的右邊，就應該 **大於 5**
* 可是 `3 < 5`

所以它違反了 BST 的定義。

這也說明了一件事：

> 驗證 BST 不能只看「父子關係」，還要看整個節點所在的位置是否符合整體範圍。

### 範例 3：有重複值也不合法

```text
   1
  /
 1
```

這棵樹也不是 BST，因為 BST 要求：

* 左邊 **嚴格小於**
* 右邊 **嚴格大於**

相等不行。

所以：

* `1` 的左孩子也是 `1`
* 不符合「嚴格小於」的規則

## 三、這題的核心思路：利用中序遍歷

這題有一個非常重要的 BST 性質：

> **如果一棵樹是合法的 BST，那麼它的中序遍歷結果一定是嚴格遞增的。**

### 為什麼？

中序遍歷的順序是：

**左子樹 → 根節點 → 右子樹**

而 BST 本身又保證：

* 左邊比根小
* 右邊比根大

所以中序遍歷後，節點值自然會由小到大排列。

### 例如這棵合法 BST

```text
       4
      / \ 
     2   6
    / \
   1   3
```

中序遍歷結果為：

```text 
[1, 2, 3, 4, 6]
```

這是一個嚴格遞增序列，所以它是合法 BST。

### 再看這棵不合法的樹

```text
       5
      / \ 
     4   6
        / \
       3   7
```

中序遍歷結果為：

```text
[4, 5, 3, 6, 7]
```

可以看到：

* 前一個是 `5`
* 後一個是 `3`

出現了 `5 > 3`

這代表中序遍歷結果不是遞增的，所以它不是合法 BST。

## 四、如何在遍歷過程中判斷？

既然合法 BST 的中序遍歷一定是 **嚴格遞增**，那麼我們只要在遍歷時記住「上一個訪問到的節點值」，然後和目前節點比較即可。

判斷規則如下：

1. 如果目前節點值 **大於** 上一個節點值，繼續
2. 如果目前節點值 **小於或等於** 上一個節點值，直接回傳 `false`

也就是說，我們只要在中序遍歷過程中檢查是否始終保持遞增即可。

## 五、先寫出中序遍歷的非遞迴版本

原始的非遞迴中序遍歷如下：

```java
public boolean isValidBST1(TreeNode node) {
   TreeNode p = node;
   LinkedList<TreeNode> stack = new LinkedList<>();
   while (p != null || !stack.isEmpty()) {
      if (p != null) {
            stack.push(p);
            p = p.left;
      } else {
            TreeNode pop = stack.pop();
            p = pop.right;
      }
   }
   return true;
}
```

這段程式的流程是：

* 不斷往左走，沿途把節點放進 stack
* 當走到最左邊後，開始彈出節點處理
* 處理完當前節點後，再去看它的右子樹

這正是中序遍歷的標準寫法。

## 六、加入驗證邏輯

接下來，我們只要在「彈出節點」的地方，加入與前一個值比較的邏輯即可。

```java
public boolean isValidBST1(TreeNode node) {
   TreeNode p = node;
   LinkedList<TreeNode> stack = new LinkedList<>();
   int prev = Integer.MIN_VALUE; // 代表上一个值
   while (p != null || !stack.isEmpty()) {
      if (p != null) {
            stack.push(p);
            p = p.left;
      } else {
            TreeNode pop = stack.pop();
            // 处理值
            if (prev >= pop.val) {
               return false;
            }
            prev = pop.val;
            p = pop.right;
      }
   }
   return true;
}
```

這段邏輯的意思是：

* `prev` 用來記錄上一個訪問到的值
* 每次中序遍歷處理到新節點時：

   * 如果 `prev >= pop.val`，表示不是嚴格遞增，直接回傳 `false`
   * 否則更新 `prev = pop.val`

## 七、為什麼 Integer.MIN_VALUE 會出問題？

這個寫法看起來很合理，但其實藏了一個邊界問題。

假設輸入的樹只有一個節點，而且值剛好是：

```text
-2147483648
```

也就是 `Integer.MIN_VALUE`

那麼一開始：

```java
int prev = Integer.MIN_VALUE;
```

接著遍歷到這個節點時：

```java
if (prev >= pop.val)
```

就會變成：

```java
if (-2147483648 >= -2147483648)
```

條件成立，程式就會錯誤地回傳 `false`。

但實際上，這棵只有一個節點的樹，明明是合法 BST。

### 正確做法：改用 long

問題的本質是：

> 我們拿來初始化 `prev` 的值，還不夠小。

因為節點值本身就是 `int`，最小只能到 `Integer.MIN_VALUE`，所以如果要找一個比所有 `int` 都更小的初始值，就可以改用：

```java
long prev = Long.MIN_VALUE;
```

這樣不管節點值是不是 `Integer.MIN_VALUE`，第一次比較都不會出錯。

## 八、修正後的程式碼

```java
public boolean isValidBST1(TreeNode node) {
   TreeNode p = node;
   LinkedList<TreeNode> stack = new LinkedList<>();
   long prev = Long.MIN_VALUE; // 代表上一个值
   while (p != null || !stack.isEmpty()) {
      if (p != null) {
            stack.push(p);
            p = p.left;
      } else {
            TreeNode pop = stack.pop();
            // 處理目前節點值
            if (prev >= pop.val) {
               return false;
            }
            prev = pop.val;
            p = pop.right;
      }
   }
   return true;
}
```

# LeetCode 98 - 驗證二叉搜索樹

> [98. 验证二叉搜索树](https://leetcode.cn/problems/validate-binary-search-tree/description/)

## 一、什麼是有效的二叉搜索樹？

二叉搜索樹（Binary Search Tree, BST）必須同時滿足：

1. 左子樹所有節點的值都 **嚴格小於** 當前節點
2. 右子樹所有節點的值都 **嚴格大於** 當前節點
3. 左右子樹本身也都必須是二叉搜索樹

注意這裡的關鍵字是：

* **所有節點**
* **嚴格小於 / 嚴格大於**

所以不能只看「父節點和左右孩子」的大小關係，而是要看整棵子樹是否都符合規則。

## 二、先看幾個例子

### 1. 合法的 BST

```text
       4
      / \
     2   6
    / \
   1   3
```

這棵樹是合法的，因為：

* `2 < 4 < 6`
* `1 < 2 < 3`
* 整體也符合 BST 規則

### 2. 不合法的 BST：子樹節點違規

```text
       5
      / \
     4   6
        / \
       3   7
```

乍看之下：

* `6` 的左邊是 `3`
* `6` 的右邊是 `7`

好像對 `6` 來說沒問題。

但實際上 `3` 位於 **根節點 `5` 的右子樹**，所以它應該要 **大於 5**，然而 `3 < 5`，因此這棵樹不合法。

這也說明了一件事：

> 不能只檢查「父節點比左孩子大、比右孩子小」
> 還必須考慮祖先節點對後代節點的限制。

### 3. 不合法的 BST：值重複

```text
   1
  /
 1
```

BST 要求是：

* 左邊 **嚴格小於**
* 右邊 **嚴格大於**

所以出現相等的值也不行。

## 三、解題思路一：利用中序遍歷

### 1. 為什麼可以用中序遍歷？

BST 有一個很重要的特性：

> **對 BST 做中序遍歷，得到的結果一定是嚴格遞增的。**

例如：

```text
       4
      / \
     2   6
    / \
   1   3
```

中序遍歷順序是：

```text
[1, 2, 3, 4, 6]
```

這是嚴格遞增的。

而這棵不合法的樹：

```text
       5
      / \
     4   6
        / \
       3   7
```

中序遍歷結果是：

```text
[4, 5, 3, 6, 7]
```

可以看到 `5 > 3`，不是遞增，因此它不是 BST。

### 2. 核心想法

在中序遍歷的過程中，記錄「前一個訪問到的節點值」`prev`。

每次訪問當前節點時：

* 如果 `current.val > prev`，繼續
* 如果 `current.val <= prev`，直接返回 `false`

## 四、做法一：非遞迴中序遍歷

### 1. 基本中序遍歷框架

```java
public boolean isValidBST1(TreeNode node) {
    TreeNode p = node;
    LinkedList<TreeNode> stack = new LinkedList<>();

    while (p != null || !stack.isEmpty()) {
        if (p != null) {
            stack.push(p);
            p = p.left;
        } else {
            TreeNode pop = stack.pop();
            p = pop.right;
        }
    }
    return true;
}
```

這段只是中序遍歷，還沒有加入驗證邏輯。

### 2. 加入驗證邏輯

```java
public boolean isValidBST1(TreeNode node) {
    TreeNode p = node;
    LinkedList<TreeNode> stack = new LinkedList<>();
    long prev = Long.MIN_VALUE; // 記錄前一個節點值

    while (p != null || !stack.isEmpty()) {
        if (p != null) {
            stack.push(p);
            p = p.left;
        } else {
            TreeNode pop = stack.pop();

            if (prev >= pop.val) {
                return false;
            }

            prev = pop.val;
            p = pop.right;
        }
    }
    return true;
}
```

### 3. 為什麼 prev 要用 long？

如果你寫成：

```java
int prev = Integer.MIN_VALUE;
```

那麼當樹中剛好有一個節點值是 `-2147483648`（也就是 `Integer.MIN_VALUE`）時：

```text
      -2147483648
```

第一次比較就會出現：

```java
prev >= pop.val
```

結果變成 `true`，導致誤判。

所以應該改成：

```java
long prev = Long.MIN_VALUE;
```

這樣就能確保 `prev` 比任何 `int` 節點值都更小。

## 五、做法二：遞迴中序遍歷

前面我們已經知道：

> 如果一棵樹是合法的二叉搜索樹（BST），那麼對它做**中序遍歷**，得到的節點值序列一定是**嚴格遞增**的。

因此，這一題除了可以用非遞迴寫法，也可以直接用**遞迴中序遍歷**來完成。

### 1. 先寫出中序遍歷的遞迴框架

中序遍歷的順序是：

1. 先遍歷左子樹
2. 再處理當前節點
3. 最後遍歷右子樹

所以遞迴框架應該是：

```java
public boolean isValidBST(TreeNode node){
    // 在遞迴遍歷二叉樹時，如果某個節點已經是 null，代表已經走到底了，這時候不需要再往下判斷，直接返回 true 即可。
    if (node == null) {
        return true;
    }
    isValidBST(node.left);
    // 處理當前節點
    isValidBST(node.right);
}
```

不過這段程式碼其實還不完整，因為有兩個問題：

1. 沒有處理 `node == null` 的情況
2. 沒有真的去判斷這棵樹是不是 BST

所以接下來我們要一步一步把它補完整。

### 2. 怎麼在中序遍歷過程中判斷它是不是 BST？

關鍵在於：

> 中序遍歷的結果必須是**嚴格遞增**的。

因此，在遍歷過程中，我們只要記住「上一個被訪問的節點值」，就可以和當前節點比較。

這裡定義一個變數 `prev`，用來記錄上一個節點的值：

```java
long prev = Long.MIN_VALUE;
```

### 3. 在中序遍歷中加入比較邏輯

當我們完成左子樹遍歷後，就輪到處理當前節點。

此時要做的事情是：

1. 檢查當前節點值是否大於 `prev`
2. 如果不是，說明中序序列沒有保持嚴格遞增，直接返回 `false`
3. 如果是，更新 `prev = node.val`
4. 再繼續遍歷右子樹

程式如下：

```java
long prev = Long.MIN_VALUE;

public boolean isValidBST(TreeNode node){
    if (node == null) {
        return true;
    }

    isValidBST(node.left);

    if (prev >= node.val) {
        return false;
    }

    prev = node.val;

    isValidBST(node.right);
}
```

但是這段程式碼還有一個問題：

> 左子樹和右子樹遞迴呼叫後的結果，我們根本沒有接住。

也就是說，就算左子樹或右子樹已經驗證失敗了，這個失敗結果也沒有被利用。

因此還需要再改進。

### 4. 左子樹和右子樹的返回值很重要

一棵樹要是合法的 BST，必須同時滿足三件事：

1. 左子樹合法
2. 當前節點和前一個節點比較合法
3. 右子樹合法

所以左子樹、右子樹遞迴呼叫的結果都必須被考慮進來。

我們可以先改成這樣：

```java
long prev = Long.MIN_VALUE;

public boolean isValidBST(TreeNode node){
    if (node == null) {
        return true;
    }

    boolean a = isValidBST(node.left);

    if (prev >= node.val) {
        return false;
    }
    prev = node.val;

    boolean b = isValidBST(node.right);

    return a && b;
}
```

這樣就比剛才完整了，因為：

* `a` 表示左子樹是否合法
* 當前節點要和 `prev` 比較
* `b` 表示右子樹是否合法
* 最後只有在三者都沒問題時，才會返回 `true`

不過，這段程式碼雖然邏輯正確，卻還不是最有效率的。

### 5. 目前這個寫法的效率問題在哪裡？

我們來看一棵**不合法的二叉搜索樹**：

```text
         4
        / \
       2   6
          / \
         3   7
```

這棵樹不合法，原因是：

* 節點 `3` 位於根節點 `4` 的右子樹中
* 所以 `3` 應該要大於 `4`
* 但實際上 `3 < 4`

因此它不是合法的 BST。

### 6. 用目前的程式碼來跑一次流程

我們來按照中序遍歷的順序，實際走一次：

#### 第 1 步：從根節點 4 開始

中序遍歷會先一路往左走，先走到 `2`。

#### 第 2 步：處理節點 2

* `2.left` 是 `null`，返回 `true`
* 比較 `prev` 和 `2`
* 一開始 `prev = Long.MIN_VALUE`，所以 `2` 合法
* 更新 `prev = 2`
* `2.right` 是 `null`，返回 `true`
* 所以節點 `2` 整體返回 `true`

#### 第 3 步：回到節點 4

* 左子樹 `2` 已經驗證通過
* 現在比較 `prev` 和 `4`
* 此時 `prev = 2`，而 `4 > 2`，合法
* 更新 `prev = 4`

#### 第 4 步：準備處理 4 的右子樹

接下來會進入節點 `6`，但中序遍歷不會先處理 `6` 本身，而是先處理 `6.left`，也就是節點 `3`。

#### 第 5 步：處理節點 3

* `3.left` 是 `null`，返回 `true`
* 接著比較 `prev` 和 `3`
* 此時 `prev = 4`
* 發現 `prev >= node.val`，也就是 `4 >= 3`
* 這代表中序序列不是嚴格遞增
* 所以節點 `3` 驗證失敗，返回 `false`

到這裡其實就已經可以確定：

> 整棵樹不是 BST 了。

### 7. 問題來了：失敗之後，程式還會繼續往下走

假設現在回到節點 `6`，那麼它的執行會變成這樣：

```java
node = 6
boolean a = isValidBST(6.left); // 也就是節點 3，結果是 false
```

此時 `a` 已經是 `false`，表示左子樹不合法。

但如果寫的是這種版本：

```java
long prev = Long.MIN_VALUE;

public boolean isValidBST(TreeNode node){
    if (node == null) {
        return true;
    }

    boolean a = isValidBST(node.left);

    if (prev >= node.val) {
        return false;
    }
    prev = node.val;

    boolean b = isValidBST(node.right);

    return a && b;
}
```

那麼即使 `a == false`，程式還是會繼續：

1. 檢查 `6` 本身
2. 更新 `prev = 6`
3. 再去檢查 `6` 的右子樹 `7`

最後雖然結果仍然是 `false`，但是中間做了很多**其實沒必要做的判斷**。

### 8. 更好的做法：剪枝

既然左子樹一旦不合法，整棵樹就不可能合法了，那就沒必要再往下判斷了。

所以我們可以在左子樹驗證完後，立刻檢查它的返回值：

```java
long prev = Long.MIN_VALUE;

public boolean isValidBST(TreeNode node){
    if (node == null) {
        return true;
    }

    boolean a = isValidBST(node.left);
    if (!a) {
        return false;
    }

    if (prev >= node.val) {
        return false;
    }
    prev = node.val;

    boolean b = isValidBST(node.right);
    return b;
}
```

這樣一來，當左子樹已經返回 `false` 時，就會立刻結束，不再繼續驗證當前節點與右子樹。

這種做法就叫做：

> **剪枝**

#### 什麼是剪枝？

剪枝的意思是：

> 在遞迴或搜尋過程中，一旦發現某條路已經不可能得到正確答案，就立刻停止，不再浪費時間繼續往下走。

在這一題裡：

* 如果左子樹不合法
* 那整棵樹一定不合法
* 所以可以直接返回 `false`

這樣可以減少不必要的計算，提高效率。

### 9. 為什麼最後只需要 return b？

這一點很多人第一次看會疑惑。

程式是這樣：

```java
boolean a = isValidBST(node.left);
if (!a) {
    return false;
}

if (prev >= node.val) {
    return false;
}
prev = node.val;

boolean b = isValidBST(node.right);
return b;
```

為什麼不是 `return a && b`？

因為：

* `a` 在前面如果是 `false`，已經直接 `return false` 了
* 所以能走到後面的時候，代表 `a` 一定是 `true`
* 當前節點和 `prev` 的比較也已經通過
* 此時只剩下右子樹是否為合法 BST 需要判斷

所以最後直接：

```java
return b;
```

就可以了。

### 10. 最終程式碼

```java
long prev = Long.MIN_VALUE;

public boolean isValidBST(TreeNode node){
    if (node == null) {
        return true;
    }

    boolean a = isValidBST(node.left);
    if (!a) {
        return false;
    }

    if (prev >= node.val) {
        return false;
    }

    prev = node.val;

    return isValidBST(node.right);
}
```

## 六、為什麼不能把 prev 直接當一般參數傳值？

在使用**中序遍歷**驗證二叉搜索樹時，我們需要記錄「**上一個剛剛訪問過的節點值**」，通常會用一個變量 `prev` 來保存。

很多人會很自然地想到：  
既然不想把 `prev` 寫成全局變量，那我把它當作遞歸方法的參數傳進去不就好了嗎？

例如下面這樣：

```java
public boolean isValidBST(TreeNode node) {
    return doValid(node, Long.MIN_VALUE);
}

private boolean doValid(TreeNode node, long prev) {
    if (node == null) {
        return true;
    }

    boolean a = doValid(node.left, prev);
    if (!a) {
        return false;
    }

    if (prev >= node.val) {
        return false;
    }

    prev = node.val;
    return doValid(node.right, prev);
}
````

這段程式碼**看起來很合理**，但其實是錯的。

### 先說結論：問題出在 prev 是「值傳遞」

`long` 是基本型別，而 Java 的參數傳遞本質上都是**值傳遞**（pass by value）。

也就是說，當你呼叫：

```java
doValid(node.left, prev)
```

實際上是把**當前這個 `prev` 的值複製一份**，交給下一層遞歸方法使用。

因此：

* 子遞歸裡修改的 `prev`
* **不會反過來改到父遞歸那一層的 `prev`**

這就是整個問題的根源。

### 為什麼這會出錯？

因為我們要的 `prev`，不是某一條遞歸分支自己的局部值，
而是**整個中序遍歷過程共享的「上一個訪問節點」**。

換句話說，我們真正需要的是：

> 當左子樹遍歷完後，父節點必須知道「左子樹最後訪問到哪個值」。

但如果 `prev` 只是普通參數，那麼左子樹裡更新過的值，只會留在左子樹那層方法裡，**不會同步更新回父節點**。

所以父節點拿到的，仍然是舊值。

### 用一棵錯誤的樹來看

下面這棵樹不是合法 BST：

```text
         3
          \
           4
          /
         5
```

原因是：

* `5` 在 `4` 的左子樹
* 但 `5 > 4`
* 左子樹的值應該比父節點小，所以這棵樹不合法

它的中序遍歷結果是：

```text
3, 5, 4
```

這不是遞增序列，所以應該回傳 `false`。

#### 這段錯誤程式是怎麼一步一步出問題的？

下面我們來模擬執行流程。

##### 第 1 步：先處理根節點 3

呼叫：

```java
doValid(3, Long.MIN_VALUE)
```

此時：

```text
node = 3
prev = Long.MIN_VALUE
```

先遞歸處理左子樹：

```java
doValid(3.left, prev)
```

但 `3.left == null`，所以直接回傳 `true`。

接著判斷當前節點 `3`：

```java
if (prev >= 3)
```

也就是：

```java
if (Long.MIN_VALUE >= 3)
```

結果顯然是 `false`，所以 `3` 合法。

然後更新：

```java
prev = 3;
```

注意：
這裡只是把**目前這一層方法裡的局部變量 `prev`** 改成 `3`。

接著遞歸處理右子樹：

```java
doValid(4, 3)
```

##### 第 2 步：進入節點 4

現在來到：

```java
doValid(4, 3)
```

此時：

```text
node = 4
prev = 3
```

根據中序遍歷規則，要先處理左子樹，所以呼叫：

```java
doValid(5, 3)
```

##### 第 3 步：進入節點 5

現在來到：

```java
doValid(5, 3)
```

此時：

```text
node = 5
prev = 3
```

先處理左子樹：

```java
doValid(null, 3)
```

回傳 `true`。

接著判斷當前節點 `5`：

```java
if (prev >= 5)
```

也就是：

```java
if (3 >= 5)
```

結果是 `false`，所以 `5` 看起來合法。

然後執行：

```java
prev = 5;
```

注意這一點非常重要：

> 這裡更新的是「節點 5 這一層方法裡自己的 `prev`」
> 不是節點 `4` 那一層的 `prev`

然後處理 `5` 的右子樹：

```java
doValid(null, 5)
```

回傳 `true`。

所以整個 `doValid(5, 3)` 最後回傳 `true`。

##### 第 4 步：回到節點 4

這裡就是最容易出錯、也是最關鍵的地方。

很多人以為，既然在節點 `5` 那一層已經做過：

```java
prev = 5;
```

那回到節點 `4` 時，`prev` 應該也變成 `5`。

**但其實不會。**

因為節點 `4` 這一層的方法，原本拿到的 `prev` 是 `3`，
而節點 `5` 那一層只是拿到一份值為 `3` 的副本。

也就是說：

* `4` 那層的 `prev` 和 `5` 那層的 `prev`
* 雖然一開始數值都一樣
* 但它們是**兩個不同的局部變量**

所以當 `5` 那層做了：

```java
prev = 5;
```

只是把 **`5` 那層自己的局部變量** 改成了 `5`。

回到 `4` 這層時，`4` 這層的 `prev` 還是原本的 `3`。

因此接下來在 `4` 這層做判斷時，實際上比較的是：

```java
if (3 >= 4)
```

結果是 `false`。

於是程式誤以為 `4` 也是合法的，最後整棵樹就錯誤地回傳了 `true`。

### 問題的本質到底是什麼？

問題的本質可以濃縮成一句話：

> **中序遍歷需要一個「整個遍歷過程共享」的 `prev`，而不是每一層遞歸各自擁有一份 `prev` 副本。**

普通參數傳值時，每一層遞歸拿到的只是自己的局部副本，所以：

* 左子樹更新的 `prev`
* 無法傳回父節點
* 父節點就拿不到真正的「上一個訪問值」

這就破壞了中序遍歷判斷 BST 的核心邏輯。

### 為什麼一開始的「全局變量版」沒有這個問題？

例如下面這種寫法：

```java
long prev = Long.MIN_VALUE;

public boolean isValidBST(TreeNode node) {
    if (node == null) {
        return true;
    }

    boolean a = isValidBST(node.left);
    if (!a) {
        return false;
    }

    if (prev >= node.val) {
        return false;
    }

    prev = node.val;

    return isValidBST(node.right);
}
```

這裡的 `prev` 是**成員變量 / 全局共享變量**。

它的特點是：

* 所有遞歸方法操作的都是**同一個 `prev`**
* 任意一層把 `prev` 改掉
* 其他層看到的也都是修改後的值

所以當左子樹最後訪問到 `5` 並執行：

```java
prev = 5;
```

回到父節點 `4` 時，父節點讀到的 `prev` 也會是 `5`。

因此就能正確判斷：

```java
if (5 >= 4)
```

結果為 `true`，於是返回 `false`，成功發現這棵樹不合法。

### 那為什麼改成物件後就可以？

例如這樣：

```java
public boolean isValidBST(TreeNode node) {
    return doValid(node, new AtomicLong(Long.MIN_VALUE));
}

private boolean doValid(TreeNode node, AtomicLong prev) {
    if (node == null) {
        return true;
    }

    boolean a = doValid(node.left, prev);
    if (!a) {
        return false;
    }

    if (prev.get() >= node.val) {
        return false;
    }

    prev.set(node.val);
    return doValid(node.right, prev);
}
```

這裡要注意一件事：

Java 雖然也是**值傳遞**，
但如果傳的是**物件引用的值**，那麼每一層遞歸拿到的，都是**指向同一個物件的引用副本**。

這句話看起來有點繞，我們拆開來看：

#### 傳 long 時

傳的是數值本身，例如：

```java
3
```

每一層方法都會拿到一份自己的 `3`。

你在某一層改成 `5`，只會改到自己那份，不會影響別層。

#### 傳 AtomicLong 時

傳的是「物件引用的值」。

假設 `prev` 指向同一個 `AtomicLong` 物件：

```java
AtomicLong prev -> [value = 3]
```

雖然每次呼叫方法時，傳進去的引用本身也是複製一份，
但這幾份引用都**指向同一個物件**。

所以某一層做了：

```java
prev.set(5);
```

其實是在修改那個共享物件內部的值。

只要物件是同一個，那麼其他遞歸層之後看到的也會是 `5`。

## 七、另一種更直觀的做法：上下限法

我其實更推薦這一種，因為它更貼近 BST 的定義。

### 1. 為什麼只看父子關係不夠？

例如：

```text
         4
        / \
       2   6
          / \
         3   7
```

如果你只看父子：

* `4 < 6`
* `3 < 6`
* `6 < 7`

看起來都沒問題。

但 `3` 位在 `4` 的右子樹，所以它應該 **大於 4**，這才是關鍵。

因此：

> 每個節點不只受到父節點約束，還受到所有祖先節點約束。

### 2. 上下限的概念

對每個節點，都給它一個合法範圍：

* 根節點：`(-∞, +∞)`
* 左子節點：`(min, 父節點值)`
* 右子節點：`(父節點值, max)`

例如：

```text
         4
        / \
       2   6
          / \
         3   7
```

* `4` 的合法範圍：`(-∞, +∞)`
* `2` 的合法範圍：`(-∞, 4)`
* `6` 的合法範圍：`(4, +∞)`
* `3` 的合法範圍：`(4, 6)` ← `3` 不在這個範圍內，所以非法
* `7` 的合法範圍：`(6, +∞)`

### 3. 程式碼

```java
public boolean isValidBST(TreeNode node) {
    return doValid(node, Long.MIN_VALUE, Long.MAX_VALUE);
}

private boolean doValid(TreeNode node, long minValue, long maxValue) {
    if (node == null) {
        return true;
    }

    if (node.val <= minValue || node.val >= maxValue) {
        return false;
    }

    return doValid(node.left, minValue, node.val)
        && doValid(node.right, node.val, maxValue);
}
```

# LeetCode 938 - 二叉搜索樹的範圍和

> [938. 二叉搜索树的范围和](https://leetcode.cn/problems/range-sum-of-bst/)

## 一、題目核心觀念

這題的重點不是單純「遍歷整棵樹後加總」，而是要利用 **二叉搜索樹（BST）** 的性質來減少不必要的遍歷。

二叉搜索樹的性質是：

* 左子樹所有節點都比目前節點小
* 右子樹所有節點都比目前節點大

正因為有這個特性，我們可以在遍歷時做 **剪枝**，直接跳過不可能落在範圍內的子樹，提升效率。

## 二、先從中序遍歷的想法開始

### 1. 為什麼會想到中序遍歷？

因為 **BST 的中序遍歷結果是遞增排序**。

例如下面這棵樹：

```text
      10
     /  \
    5    15
   / \    \
  3   7    18
```

中序遍歷順序會是：

```text
3, 5, 7, 10, 15, 18
```

如果我們要找 `[7, 15]` 區間內的和，那麼中序遍歷可以讓我們依照由小到大的順序來檢查每個節點。

## 三、解法一：中序遍歷（非遞歸）

### 思路

使用堆疊模擬中序遍歷流程：

1. 一路往左走，並把沿途節點壓入堆疊
2. 左邊走到底後，彈出堆疊頂端節點
3. 處理該節點的值
4. 再轉向它的右子樹

只要節點值落在 `[low, high]` 範圍內，就把它加入總和。

### 基本寫法

```java
public int rangeSumBST1(TreeNode node, int low, int high) {
    TreeNode p = node;
    LinkedList<TreeNode> stack = new LinkedList<>();
    int sum = 0;

    while (p != null || !stack.isEmpty()) {
        if (p != null) {
            stack.push(p);
            p = p.left;
        } else {
            TreeNode pop = stack.pop();
            if (pop.val >= low && pop.val <= high) {
                sum += pop.val;
            }
            p = pop.right;
        }
    }
    return sum;
}
```

## 四、中序遍歷可以怎麼優化？

### 1. 為什麼能提早停止？

因為中序遍歷的結果是遞增的。

所以當遍歷到某個節點 `pop.val > high` 時，代表：

* 目前節點已經超過上限
* 後面還沒遍歷到的節點只會更大
* 因此後面不用再看了，可以直接 `break`

所以可以加上這段：

```java
if (pop.val > high) {
    break;
}
```

### 2. 為什麼 pop.val <= high 可以省略？

原本判斷是：

```java
if (pop.val >= low && pop.val <= high) {
    sum += pop.val;
}
```

但既然我們前面已經先判斷：

```java
if (pop.val > high) {
    break;
}
```

能走到加總這裡，就表示目前一定是 `pop.val <= high`。
因此只需要判斷它是否大於等於 `low` 即可。

可以簡化成：

```java
if (pop.val >= low) {
    sum += pop.val;
}
```

### 3. 優化後的完整程式碼

```java
public int rangeSumBST1(TreeNode node, int low, int high) {
    TreeNode p = node;
    LinkedList<TreeNode> stack = new LinkedList<>();
    int sum = 0;

    while (p != null || !stack.isEmpty()) {
        if (p != null) {
            stack.push(p);
            p = p.left;
        } else {
            TreeNode pop = stack.pop();

            if (pop.val > high) {
                break;
            }

            if (pop.val >= low) {
                sum += pop.val;
            }

            p = pop.right;
        }
    }
    return sum;
}
```

## 五、中序遍歷的問題在哪裡？

雖然中序遍歷可以利用「遞增」這件事，當遇到 `> high` 時提早停止，但它仍然有一個限制：

> 它對於小於 `low` 的節點，幾乎沒辦法提早跳過。

例如：

```text
      10
     /  \
    5    15
   / \    \
  3   7    18      low = 7, high = 15
```

中序遍歷順序是：

```text
3 -> 5 -> 7 -> 10 -> 15 -> 18
```

你會發現：

* `3` 明明不在範圍內
* `5` 也不在範圍內

但因為中序遍歷一定是從最小值開始走，所以這些節點還是得先遍歷到，沒辦法一開始就跳過。

因此，中序遍歷雖然可以做部分優化，但 **剪枝能力還不夠強**。

## 六、更好的做法：利用 BST 性質直接遞歸剪枝

這題更推薦的做法，是直接從根節點往下遞歸，並根據當前節點值決定要不要走左右子樹。

### 1. 剪枝規則

假設目前在節點 `node`：

#### 情況一： node.val < low

表示目前節點太小了。

因為 BST 的左子樹只會更小，所以：

* 左子樹全部不可能在範圍內
* 左子樹可以整棵跳過
* 只需要去右子樹找答案

即：

```java
return rangeSumBST(node.right, low, high);
```

#### 情況二： node.val > high

表示目前節點太大了。

因為 BST 的右子樹只會更大，所以：

* 右子樹全部不可能在範圍內
* 右子樹可以整棵跳過
* 只需要去左子樹找答案

即：

```java
return rangeSumBST(node.left, low, high);
```

#### 情況三：low <= node.val <= high

表示目前節點就在範圍內。

那就要：

1. 把目前節點值加進答案
2. 左右子樹都繼續遞歸

即：

```java
return node.val
    + rangeSumBST(node.left, low, high)
    + rangeSumBST(node.right, low, high);
```

### 遞歸寫法

```java
public int rangeSumBST(TreeNode node, int low, int high) {
    if (node == null) {
        return 0;
    }

    if (node.val < low) {
        return rangeSumBST(node.right, low, high);
    }

    if (node.val > high) {
        return rangeSumBST(node.left, low, high);
    }

    return node.val
            + rangeSumBST(node.left, low, high)
            + rangeSumBST(node.right, low, high);
}
```

### 為什麼這種寫法更高效？

因為它能做到真正的「整棵子樹剪掉」。

還是看這棵樹：

```text
      10
     /  \
    5    15
   / \    \
  3   7    18      low = 7, high = 15
```

執行流程大致如下：

1. `10` 在範圍內，加入總和
2. 往左到 `5`
3. `5 < 7`，表示 `5` 的左子樹全部都太小
4. 所以 `3` 根本不用看，直接去 `5` 的右子樹 `7`
5. `7` 在範圍內，加入總和
6. 回到右邊 `15`
7. `15` 在範圍內，加入總和
8. 再往右到 `18`
9. `18 > 15`，表示 `18` 的右子樹全部都太大，直接剪掉

最後得到：

```text
10 + 7 + 15 = 32
```

這就是 BST 剪枝帶來的效率提升。

### 完整範例

```java
public class Leetcode938 {

    public int rangeSumBST(TreeNode node, int low, int high) {
        if (node == null) {
            return 0;
        }

        if (node.val < low) {
            return rangeSumBST(node.right, low, high);
        }

        if (node.val > high) {
            return rangeSumBST(node.left, low, high);
        }

        return node.val
                + rangeSumBST(node.left, low, high)
                + rangeSumBST(node.right, low, high);
    }

    public static void main(String[] args) {
        /*
                 10
                /  \
               5    15
              / \    \
             3   7    18        low = 7, high = 15
         */
        TreeNode n1 = new TreeNode(5, new TreeNode(3), new TreeNode(7));
        TreeNode n2 = new TreeNode(15, null, new TreeNode(18));
        TreeNode root = new TreeNode(10, n1, n2);

        int sum = new Leetcode938().rangeSumBST(root, 7, 15);
        System.out.println(sum); // 32
    }
}
```

# LeetCode 1008：前序遍歷構造二叉搜尋樹

> [1008. 前序遍历构造二叉搜索树](https://leetcode.cn/problems/construct-binary-search-tree-from-preorder-traversal/description/)

題目會給你一個 **二叉搜尋樹（BST）的前序遍歷結果**，要你把原本的 BST 重建出來。原筆記主要整理了三種思路：

1. 逐個插入法
2. 上限法
3. 分治法

## 一、先理解題目核心

### 1. 什麼是前序遍歷？

前序遍歷順序是：

**根節點 → 左子樹 → 右子樹**

例如：

```text
8, 5, 1, 7, 10, 12
```

代表整棵樹的根節點一定是 `8`。
而因為它又是 BST，所以：

* 比 `8` 小的值會在左子樹
* 比 `8` 大的值會在右子樹

### 2. 什麼是 BST？

BST（Binary Search Tree，二叉搜尋樹）有這個性質：

* 左子樹所有節點都比根小
* 右子樹所有節點都比根大
* 左右子樹本身也都要是 BST

## 二、方法一：逐個插入法

這個方法最直觀。

想法很簡單：
既然這些數字本來就是某棵 BST 的前序遍歷結果，那我就按照順序，一個一個插入到一棵空 BST 裡。

例如輸入：

```text
8, 5, 1, 7, 10, 12
```

插入過程如下：

### 第 1 步：插入 8

```text
8
```

### 第 2 步：插入 5

`5 < 8`，所以放到 `8` 的左邊

```text
    8
   /
  5
```

### 第 3 步：插入 1

`1 < 8`，往左走
`1 < 5`，再往左走

```text
    8
   /
  5
 /
1
```

### 第 4 步：插入 7

`7 < 8`，往左
`7 > 5`，放到 `5` 的右邊

```text
    8
   /
  5
 / \
1   7
```

### 第 5 步：插入 10

`10 > 8`，放到右邊

```text
    8
   / \
  5   10
 / \
1   7
```

### 第 6 步：插入 12

`12 > 8`，往右
`12 > 10`，再往右

```text
    8
   / \
  5   10
 / \    \
1   7    12
```

### 方法一的程式核心

```java
public TreeNode bstFromPreorder(int[] preorder) {
    TreeNode root = new TreeNode(preorder[0]);
    for (int i = 1; i < preorder.length; i++) {
        insert(root, preorder[i]);
    }
    return root;
}

private TreeNode insert(TreeNode node, int val) {
    if (node == null) {
        return new TreeNode(val);
    }
    if (val < node.val) {
        node.left = insert(node.left, val);
    } else if (val > node.val) {
        node.right = insert(node.right, val);
    }
    return node;
}
```

### 方法一優缺點

**優點**

* 最容易理解
* 幾乎不用額外思考

**缺點**

* 每插入一個值都可能一路往下找位置
* 最差情況會退化成 `O(n²)`

## 三、方法二：上限法

### 1. 上限法的核心想法

前序遍歷是「根 → 左 → 右」，所以我們在遞迴時，會一直按照陣列順序往下讀。
但不是讀到一個值就一定建立節點，而是先檢查：

**這個值有沒有超過目前允許的上限 `max`？**

* 如果超過：代表它不屬於目前這個子樹，回傳 `null`
* 如果沒超過：才建立節點，然後遞迴建立它的左右子樹

### 2. 為什麼只需要「上限」？

因為前序遍歷的順序本身，已經隱含了下界資訊。

例如目前處理到 `5`：

* 它的左子樹值一定要 `< 5`
* 它的右子樹值一定要 `< 父節點給它的上限`

所以：

* 建左子樹時，上限是 `5`
* 建右子樹時，上限沿用父層傳下來的 `max`

### 3. 用例子理解

輸入：

```text
8, 5, 1, 7, 10, 12
```

#### 建立 8

`8` 沒超過初始上限 `+∞`，可以建立。
接著：

* `8` 左子樹上限 = `8`
* `8` 右子樹上限 = `+∞`

#### 建立 5

下一個值是 `5`，`5 <= 8`，可以作為 `8` 的左子樹。
接著：

* `5` 左子樹上限 = `5`
* `5` 右子樹上限 = `8`

#### 建立 1

下一個值是 `1`，`1 <= 5`，可以作為 `5` 的左子樹。
接著：

* `1` 左子樹上限 = `1`
* `1` 右子樹上限 = `5`

#### 看 7

此時要試著放到 `1` 的左邊：
`7 > 1`，超過上限，所以左子樹是 `null`

再試著放到 `1` 的右邊：
`7 > 5`，還是超過上限，所以右子樹也是 `null`

這表示 `1` 建完了。
之後回到 `5`，再看 `7` 是否能放到 `5` 的右子樹：
`7 <= 8`，可以。於是 `7` 成為 `5` 的右子節點。

這就是上限法最關鍵的地方：

> 一個值如果不符合目前子樹的上限，就先不要消耗它，直接回傳 `null`，讓上層去判斷它是不是屬於別的位置。

### 4. 上限法程式碼

```java
public TreeNode bstFromPreorder(int[] preorder) {
    i = 0;
    return build(preorder, Integer.MAX_VALUE);
}

int i = 0;

private TreeNode build(int[] preorder, int max) {
    if (i == preorder.length) {
        return null;
    }

    int val = preorder[i];
    if (val > max) {
        return null;
    }

    TreeNode node = new TreeNode(val);
    i++;

    node.left = build(preorder, val);
    node.right = build(preorder, max);

    return node;
}
```

### 5. 為什麼 i++ 一定要先做？

```java
i++;
node.left = insert(preorder, val);
node.right = insert(preorder, max);
```

不能寫成：

```java
node.left = insert(preorder, val);
node.right = insert(preorder, max);
i++;
```

原因是：

你在建立完當前節點後，就要立刻去看「下一個值」能不能作為它的左孩子或右孩子。
如果不先 `i++`，遞迴時還會一直拿到目前這個值，流程就錯了。

### 6. 方法二的優缺點

**優點**

* 只掃描陣列一次
* 時間複雜度可達 `O(n)`

**缺點**

* 不容易一眼想到
* 通常會用到全域索引 `i`，可讀性稍微差一點

## 四、方法三：分治法

### 1. 分治法的觀察

前序遍歷第一個值一定是根節點。

例如：

```text
8, 5, 1, 7, 10, 12
```

* `8` 是根
* 後面連續比 `8` 小的部分，是左子樹：`5, 1, 7`
* 剩下比 `8` 大的部分，是右子樹：`10, 12`

接著對左子樹、右子樹再做一樣的事情：

* `5` 是左子樹根

* `1` 是 `5` 的左子樹

* `7` 是 `5` 的右子樹

* `10` 是右子樹根

* 左子樹空

* `12` 是右子樹

### 2. 分治法程式碼

```java
public TreeNode bstFromPreorder(int[] preorder) {
    return partition(preorder, 0, preorder.length - 1);
}

private TreeNode partition(int[] preorder, int start, int end) {
    if (start > end) {
        return null;
    }

    TreeNode root = new TreeNode(preorder[start]);

    int index = start + 1;
    while (index <= end && preorder[index] < preorder[start]) {
        index++;
    }

    root.left = partition(preorder, start + 1, index - 1);
    root.right = partition(preorder, index, end);

    return root;
}
```

### 3. 方法三優缺點

**優點**

* 比上限法更直觀
* 容易從「前序遍歷的結構」去理解

**缺點**

* 每一層都可能要掃描一次區間找分界點
* 最差情況也是 `O(n²)`

# LeetCode 235 - 二叉搜尋樹的最近公共祖先

> [235. 二叉搜索树的最近公共祖先](https://leetcode.cn/problems/lowest-common-ancestor-of-a-binary-search-tree/description/)

## 一、什麼是最近公共祖先？

最近公共祖先指的是：

> 在一棵樹中，同時是 `p` 和 `q` 祖先的節點裡，**離它們最近的那一個**。

### 例子

```text
           6
        /     \ 
       2       8
      / \     / \
     0   4   7   9
        /  \
       3    5
```

### 例 1：找 2 和 8 的最近公共祖先

* `2` 的祖先有：`2、6`
* `8` 的祖先有：`8、6`

它們共同的祖先是 `6`，而且 `6` 是離它們最近的，所以答案是：

```text
6
```

### 例 2：找 4 和 5 的最近公共祖先

* `4` 的祖先有：`4、2、6`
* `5` 的祖先有：`5、4、2、6`

它們共同的祖先有 `4、2、6`，
其中離它們最近的是 `4`，所以答案是：

```text
4
```

這裡要特別注意：

> 一個節點也可以是它自己的祖先。
> 所以當 `p` 或 `q` 本身就是另一個節點的祖先時，答案有可能就是它自己。

## 二、這題的關鍵：利用 BST 的性質

二叉搜尋樹有一個很重要的特性：

* 左子樹的值都比目前節點小
* 右子樹的值都比目前節點大

因此，我們在判斷某個節點是不是 `p` 和 `q` 的最近公共祖先時，可以利用大小關係快速判斷。

## 三、最重要的結論

### 結論：

> 如果 `p` 和 `q` 分別位於某個節點的兩側，
> 那麼這個節點就是它們的最近公共祖先。

所謂「在兩側」的意思是：

* 一個在左邊
* 一個在右邊

或者：

* 其中一個剛好等於目前節點
* 另一個在左邊或右邊

## 四、怎麼判斷？

假設目前正在看某個節點 `a`：

### 情況 1：p 和 q 都比 a 小

代表它們都在 `a` 的左子樹中。
所以最近公共祖先一定也在左子樹，應該往左找。

### 情況 2：p 和 q 都比 a 大

代表它們都在 `a` 的右子樹中。
所以最近公共祖先一定也在右子樹，應該往右找。

### 情況 3：p 和 q 一個在左、一個在右

或者其中一個剛好等於 `a`

這表示目前這個節點 `a`，就是它們第一次「分開」的地方，
因此 `a` 就是最近公共祖先。

## 五、用題目的例子來理解

### 例 1：找 2 和 8

從根節點 `6` 開始看：

* `2 < 6`
* `8 > 6`

一個在左邊，一個在右邊，代表它們分別在 `6` 的兩側。

所以：

```text
6 就是最近公共祖先
```

---

## 例 2：找 4 和 5

從根節點 `6` 開始看：

* `4 < 6`
* `5 < 6`

兩個都在左邊，表示最近公共祖先不可能是 `6`，要往左找。

來到節點 `2`：

* `4 > 2`
* `5 > 2`

兩個都在右邊，表示最近公共祖先也不會是 `2`，繼續往右找。

來到節點 `4`：

* `4 == 4`
* `5 > 4`

這時候可以理解成：

* `4` 本身就在目前節點
* `5` 在 `4` 的右邊

也就是說，它們已經在 `4` 的兩側（或其中一個就是目前節點），
所以答案就是：

```text
4
```

## 六、解題思路總結

我們從根節點一路往下找：

1. 如果 `p` 和 `q` 都比目前節點小，就往左走
2. 如果 `p` 和 `q` 都比目前節點大，就往右走
3. 否則，代表它們已經分開，或其中一個就是目前節點
   此時目前節點就是最近公共祖先

## 七、程式碼

```java
public class E07Leetcode235 {
    public TreeNode lowestCommonAncestor(
         TreeNode root, 
         TreeNode p, 
         TreeNode q
    ) {
        TreeNode a = root;
        while (
            (p.val < a.val && q.val < a.val) || 
            (a.val < p.val && a.val < q.val)
        ) { // p 和 q 在同一側
            if (p.val < a.val) {
                a = a.left;
            } else {
                a = a.right;
            }
        }
        return a;
    }

    public static void main(String[] args) {
        TreeNode root = new TreeNode(6,
                new TreeNode(2, new TreeNode(0), new TreeNode(4, new TreeNode(3), new TreeNode(5))),
                new TreeNode(8, new TreeNode(7), new TreeNode(9)));

        TreeNode t1 = new E07Leetcode235().lowestCommonAncestor(root, new TreeNode(2), new TreeNode(8));
        System.out.println(t1.val); // 6

        TreeNode t2 = new E07Leetcode235().lowestCommonAncestor(root, new TreeNode(4), new TreeNode(5));
        System.out.println(t2.val); // 4
    }
}
```

## 八、程式碼解釋

### 1. a = root

先從根節點開始找。

```java
TreeNode a = root;
```

### 2. while 的條件：判斷 p 和 q 是否還在同一側

```java
while (
    (p.val < a.val && q.val < a.val) || 
    (a.val < p.val && a.val < q.val)
)
```

這段的意思是：

* `p` 和 `q` 都比 `a` 小 → 都在左邊
* `p` 和 `q` 都比 `a` 大 → 都在右邊

只要它們還在同一側，就表示還沒找到最近公共祖先，要繼續往下找。

### 3. 決定往左還是往右

```java
if (p.val < a.val) {
    a = a.left;
} else {
    a = a.right;
}
```

因為在 `while` 裡已經保證 `p` 和 `q` 在同一側了：

* 如果 `p.val < a.val`，代表它們都在左邊 → 往左走
* 否則代表它們都在右邊 → 往右走

### 4. 跳出迴圈後，a 就是答案

當迴圈結束時，表示出現以下情況之一：

* `p` 和 `q` 分別在 `a` 的左右兩側
* `p` 或 `q` 其中之一等於 `a`

此時 `a` 就是最近公共祖先。

```java
return a;
```
