# 堆的建堆：如何把一個普通數組調整成大頂堆

在學堆（Heap）時，有一個非常重要的方法叫做 **建堆（Build Heap）**。

假設現在有一個初始數組：

```text
[1, 2, 3, 4, 5, 6, 7]
```

把它畫成完全二元樹後會是：

```text
                1
              /   \
             2     3
            / \   / \
           4  5  6   7
```

但這棵樹**不是大頂堆**。

因為大頂堆要求：

> 任意一個父節點，都必須大於等於它的左右孩子。

例如這裡根節點 `1` 就比 `2`、`3` 小，所以不符合大頂堆的定義。

我們要做的事，就是把這個數組調整成一個合法的大頂堆，例如：

```text
[7, 5, 6, 4, 2, 1, 3]
```

對應的樹如下：

```text
                7
              /   \
             5     6
            / \   / \
           4  2  1   3
```

這個調整的過程，就叫做 **建堆**。

# 建堆的第一種方法：逐個插入（威廉姆斯建堆）

第一種思路很直觀：

1. 先準備一個空堆
2. 把元素一個一個插入堆中
3. 每插入一個元素，就讓它回到正確位置

因為「插入堆」這個操作本身就會維持大頂堆性質，所以最後整個堆自然就建好了。

這種方法通常稱為 **威廉姆斯建堆算法（Williams Build Heap）**。

## 1. 插入時發生什麼事？

以大頂堆為例，新元素插入到數組尾端後，要跟父節點比較：

* 如果比父節點小：位置正確，停止
* 如果比父節點大：和父節點交換，繼續往上比較

這個過程叫做 **上浮（heapify up / sift up）**。

## 2. 範例：依序插入 2、1、3

### 插入 2

```text
[2]
```

```text
2
```

---

### 插入 1

`1` 放到尾端後，和父節點 `2` 比較：

* `1 < 2`，不用動

```text
[2, 1]
```

```text
  2
 /
1
```

### 插入 3

先放到尾端：

```text
[2, 1, 3]
```

```text
   2
 /   \
1     3
```

此時 `3 > 2`，不符合大頂堆，所以 `3` 要和父節點 `2` 交換：

```text
[3, 1, 2]
```

```text
   3
 /   \
1     2
```

這樣就符合大頂堆了。

## 3. 威廉姆斯建堆的時間複雜度

假設有 `n` 個元素：

* 要插入 `n` 次
* 每次插入，最差要一路往上比較到根節點
* 堆的高度約為 `log n`

所以總時間複雜度為：

$$
O(n \log n)
$$

# 更快的建堆方法：佛洛依德建堆

除了逐個插入，還有一種更快的方法，叫做 **佛洛依德建堆算法（Floyd Build Heap）**。

它的核心想法是：

> 不從空堆開始插入，而是直接在原數組上，從後往前對每個非葉子節點做「下沉」。

這樣效率更高。

## 1. 什麼是下沉？

下沉（sift down / heapify down）的意思是：

* 把某個節點當成父節點
* 和它左右孩子中較大的那個比較
* 如果孩子比較大，就交換
* 交換後繼續往下，直到不需要交換為止

## 2. 佛洛依德建堆的兩個步驟

### 步驟一：找到最後一個非葉子節點

在數組中：

* 葉子節點：沒有孩子
* 非葉子節點：至少有一個孩子

對長度為 `n` 的數組，最後一個非葉子節點的索引是：

$$
\lfloor n/2 \rfloor - 1
$$

以數組 `[1,2,3,4,5,6,7]` 為例：

* `n = 7`
* 最後一個非葉子節點索引 = `7/2 - 1 = 2`
* 索引 `2` 的值就是 `3`

所以要從 `3` 開始往前處理。

### 步驟二：從後往前，依序做下沉

也就是依序處理：

* 索引 `2` → 值 `3`
* 索引 `1` → 值 `2`
* 索引 `0` → 值 `1`

## 佛洛依德建堆範例

原始數組：

```text
[1, 2, 3, 4, 5, 6, 7]
```

對應樹：

```text
                1
              /   \
             2     3
            / \   / \
           4  5  6   7
```

### 1. 先處理節點 3

`3` 的左右孩子是 `6` 和 `7`。
其中較大的是 `7`，而且 `7 > 3`，所以交換：

```text
[1, 2, 7, 4, 5, 6, 3]
```

```text
                1
              /   \
             2     7
            / \   / \
           4  5  6   3
```

交換後，`3` 已經沒有孩子了，停止。

### 2. 再處理節點 2

現在看值 `2`，它的左右孩子是 `4` 和 `5`。
其中較大的是 `5`，而且 `5 > 2`，所以交換：

```text
[1, 5, 7, 4, 2, 6, 3]
```

```text
                1
              /   \
             5     7
            / \   / \
           4  2  6   3
```

交換後，`2` 已經沒有孩子了，停止。

### 3. 最後處理根節點 1

現在看值 `1`，它的左右孩子是 `5` 和 `7`。
其中較大的是 `7`，而且 `7 > 1`，所以交換：

```text
[7, 5, 1, 4, 2, 6, 3]
```

```text
                7
              /   \
             5     1
            / \   / \
           4  2  6   3
```

但這時還沒結束，因為 `1` 被換到下面後，仍然有孩子。

繼續看 `1` 的左右孩子 `6` 和 `3`：

* 較大的是 `6`
* `6 > 1`

所以再交換一次：

```text
[7, 5, 6, 4, 2, 1, 3]
```

```text
                7
              /   \
             5     6
            / \   / \
           4  2  1   3
```

此時 `1` 沒有孩子了，停止。

建堆完成。

# 為什麼佛洛依德建堆比較快？

很多人看到「每個節點都可能下沉」，直覺會以為時間複雜度也是 `O(n log n)`。

但實際上不是。

原因在於：

> 不是每個節點都會下沉很多層。
> 越靠近底層的節點越多，但它們幾乎不用移動；
> 真正可能移動很多次的，只有少數靠近根的節點。

## 1. 用這個例子來看

```text
                1
              /   \
             2     3
            / \   / \
           4  5  6   7
```

如果用這裡的高度定義：

* 最底層 `4、5、6、7` 高度為 `1`
* `2、3` 高度為 `2`
* `1` 高度為 `3`

則：

| 節點所在層   | 節點數量 | 高度 | 最多交換次數 |
| ------- | ---- | -- | ------ |
| 4、5、6、7 | 4    | 1  | 0      |
| 2、3     | 2    | 2  | 1      |
| 1       | 1    | 3  | 2      |

所以最多交換總次數為：

$$
4 \times 0 + 2 \times 1 + 1 \times 2 = 4
$$

可以看出：

* 底層節點最多，但幾乎不動
* 上層節點雖然可能移動比較多次，但數量很少

## 2. 更一般化的觀察

若堆高度為 `h`，那麼第 `i` 層的節點數大約是：

$$
\frac{2^h}{2^i}
$$

而該層節點最多下沉：

$$
i - 1
$$

所以總工作量可寫成：

$$
\sum_{i=1}^{h}\left(\frac{2^h}{2^i}(i-1)\right)
$$

這個總和最終會落在：

$$
O(n)
$$

# MaxHeap（大頂堆）核心操作整理

```java
public class MaxHeap {
    int[] array;
    int size;

    public MaxHeap(int capacity) {
        this.array = new int[capacity];
    }

    public int peek() {
        return array[0];
    }

    public int poll() {
        int top = array[0];
        swap(0, size - 1);
        size--;
        down(0);
        return top;
    }

    public int poll(int index) {
        int deleted = array[index];
        swap(index, size - 1);
        size--;
        down(index);
        return deleted;
    }

    public void replace(int replaced) {
        array[0] = replaced;
        down(0);
    }

    public boolean offer(int offered) {
        if (size == array.length) {
            return false;
        }
        up(offered);
        size++;
        return true;
    }

    private void up(int offered) {
        int child = size;
        while (child > 0) {
            int parent = (child - 1) / 2;
            if (offered > array[parent]) {
                array[child] = array[parent];
            } else {
                break;
            }
            child = parent;
        }
        array[child] = offered;
    }

    public MaxHeap(int[] array) {
        this.array = array;
        this.size = array.length;
        heapify();
    }

    private void heapify() {
        for (int i = size / 2 - 1; i >= 0; i--) {
            down(i);
        }
    }

    private void down(int parent) {
        int left = parent * 2 + 1;
        int right = left + 1;
        int max = parent;
        if (left < size && array[left] > array[max]) {
            max = left;
        }
        if (right < size && array[right] > array[max]) {
            max = right;
        }
        if (max != parent) {
            swap(max, parent);
            down(max);
        }
    }

    private void swap(int i, int j) {
        int t = array[i];
        array[i] = array[j];
        array[j] = t;
    }

    /**
     * 堆排序（升序）
     * 排序后，array 不再保持堆结构，而是变成有序数组
     */
    public void heapSort() {
        // 先确保当前数组是大顶堆
        heapify();

        // 从最后一个元素开始，把堆顶交换到尾部
        for (int end = size - 1; end > 0; end--) {
            swap(0, end);      // 当前最大值放到尾部
            down(0, end);      // end 之前仍然视为堆，重新下潜调整
        }
    }
}
```

## 什麼是大頂堆？

大頂堆（Max Heap）的特性是：

> 每個父節點的值，都要大於等於它的左右孩子。

例如下面這個陣列：

```text
[7, 5, 6, 4, 2, 1, 3]
```

對應的樹為：

```text
                7
              /   \
             5     6
            / \   / \
           4  2  1   3
```

這就是一個合法的大頂堆，因為：

* `7 > 5, 6`
* `5 > 4, 2`
* `6 > 1, 3`

## 獲取堆頂元素 peek()

```java
public int peek() {
    return array[0];
}
```

在堆中，**堆頂元素一定存放在索引 `0`**。

所以要取得最大值，只要直接回傳 `array[0]` 即可。

例如：

```text
[7, 5, 6, 4, 2, 1, 3]
```

堆頂就是：

```text
7
```

## 刪除堆頂元素 poll()

```java
public int poll() {
    int top = array[0];
    swap(0, size - 1);
    size--;
    down(0);
    return top;
}
```

刪除堆頂是堆中非常重要的操作。
整個流程可以分成三步：

### 第 1 步：先把堆頂和最後一個元素交換

原本的堆：

```text
[7, 5, 6, 4, 2, 1, 3]
```

```text
                7
              /   \
             5     6
            / \   / \
           4  2  1   3
```

先交換堆頂和最後一個元素：

```text
[3, 5, 6, 4, 2, 1, 7]
```

這樣做的原因是：

> 陣列尾端的元素最容易刪除。
> 如果直接刪除開頭元素，後面的元素都要往前搬移，成本很高。

### 第 2 步：刪除最後一個元素

交換後，把最後一個元素移除，也就是把 `size` 減 1：

```text
[3, 5, 6, 4, 2, 1]
```

```text
                3
              /   \
             5     6
            / \   /
           4  2  1
```

這時 `7` 已經被刪掉了。

### 第 3 步：讓新堆頂往下潛（down）

現在新的堆頂是 `3`，但它不一定符合大頂堆特性。

它的左右孩子是 `5` 和 `6`，其中較大的是 `6`，而 `6 > 3`，所以要交換：

```text
[6, 5, 3, 4, 2, 1]
```

```text
                6
              /   \
             5     3
            / \   /
           4  2  1
```

接著再看 `3` 是否還需要繼續下潛。

`3` 的孩子是 `1`，而 `1 < 3`，所以不需要再交換，調整完成。

### 刪除堆頂的核心思路

可以總結成一句話：

> **先把堆頂換到最後刪掉，再讓交換上來的元素往下潛。**

## 刪除指定索引元素 poll(int index)

```java
public int poll(int index) {
    int deleted = array[index];
    swap(index, size - 1);
    size--;
    down(index);
    return deleted;
}
```

刪除指定位置的元素，思路和刪除堆頂其實很像。

### 範例：刪除索引 1 的元素 5

原本的堆：

```text
[7, 5, 6, 4, 2, 1, 3]
```

```text
                7
              /   \
             5     6
            / \   / \
           4  2  1   3
```

#### 第 1 步：和最後一個元素交換

把索引 `1` 的 `5` 和最後一個元素 `3` 交換：

```text
[7, 3, 6, 4, 2, 1, 5]
```

```text
                7
              /   \
             3     6
            / \   / \
           4  2  1   5
```

#### 第 2 步：刪除最後一個元素

把最後一個元素移除：

```text
[7, 3, 6, 4, 2, 1]
```

```text
                7
              /   \
             3     6
            / \   /
           4  2  1
```

#### 第 3 步：對交換上來的元素做調整

現在索引 `1` 的元素是 `3`，它可能破壞堆結構。

`3` 的左右孩子是 `4` 和 `2`，其中較大的是 `4`，而 `4 > 3`，所以交換：

```text
[7, 4, 6, 3, 2, 1]
```

```text
                7
              /   \
             4     6
            / \   /
           3  2  1
```

此時 `3` 已經沒有孩子了，因此調整完成。

### 這個操作的本質

刪除指定索引元素，本質上也是：

> **先把要刪除的元素換到尾部刪掉，再對交換上來的元素重新調整。**

## 替換堆頂元素 replace(int replaced)

```java
public void replace(int replaced) {
    array[0] = replaced;
    down(0);
}
```

這個方法的意思是：

> 直接用新值覆蓋堆頂，然後讓它往下潛，恢復大頂堆。

### 範例：把堆頂 7 替換成 0

原本：

```text
[7, 5, 6, 4, 2, 1, 3]
```

```text
                7
              /   \
             5     6
            / \   / \
           4  2  1   3
```

直接替換：

```text
[0, 5, 6, 4, 2, 1, 3]
```

```text
                0
              /   \
             5     6
            / \   / \
           4  2  1   3
```

此時明顯不符合大頂堆。

#### 開始下潛

`0` 的左右孩子是 `5` 和 `6`，其中較大的是 `6`，交換：

```text
[6, 5, 0, 4, 2, 1, 3]
```

```text
                6
              /   \
             5     0
            / \   / \
           4  2  1   3
```

繼續看 `0`：

它的左右孩子是 `1` 和 `3`，其中較大的是 `3`，交換：

```text
[6, 5, 3, 4, 2, 1, 0]
```

```text
                6
              /   \
             5     3
            / \   / \
           4  2  1   0
```

此時 `0` 已經沒有孩子，調整完成。

### 替換堆頂的核心思路

> **直接改掉堆頂，然後往下潛。**

這比「先刪除再插入」更直接。

## 插入新元素 offer(int offered)

```java
public boolean offer(int offered) {
    if (size == array.length) {
        return false;
    }
    up(offered);
    size++;
    return true;
}
```

新增元素時，會把元素放到堆尾，然後透過 `up()` 讓它**往上浮**。

### up() 的作用

```java
private void up(int offered) {
    int child = size;
    while (child > 0) {
        int parent = (child - 1) / 2;
        if (offered > array[parent]) {
            array[child] = array[parent];
        } else {
            break;
        }
        child = parent;
    }
    array[child] = offered;
}
```

這段程式的邏輯是：

1. 新元素先視為放在尾部
2. 不斷和父節點比較
3. 如果比父節點大，就把父節點往下搬
4. 一直往上，直到遇到更大的父節點或到達根節點
5. 最後把新元素放到正確位置

這種寫法不是「一路 swap」，而是採用**搬移父節點，最後一次放入新值**，效率會更好一些。

### 範例：插入 8

原本的堆：

```text
[7, 5, 6, 4, 2, 1, 3]
```

如果要新增 `8`，先把它放到尾部：

```text
[7, 5, 6, 4, 2, 1, 3, 8]
```

```text
                7
              /   \
             5     6
            / \   / \
           4   2 1   3
          /
         8
```

#### 第 1 次比較

`8` 的父節點索引為：

$$
\left\lfloor \frac{i - 1}{2} \right\rfloor
$$

目前 `8` 在索引 `7`，所以父節點索引是：

$$
(7 - 1) / 2 = 3
$$

也就是值 `4`。

因為 `8 > 4`，所以 `8` 往上浮：

```text
[7, 5, 6, 8, 2, 1, 3, 4]
```

```text
                7
              /   \
             5     6
            / \   / \
           8   2 1   3
          /
         4
```

#### 第 2 次比較

現在 `8` 到了索引 `3`，父節點是索引 `1`，值為 `5`。

因為 `8 > 5`，繼續往上浮：

```text
[7, 8, 6, 5, 2, 1, 3, 4]
```

```text
                7
              /   \
             8     6
            / \   / \
           5   2 1   3
          /
         4
```

#### 第 3 次比較

現在 `8` 到了索引 `1`，父節點是索引 `0`，值為 `7`。

因為 `8 > 7`，繼續往上浮：

```text
[8, 7, 6, 5, 2, 1, 3, 4]
```

```text
                8
              /   \
             7     6
            / \   / \
           5   2 1   3
          /
         4
```

此時 `8` 已經到達堆頂，上浮完成。

### 插入元素的核心思路

> **先放到尾部，再一路和父節點比較，必要時往上浮。**

## 建堆 heapify()

```java
public MaxHeap(int[] array) {
    this.array = array;
    this.size = array.length;
    heapify();
}
```

如果一開始就給你一個陣列，那就可以直接建堆。

```java
private void heapify() {
    for (int i = size / 2 - 1; i >= 0; i--) {
        down(i);
    }
}
```

### 為什麼從 size / 2 - 1 開始？

因為：

> `size / 2 - 1` 是最後一個非葉子節點的索引。

葉子節點本身不需要下潛，因為它沒有孩子。

所以建堆時，只需要從最後一個非葉子節點開始，往前依次做 `down()` 即可。

### down() 的作用

```java
private void down(int parent) {
    int left = parent * 2 + 1;
    int right = left + 1;
    int max = parent;

    if (left < size && array[left] > array[max]) {
        max = left;
    }
    if (right < size && array[right] > array[max]) {
        max = right;
    }
    if (max != parent) {
        swap(max, parent);
        down(max);
    }
}
```

這個方法的邏輯是：

1. 找出父節點、左孩子、右孩子三者中最大的值
2. 如果最大的不是父節點，表示堆性質被破壞
3. 那就把父節點和較大的孩子交換
4. 再繼續對新的位置遞迴下潛

## 堆的排序 heapSort()

### 堆排序的核心思路

堆排序的流程可以分成 3 步：

1. 先把未排序數組透過 `heapify()` 建成**大頂堆**
2. 將**堆頂元素**與**堆底元素**交換
   這樣最大值就被放到數組最後面
3. 把堆的有效範圍縮小 1，然後對新的堆頂做**下潛調整**
4. 重複以上步驟，直到堆中只剩一個元素

### 為什麼可以排序？

因為在大頂堆中：

> 堆頂永遠是目前最大的元素

所以每做一次：

* 把堆頂和最後一個元素交換
* 再把堆的範圍縮小

就等於把「目前最大值」放到了正確的最終位置。

如此反覆執行，最後整個數組就會變成**由小到大排列**。

### 堆排序範例

假設有一個未排序數組：

```text
[2, 3, 1, 7, 6, 4, 5]
```

#### 第 1 步：先建立大頂堆

對數組執行 `heapify()` 之後，可得到：

```text
[7, 6, 5, 3, 2, 4, 1]
```

```text
                7
              /   \
             6     5
            / \   / \
           3  2  4   1
```

#### 第 2 步：把堆頂和堆底交換

把 `7` 和最後一個元素 `1` 交換：

```text
[1, 6, 5, 3, 2, 4, 7]
```

```text
                1
              /   \
             6     5
            / \   / \
           3  2  4   7
```

此時最後面的 `7` 已經排好位置了，不再屬於堆的一部分。

#### 第 3 步：縮小堆範圍，重新下潛調整

現在只看前 6 個元素：

```text
[1, 6, 5, 3, 2, 4]
```

讓 `1` 下潛後，調整成新的大頂堆：

```text
[6, 3, 5, 1, 2, 4, 7]
```

```text
                6
              /   \
             3     5
            / \   /
           1  2  4
```

##### 再做一次：把 6 交換到尾部

交換堆頂和目前堆底：

```text
[4, 3, 5, 1, 2, 6, 7]
```

此時 `6` 已經排好。

重新調整前 5 個元素：

```text
[5, 3, 4, 1, 2, 6, 7]
```

```text
                5
              /   \
             3     4
            / \
           1   2
```

##### 再把 5 交換到尾部

交換後：

```text
[2, 3, 4, 1, 5, 6, 7]
```

調整前 4 個元素：

```text
[4, 3, 2, 1, 5, 6, 7]
```

```text
                4
              /   \
             3     2
            /
           1
```

##### 再把 4 交換到尾部

交換後：

```text
[1, 3, 2, 4, 5, 6, 7]
```

調整前 3 個元素：

```text
[3, 1, 2, 4, 5, 6, 7]
```

```text
                3
              /   \
             1     2
```

##### 再把 3 交換到尾部

交換後：

```text
[2, 1, 3, 4, 5, 6, 7]
```

調整前 2 個元素：

```text
[2, 1, 3, 4, 5, 6, 7]
```

```text
                2
              /
             1
```

##### 最後把 2 交換到尾部

交換後：

```text
[1, 2, 3, 4, 5, 6, 7]
```

排序完成。

### 堆排序過程總結

整個過程其實就是一直重複這兩件事：

1. 把堆頂最大值交換到目前堆的最後面
2. 讓新的堆頂往下潛，恢復大頂堆

所以你可以把它理解成：

> **每一輪都把當前最大值「選出來」，放到數組尾端。**

### heapSort() 方法是怎麼工作的？

這段方法：

```java
public void heapSort() {
    heapify();

    for (int end = size - 1; end > 0; end--) {
        swap(0, end);
        down(0, end);
    }
}
```

可以拆成兩部分理解。

#### 1. heapify();

先把整個數組整理成大頂堆。

例如：

```text
[2, 3, 1, 7, 6, 4, 5]
```

建堆後變成：

```text
[7, 6, 5, 3, 2, 4, 1]
```

#### 2. for (int end = size - 1; end > 0; end--)

這段迴圈表示：

* `end` 是目前堆的最後位置
* 每一輪都把 `array[0]`（最大值）換到 `end`
* 然後只針對 `0 ~ end - 1` 這一段重新維護堆

例如第一輪：

* 交換 `0` 和 `6`
* 最大值 `7` 就固定在最後面
* 然後只調整前 6 個元素

第二輪：

* 再把目前堆頂 `6` 交換到索引 `5`
* 然後只調整前 5 個元素

如此重複，直到排序完成。

# LeetCode 215 - 數組中的第 K 個最大元素

> [215. 数组中的第K个最大元素](https://leetcode.cn/problems/kth-largest-element-in-an-array/description/)

這題可以用 **小頂堆（Min Heap）** 來解。

## 一、核心想法

我們的目標不是把整個數組都排序，而是只要找出 **第 K 大** 的元素。

因此可以維護一個 **大小固定為 `k` 的小頂堆**：

* 堆裡永遠保存「目前看過的元素中，最大的 `k` 個」
* 而這 `k` 個元素裡面，**最小的那個** 就放在堆頂
* 所以最後堆頂元素，就是 **第 K 大元素**

## 二、解題思路

### 步驟 1：先把前 `k` 個元素放進小頂堆

因為我們要維護「前 `k` 大元素」，一開始先拿前 `k` 個元素建立一個小頂堆。

### 步驟 2：從第 `k` 個元素開始，逐一比較

對於後面的每個元素 `num`：

* 如果 `num <= 堆頂元素`

   * 代表它不可能進入「前 k 大」
   * 直接略過

* 如果 `num > 堆頂元素`

   * 代表它比目前前 `k` 大裡最小的那個還大
   * 所以它應該加入前 `k` 大
   * 這時就用它 **替換堆頂元素**，再重新調整堆

### 步驟 3：遍歷完成後

小頂堆中留下的，就是整個數組中 **最大的 `k` 個元素**

而堆頂元素剛好是這 `k` 個元素中最小的，也就是：**第 K 大元素**

## 三、為什麼要用小頂堆？

很多人一開始會想：

「找第 K 大，不是應該用大頂堆嗎？」

其實這題更適合用 **小頂堆**，原因是：

* 我們只需要保留「最大的 `k` 個」
* 這時最重要的是，能快速知道這 `k` 個裡面 **最小的是誰**
* 因為只要新元素比這個最小值還大，就可以把它擠掉

而小頂堆的堆頂，剛好就是目前這 `k` 個元素中的最小值，非常適合這題。

## 四、例子說明

題目數組：

```text
[3, 2, 1, 5, 6, 4]
```

要找第 `2` 大元素，也就是 `k = 2`

### 第一步：先放前 2 個元素進小頂堆

先加入：

```text
[3, 2]
```

建立小頂堆後：

```text
[2, 3]
```

堆的結構如下：

```text
    2
   /
  3
```

這表示：

* 目前看過的元素中，前 2 大是 `2` 和 `3`
* 其中最小的是 `2`，所以放在堆頂

### 第二步：處理後續元素

#### 處理元素 1

`1 < 2`（堆頂）

表示 `1` 不可能進入前 2 大，所以略過。

小頂堆不變：

```text
[2, 3]
```

#### 處理元素 5

`5 > 2`（堆頂）

表示 `5` 應該進入前 2 大，因此用 `5` 替換堆頂 `2`：

```text
[5, 3]
```

此時還不是合法的小頂堆，因為堆頂 `5` 比子節點 `3` 大，所以需要下沉調整：

```text
[3, 5]
```

堆結構變成：

```text
    3
   /
  5
```

#### 處理元素 6

`6 > 3`（堆頂）

所以用 `6` 替換堆頂：

```text
[6, 5]
```

重新調整後：

```text
[5, 6]
```

堆結構：

```text
    5
   /
  6
```

#### 處理元素 4

`4 < 5`（堆頂）

代表 `4` 不夠大，無法進入前 2 大，直接略過。

小頂堆維持不變：

```text
[5, 6]
```

## 五、最後結果

遍歷完成後，小頂堆中保留的是：

```text
[5, 6]
```

也就是整個數組中最大的 2 個元素。

其中堆頂元素是：

```text
5
```

所以答案就是：**第 2 大元素 = 5**

## 六、流程總結

整個過程可以簡單記成：

1. 先把前 `k` 個元素放進小頂堆
2. 之後每遇到一個新元素：

   * 比堆頂小或相等 → 略過
   * 比堆頂大 → 替換堆頂並調整堆
3. 最後堆頂就是第 `k` 大元素

## 七、程式碼

### MinHeap

```java
public class MinHeap {
    int[] array;
    int size;

    public MinHeap(int capacity) {
        this.array = new int[capacity];
    }

    public boolean isFull() {
        return size == array.length;
    }

    /**
     * 取得堆頂元素
     */
    public int peek() {
        return array[0];
    }

    /**
     * 刪除堆頂元素
     */
    public int poll() {
        int top = array[0];
        swap(0, size - 1);
        size--;
        down(0);
        return top;
    }

    /**
     * 刪除指定索引位置的元素
     */
    public int poll(int index) {
        int deleted = array[index];
        swap(index, size - 1);
        size--;
        down(index);
        return deleted;
    }

    /**
     * 用新元素替換堆頂元素
     */
    public void replace(int replaced) {
        array[0] = replaced;
        down(0);
    }

    /**
     * 新增元素到堆中
     */
    public boolean offer(int offered) {
        if (size == array.length) {
            return false;
        }
        up(offered);
        size++;
        return true;
    }

    /**
     * 上浮：讓新元素找到正確位置
     */
    private void up(int offered) {
        int child = size;
        while (child > 0) {
            int parent = (child - 1) / 2;
            if (offered < array[parent]) {
                array[child] = array[parent];
            } else {
                break;
            }
            child = parent;
        }
        array[child] = offered;
    }

    public MinHeap(int[] array) {
        this.array = array;
        this.size = array.length;
        heapify();
    }

    /**
     * 建堆
     */
    private void heapify() {
        for (int i = size / 2 - 1; i >= 0; i--) {
            down(i);
        }
    }

    /**
     * 下沉：讓某個節點恢復小頂堆性質
     */
    private void down(int parent) {
        int left = parent * 2 + 1;
        int right = left + 1;
        int min = parent;

        if (left < size && array[left] < array[min]) {
            min = left;
        }
        if (right < size && array[right] < array[min]) {
            min = right;
        }

        if (min != parent) {
            swap(min, parent);
            down(min);
        }
    }

    /**
     * 交換兩個索引位置的元素
     */
    private void swap(int i, int j) {
        int t = array[i];
        array[i] = array[j];
        array[j] = t;
    }
}
```

### 解題主程式

```java
public class E02Leetcode215 {

    public int findKthLargest(int[] numbers, int k) {
        MinHeap heap = new MinHeap(k);

        // 先放入前 k 個元素
        for (int i = 0; i < k; i++) {
            heap.offer(numbers[i]);
        }

        // 從第 k 個元素開始比較
        for (int i = k; i < numbers.length; i++) {
            if (numbers[i] > heap.peek()) {
                heap.replace(numbers[i]);
            }
        }

        // 堆頂就是第 k 大元素
        return heap.peek();
    }

    public static void main(String[] args) {
        // 應為 5
        System.out.println(new E02Leetcode215().findKthLargest(new int[]{3, 2, 1, 5, 6, 4}, 2));
        // 應為 4
        System.out.println(new E02Leetcode215().findKthLargest(new int[]{3, 2, 3, 1, 2, 4, 5, 5, 6}, 4));
    }
}
```
