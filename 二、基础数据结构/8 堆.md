# 堆的建堆：如何把一個普通數組調整成大頂堆

在學堆（Heap）時，有一個非常重要的方法叫做 **建堆（Build Heap）**。

假設現在有一個初始數組：

```text
[1, 2, 3, 4, 5, 6, 7]
```

把它畫成完全二元樹後會是：

```text
                1
              /   \
             2     3
            / \   / \
           4  5  6   7
```

但這棵樹**不是大頂堆**。

因為大頂堆要求：

> 任意一個父節點，都必須大於等於它的左右孩子。

例如這裡根節點 `1` 就比 `2`、`3` 小，所以不符合大頂堆的定義。

我們要做的事，就是把這個數組調整成一個合法的大頂堆，例如：

```text
[7, 5, 6, 4, 2, 1, 3]
```

對應的樹如下：

```text
                7
              /   \
             5     6
            / \   / \
           4  2  1   3
```

這個調整的過程，就叫做 **建堆**。

# 建堆的第一種方法：逐個插入（威廉姆斯建堆）

第一種思路很直觀：

1. 先準備一個空堆
2. 把元素一個一個插入堆中
3. 每插入一個元素，就讓它回到正確位置

因為「插入堆」這個操作本身就會維持大頂堆性質，所以最後整個堆自然就建好了。

這種方法通常稱為 **威廉姆斯建堆算法（Williams Build Heap）**。

## 1. 插入時發生什麼事？

以大頂堆為例，新元素插入到數組尾端後，要跟父節點比較：

* 如果比父節點小：位置正確，停止
* 如果比父節點大：和父節點交換，繼續往上比較

這個過程叫做 **上浮（heapify up / sift up）**。

## 2. 範例：依序插入 2、1、3

### 插入 2

```text
[2]
```

```text
2
```

---

### 插入 1

`1` 放到尾端後，和父節點 `2` 比較：

* `1 < 2`，不用動

```text
[2, 1]
```

```text
  2
 /
1
```

### 插入 3

先放到尾端：

```text
[2, 1, 3]
```

```text
   2
 /   \
1     3
```

此時 `3 > 2`，不符合大頂堆，所以 `3` 要和父節點 `2` 交換：

```text
[3, 1, 2]
```

```text
   3
 /   \
1     2
```

這樣就符合大頂堆了。

## 3. 威廉姆斯建堆的時間複雜度

假設有 `n` 個元素：

* 要插入 `n` 次
* 每次插入，最差要一路往上比較到根節點
* 堆的高度約為 `log n`

所以總時間複雜度為：

$$
O(n \log n)
$$

# 更快的建堆方法：佛洛依德建堆

除了逐個插入，還有一種更快的方法，叫做 **佛洛依德建堆算法（Floyd Build Heap）**。

它的核心想法是：

> 不從空堆開始插入，而是直接在原數組上，從後往前對每個非葉子節點做「下沉」。

這樣效率更高。

## 1. 什麼是下沉？

下沉（sift down / heapify down）的意思是：

* 把某個節點當成父節點
* 和它左右孩子中較大的那個比較
* 如果孩子比較大，就交換
* 交換後繼續往下，直到不需要交換為止

## 2. 佛洛依德建堆的兩個步驟

### 步驟一：找到最後一個非葉子節點

在數組中：

* 葉子節點：沒有孩子
* 非葉子節點：至少有一個孩子

對長度為 `n` 的數組，最後一個非葉子節點的索引是：

$$
\lfloor n/2 \rfloor - 1
$$

以數組 `[1,2,3,4,5,6,7]` 為例：

* `n = 7`
* 最後一個非葉子節點索引 = `7/2 - 1 = 2`
* 索引 `2` 的值就是 `3`

所以要從 `3` 開始往前處理。

### 步驟二：從後往前，依序做下沉

也就是依序處理：

* 索引 `2` → 值 `3`
* 索引 `1` → 值 `2`
* 索引 `0` → 值 `1`

## 佛洛依德建堆範例

原始數組：

```text
[1, 2, 3, 4, 5, 6, 7]
```

對應樹：

```text
                1
              /   \
             2     3
            / \   / \
           4  5  6   7
```

### 1. 先處理節點 3

`3` 的左右孩子是 `6` 和 `7`。
其中較大的是 `7`，而且 `7 > 3`，所以交換：

```text
[1, 2, 7, 4, 5, 6, 3]
```

```text
                1
              /   \
             2     7
            / \   / \
           4  5  6   3
```

交換後，`3` 已經沒有孩子了，停止。

### 2. 再處理節點 2

現在看值 `2`，它的左右孩子是 `4` 和 `5`。
其中較大的是 `5`，而且 `5 > 2`，所以交換：

```text
[1, 5, 7, 4, 2, 6, 3]
```

```text
                1
              /   \
             5     7
            / \   / \
           4  2  6   3
```

交換後，`2` 已經沒有孩子了，停止。

### 3. 最後處理根節點 1

現在看值 `1`，它的左右孩子是 `5` 和 `7`。
其中較大的是 `7`，而且 `7 > 1`，所以交換：

```text
[7, 5, 1, 4, 2, 6, 3]
```

```text
                7
              /   \
             5     1
            / \   / \
           4  2  6   3
```

但這時還沒結束，因為 `1` 被換到下面後，仍然有孩子。

繼續看 `1` 的左右孩子 `6` 和 `3`：

* 較大的是 `6`
* `6 > 1`

所以再交換一次：

```text
[7, 5, 6, 4, 2, 1, 3]
```

```text
                7
              /   \
             5     6
            / \   / \
           4  2  1   3
```

此時 `1` 沒有孩子了，停止。

建堆完成。

# 為什麼佛洛依德建堆比較快？

很多人看到「每個節點都可能下沉」，直覺會以為時間複雜度也是 `O(n log n)`。

但實際上不是。

原因在於：

> 不是每個節點都會下沉很多層。
> 越靠近底層的節點越多，但它們幾乎不用移動；
> 真正可能移動很多次的，只有少數靠近根的節點。

## 1. 用這個例子來看

```text
                1
              /   \
             2     3
            / \   / \
           4  5  6   7
```

如果用這裡的高度定義：

* 最底層 `4、5、6、7` 高度為 `1`
* `2、3` 高度為 `2`
* `1` 高度為 `3`

則：

| 節點所在層   | 節點數量 | 高度 | 最多交換次數 |
| ------- | ---- | -- | ------ |
| 4、5、6、7 | 4    | 1  | 0      |
| 2、3     | 2    | 2  | 1      |
| 1       | 1    | 3  | 2      |

所以最多交換總次數為：

$$
4 \times 0 + 2 \times 1 + 1 \times 2 = 4
$$

可以看出：

* 底層節點最多，但幾乎不動
* 上層節點雖然可能移動比較多次，但數量很少

## 2. 更一般化的觀察

若堆高度為 `h`，那麼第 `i` 層的節點數大約是：

$$
\frac{2^h}{2^i}
$$

而該層節點最多下沉：

$$
i - 1
$$

所以總工作量可寫成：

$$
\sum_{i=1}^{h}\left(\frac{2^h}{2^i}(i-1)\right)
$$

這個總和最終會落在：

$$
O(n)
$$

# MaxHeap（大頂堆）核心操作整理

```java
public class MaxHeap {
    int[] array;
    int size;

    public MaxHeap(int capacity) {
        this.array = new int[capacity];
    }

    public int peek() {
        return array[0];
    }

    public int poll() {
        int top = array[0];
        swap(0, size - 1);
        size--;
        down(0);
        return top;
    }

    public int poll(int index) {
        int deleted = array[index];
        swap(index, size - 1);
        size--;
        down(index);
        return deleted;
    }

    public void replace(int replaced) {
        array[0] = replaced;
        down(0);
    }

    public boolean offer(int offered) {
        if (size == array.length) {
            return false;
        }
        up(offered);
        size++;
        return true;
    }

    private void up(int offered) {
        int child = size;
        while (child > 0) {
            int parent = (child - 1) / 2;
            if (offered > array[parent]) {
                array[child] = array[parent];
            } else {
                break;
            }
            child = parent;
        }
        array[child] = offered;
    }

    public MaxHeap(int[] array) {
        this.array = array;
        this.size = array.length;
        heapify();
    }

    private void heapify() {
        for (int i = size / 2 - 1; i >= 0; i--) {
            down(i);
        }
    }

    private void down(int parent) {
        int left = parent * 2 + 1;
        int right = left + 1;
        int max = parent;
        if (left < size && array[left] > array[max]) {
            max = left;
        }
        if (right < size && array[right] > array[max]) {
            max = right;
        }
        if (max != parent) {
            swap(max, parent);
            down(max);
        }
    }

    private void swap(int i, int j) {
        int t = array[i];
        array[i] = array[j];
        array[j] = t;
    }
}
```

## 什麼是大頂堆？

大頂堆（Max Heap）的特性是：

> 每個父節點的值，都要大於等於它的左右孩子。

例如下面這個陣列：

```text
[7, 5, 6, 4, 2, 1, 3]
```

對應的樹為：

```text
                7
              /   \
             5     6
            / \   / \
           4  2  1   3
```

這就是一個合法的大頂堆，因為：

* `7 > 5, 6`
* `5 > 4, 2`
* `6 > 1, 3`

## 獲取堆頂元素 peek()

```java
public int peek() {
    return array[0];
}
```

在堆中，**堆頂元素一定存放在索引 `0`**。

所以要取得最大值，只要直接回傳 `array[0]` 即可。

例如：

```text
[7, 5, 6, 4, 2, 1, 3]
```

堆頂就是：

```text
7
```

## 刪除堆頂元素 poll()

```java
public int poll() {
    int top = array[0];
    swap(0, size - 1);
    size--;
    down(0);
    return top;
}
```

刪除堆頂是堆中非常重要的操作。
整個流程可以分成三步：

### 第 1 步：先把堆頂和最後一個元素交換

原本的堆：

```text
[7, 5, 6, 4, 2, 1, 3]
```

```text
                7
              /   \
             5     6
            / \   / \
           4  2  1   3
```

先交換堆頂和最後一個元素：

```text
[3, 5, 6, 4, 2, 1, 7]
```

這樣做的原因是：

> 陣列尾端的元素最容易刪除。
> 如果直接刪除開頭元素，後面的元素都要往前搬移，成本很高。

### 第 2 步：刪除最後一個元素

交換後，把最後一個元素移除，也就是把 `size` 減 1：

```text
[3, 5, 6, 4, 2, 1]
```

```text
                3
              /   \
             5     6
            / \   /
           4  2  1
```

這時 `7` 已經被刪掉了。

### 第 3 步：讓新堆頂往下潛（down）

現在新的堆頂是 `3`，但它不一定符合大頂堆特性。

它的左右孩子是 `5` 和 `6`，其中較大的是 `6`，而 `6 > 3`，所以要交換：

```text
[6, 5, 3, 4, 2, 1]
```

```text
                6
              /   \
             5     3
            / \   /
           4  2  1
```

接著再看 `3` 是否還需要繼續下潛。

`3` 的孩子是 `1`，而 `1 < 3`，所以不需要再交換，調整完成。

### 刪除堆頂的核心思路

可以總結成一句話：

> **先把堆頂換到最後刪掉，再讓交換上來的元素往下潛。**

## 刪除指定索引元素 poll(int index)

```java
public int poll(int index) {
    int deleted = array[index];
    swap(index, size - 1);
    size--;
    down(index);
    return deleted;
}
```

刪除指定位置的元素，思路和刪除堆頂其實很像。

### 範例：刪除索引 1 的元素 5

原本的堆：

```text
[7, 5, 6, 4, 2, 1, 3]
```

```text
                7
              /   \
             5     6
            / \   / \
           4  2  1   3
```

#### 第 1 步：和最後一個元素交換

把索引 `1` 的 `5` 和最後一個元素 `3` 交換：

```text
[7, 3, 6, 4, 2, 1, 5]
```

```text
                7
              /   \
             3     6
            / \   / \
           4  2  1   5
```

#### 第 2 步：刪除最後一個元素

把最後一個元素移除：

```text
[7, 3, 6, 4, 2, 1]
```

```text
                7
              /   \
             3     6
            / \   /
           4  2  1
```

#### 第 3 步：對交換上來的元素做調整

現在索引 `1` 的元素是 `3`，它可能破壞堆結構。

`3` 的左右孩子是 `4` 和 `2`，其中較大的是 `4`，而 `4 > 3`，所以交換：

```text
[7, 4, 6, 3, 2, 1]
```

```text
                7
              /   \
             4     6
            / \   /
           3  2  1
```

此時 `3` 已經沒有孩子了，因此調整完成。

### 這個操作的本質

刪除指定索引元素，本質上也是：

> **先把要刪除的元素換到尾部刪掉，再對交換上來的元素重新調整。**

## 替換堆頂元素 replace(int replaced)

```java
public void replace(int replaced) {
    array[0] = replaced;
    down(0);
}
```

這個方法的意思是：

> 直接用新值覆蓋堆頂，然後讓它往下潛，恢復大頂堆。

### 範例：把堆頂 7 替換成 0

原本：

```text
[7, 5, 6, 4, 2, 1, 3]
```

```text
                7
              /   \
             5     6
            / \   / \
           4  2  1   3
```

直接替換：

```text
[0, 5, 6, 4, 2, 1, 3]
```

```text
                0
              /   \
             5     6
            / \   / \
           4  2  1   3
```

此時明顯不符合大頂堆。

#### 開始下潛

`0` 的左右孩子是 `5` 和 `6`，其中較大的是 `6`，交換：

```text
[6, 5, 0, 4, 2, 1, 3]
```

```text
                6
              /   \
             5     0
            / \   / \
           4  2  1   3
```

繼續看 `0`：

它的左右孩子是 `1` 和 `3`，其中較大的是 `3`，交換：

```text
[6, 5, 3, 4, 2, 1, 0]
```

```text
                6
              /   \
             5     3
            / \   / \
           4  2  1   0
```

此時 `0` 已經沒有孩子，調整完成。

### 替換堆頂的核心思路

> **直接改掉堆頂，然後往下潛。**

這比「先刪除再插入」更直接。

## 插入新元素 offer(int offered)

```java
public boolean offer(int offered) {
    if (size == array.length) {
        return false;
    }
    up(offered);
    size++;
    return true;
}
```

新增元素時，會把元素放到堆尾，然後透過 `up()` 讓它**往上浮**。

### up() 的作用

```java
private void up(int offered) {
    int child = size;
    while (child > 0) {
        int parent = (child - 1) / 2;
        if (offered > array[parent]) {
            array[child] = array[parent];
        } else {
            break;
        }
        child = parent;
    }
    array[child] = offered;
}
```

這段程式的邏輯是：

1. 新元素先視為放在尾部
2. 不斷和父節點比較
3. 如果比父節點大，就把父節點往下搬
4. 一直往上，直到遇到更大的父節點或到達根節點
5. 最後把新元素放到正確位置

這種寫法不是「一路 swap」，而是採用**搬移父節點，最後一次放入新值**，效率會更好一些。

### 範例：插入 8

原本的堆：

```text
[7, 5, 6, 4, 2, 1, 3]
```

如果要新增 `8`，先把它放到尾部：

```text
[7, 5, 6, 4, 2, 1, 3, 8]
```

```text
                7
              /   \
             5     6
            / \   / \
           4   2 1   3
          /
         8
```

#### 第 1 次比較

`8` 的父節點索引為：

$$
\left\lfloor \frac{i - 1}{2} \right\rfloor
$$

目前 `8` 在索引 `7`，所以父節點索引是：

$$
(7 - 1) / 2 = 3
$$

也就是值 `4`。

因為 `8 > 4`，所以 `8` 往上浮：

```text
[7, 5, 6, 8, 2, 1, 3, 4]
```

```text
                7
              /   \
             5     6
            / \   / \
           8   2 1   3
          /
         4
```

#### 第 2 次比較

現在 `8` 到了索引 `3`，父節點是索引 `1`，值為 `5`。

因為 `8 > 5`，繼續往上浮：

```text
[7, 8, 6, 5, 2, 1, 3, 4]
```

```text
                7
              /   \
             8     6
            / \   / \
           5   2 1   3
          /
         4
```

#### 第 3 次比較

現在 `8` 到了索引 `1`，父節點是索引 `0`，值為 `7`。

因為 `8 > 7`，繼續往上浮：

```text
[8, 7, 6, 5, 2, 1, 3, 4]
```

```text
                8
              /   \
             7     6
            / \   / \
           5   2 1   3
          /
         4
```

此時 `8` 已經到達堆頂，上浮完成。

### 插入元素的核心思路

> **先放到尾部，再一路和父節點比較，必要時往上浮。**

## 建堆 heapify()

```java
public MaxHeap(int[] array) {
    this.array = array;
    this.size = array.length;
    heapify();
}
```

如果一開始就給你一個陣列，那就可以直接建堆。

```java
private void heapify() {
    for (int i = size / 2 - 1; i >= 0; i--) {
        down(i);
    }
}
```

### 為什麼從 size / 2 - 1 開始？

因為：

> `size / 2 - 1` 是最後一個非葉子節點的索引。

葉子節點本身不需要下潛，因為它沒有孩子。

所以建堆時，只需要從最後一個非葉子節點開始，往前依次做 `down()` 即可。

### down() 的作用

```java
private void down(int parent) {
    int left = parent * 2 + 1;
    int right = left + 1;
    int max = parent;

    if (left < size && array[left] > array[max]) {
        max = left;
    }
    if (right < size && array[right] > array[max]) {
        max = right;
    }
    if (max != parent) {
        swap(max, parent);
        down(max);
    }
}
```

這個方法的邏輯是：

1. 找出父節點、左孩子、右孩子三者中最大的值
2. 如果最大的不是父節點，表示堆性質被破壞
3. 那就把父節點和較大的孩子交換
4. 再繼續對新的位置遞迴下潛
