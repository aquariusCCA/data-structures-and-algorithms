# 动态数组

> 所属章节：[二、基础数据结构](../README.md) / [數組](./README.md)
> 关键字：DynamicArray、插入、查詢、遍歷、刪除、擴容、懶加載、均攤複雜度
> 建議回查情境：實作 DynamicArray、確認插入刪除搬移邏輯、比較 addLast/add/remove 複雜度或複習擴容策略時

## 本节导读

這一節從固定長度數組出發，說明如何用底層 int[] 實作可變長度的 DynamicArray。內容依序涵蓋插入、查詢、遍歷、刪除、擴容與懶加載，最後整理各操作的時間複雜度。

## 你會在這篇學到什麼

- DynamicArray 需要維護的 size、capacity 與底層 array
- 插入與刪除時為什麼需要搬移元素
- foreach、Iterator 與 Stream 三種遍歷方式
- 擴容與懶加載的基本寫法
- 查詢、插入、刪除的時間複雜度差異

---

Java 的數組（array）在建立時長度就固定，之後無法直接改變長度。
因此若要在中間插入或刪除元素，通常需要把後面的元素依序往後或往前移動，才能維持連續的索引結構。
ArrayList 則是以數組作為底層儲存結構的可變長度容器：當容量不足時，會建立更大的新數組，並把原本的元素複製過去，從而達到「看起來可以變長」的效果；但在中間插入或刪除元素時，仍然通常需要移動其他元素。

思考一下，DynamicArray 這個類裡面需要哪些屬性？

```java
public class DynamicArray {
    // 已存放元素数量（有效区间为 [0, size)）
    private int size = 0; // 逻辑大小，表示動態數組裡有效的元素個數
    private int capacity = 8; // capacity：底层数组的容量
    private int[] array = new int[capacity]; // 存放元素的数组
}
```

## 插入功能實現

```java
/**
 * 动态数组
 */
public class DynamicArray {

	private int size = 0;
	private int capacity = 8;
	private int[] array = new int[capacity];

	/**
	 * 向最后位置 [size] 添加元素
	 *
	 * @param element 待添加元素
	 */
	public void addLast(int element) {
        // size：当前已存放的元素个数，也是「下一个可写入的位置索引」
        // 例如：已有 0 ~ size-1 共 size 个元素，那么尾插 addLast() 就应该写到 array[size]
		array[size] = element;
		size++;
	}

    /**
     * 向 [0 .. size] 位置添加元素
     *
     * @param index   索引位置
     * @param element 待添加元素
     */
    public void add(int index, int element) {
        // 添加逻辑
        if (index >= 0 && index < size) {
            // 向后挪动, 空出待插入位置
            System.arraycopy(array, index, array, index + 1, size - index);
            array[index] = element;
            size++;
        }
    }
}
```

### 插入代碼改進
**方法复用**：将 `addLast` 的逻辑改为调用 `add(size, element)`，使得所有插入逻辑都在 `add` 方法中处理。这样可以减少重复代码，并简化 `addLast` 的实现。

```java
public class DynamicArray {
	private int size = 0;
	private int capacity = 8;
	private int[] array = new int[capacity];

	public void addLast(int element) {
		add(size, element);
	}

	public void add(int index, int element) {
		// 添加逻辑
		if (index >= 0 && index < size) {
			// 向后挪动, 空出待插入位置
			System.arraycopy(array, index, array, index + 1, size - index);
			array[index] = element;
			size++;
		} else if (index == size) {
			array[index] = element;
			size++;
		}
	}
}
```

在原版的 `add` 方法中，有两种情况：

1. **插入位置在 `0` 到 `size-1` 之间**：当 `index` 在 `0` 到 `size-1` 范围时，调用 `System.arraycopy` 将 `index` 及之后的元素向后移动一位，以腾出插入位置，然后将 `element` 插入到 `array[index]`，并增加 `size`。
2. **插入位置为 `size`**：如果 `index == size`，意味着在末尾插入一个元素。此时，不需要移动任何元素，直接将 `element` 插入到末尾 `array[size]`，并增加 `size`。

在改进版中，对上述逻辑进行了简化：

1. 首先检查 `index` 是否在 `0` 到 `size` 范围内。
2. 若 `index` 在 `0` 到 `size-1` 之间，则调用 `System.arraycopy` 将当前位置及之后的元素向后移动一位，腾出插入位置。
3. 无论插入位置在哪里，都直接将 `element` 插入到 `array[index]`，并增加 `size`。

```java
public void addLast(int element) {
    add(size, element);
}

public void add(int index, int element) {
    // 添加逻辑
    if (index >= 0 && index < size) {
        // 向后挪动, 空出待插入位置
        System.arraycopy(array, index, array, index + 1, size - index);
    }
    array[index] = element;
    size++;
}
```

## 查詢元素

```java
/**
 * 动态数组
 */
public class DynamicArray {
	private int size = 0;
	private int capacity = 8;
	private int[] array = new int[capacity];

	/**
	 * 查询元素
	 *
	 * @param index 索引位置, 在 [0..size) 区间内
	 * @return 该索引位置的元素
	 */
	public int get(int index) {
		return array[index];
	}
}
```

## 元素遍歷

### foreach 遍历

```java
public class DynamicArray {
	private int size = 0;
	private int capacity = 8;
	private int[] array = new int[capacity];

	public void foreach() {
		for (int i = 0; i < size; i++) {
            System.out.println(array[i]);
		}
	}
}
```

```java
@Test
@DisplayName("测试遍历")
public void test() {
  DynamicArray dynamicArray = new DynamicArray();
  dynamicArray.addLast(1);
  dynamicArray.addLast(2);
  dynamicArray.addLast(3);
  dynamicArray.addLast(4);

  dynamicArray.foreach();
}
```

思考一下這樣寫好不好，這樣寫相當於這個 foreach 方法只能做一件事情，就是打印每個元素。

如果外界想要拿到每一個元素，做一些別的操作，比如說：把元素插入到數據庫等，那這個 foreach 方法就不能使用了。

那能不能把它要做的事情不要寫死在方法裡面，而是透過方法的參數傳遞進來呢？

如果在 `foreach()` 方法裡直接寫死 `System.out.println`，那這個方法就只剩下一個用途：列印每個元素。
遍歷邏輯（走訪每個元素）和業務行為（要對元素做什麼）被綁在一起，導致可重用性很差。

一旦呼叫端想對元素做其他事情，例如：

* 寫入資料庫
* 累加統計、計算總和
* 過濾、轉換、收集到另一個容器

就必須重新寫一份遍歷程式碼，原本的 `foreach()` 幾乎派不上用場。

更好的設計是：遍歷只負責提供元素，行為交由呼叫端決定。

做法是把「對每個元素要執行的操作」抽成參數（例如 Consumer），由外部傳入 callback，讓遍歷方法保持通用。

```java
public class DynamicArray {
	private int size = 0;
	private int capacity = 8;
	private int[] array = new int[capacity];

  /**
   * 遍历有效元素（索引范围 [0, size)），并把每个元素交给调用方处理。
   *
   * 说明：
   * - 遍历本身只负责「按顺序提供元素」
   * - 具体要对元素做什么（打印、入库、统计等）由参数 consumer 决定
   *
   * @param consumer 对每个元素要执行的操作
   */
	public void foreach(Consumer<Integer> consumer) {
		for (int i = 0; i < size; i++) {
			// 提供 array[i]
			// 返回 void
			consumer.accept(array[i]);
		}
	}
}
```

```java
@Test
@DisplayName("测试 foreach 遍历")
public void test() {
  DynamicArray dynamicArray = new DynamicArray();
  dynamicArray.addLast(1);
  dynamicArray.addLast(2);
  dynamicArray.addLast(3);
  dynamicArray.addLast(4);

  dynamicArray.foreach((element) -> {
      System.out.println(element);
  });
}
```

### 迭代器遍历

```java
public class DynamicArray implements Iterable<Integer> {
	private int size = 0;
	private int capacity = 8;
	private int[] array = new int[capacity];

	/**
	 * 遍历方法 - 迭代器遍历
	 */
	@Override
	public Iterator<Integer> iterator() {
		return new Iterator<Integer>() {
			int i = 0;

			@Override
			public boolean hasNext() { // 有没有下一个元素
				return i < size;
			}

			@Override
			public Integer next() { // 返回当前元素,并移动到下一个元素
				return array[i++];
			}
		};
	}
}
```

```java
@Test
@DisplayName("测试 Iterator 遍历")
public void test() {
    DynamicArray dynamicArray = new DynamicArray();
    dynamicArray.addLast(1);
    dynamicArray.addLast(2);
    dynamicArray.addLast(3);
    dynamicArray.addLast(4);

    assertIterableEquals(List.of(1, 2, 3, 4), dynamicArray);

    for (Integer element : dynamicArray) {
        System.out.println(element);
    }
}
```

### Stream 遍历

```java
public class DynamicArray {
	private int size = 0;
	private int capacity = 8;
	private int[] array = new int[capacity];

	/**
	 * 遍历方法 - stream 遍历
	 *
	 * @return stream 流
	 */
	public IntStream stream() {
		return IntStream.of(Arrays.copyOfRange(array, 0, size));
	}
}
```

```java
@Test
@DisplayName("测试 stream 遍历")
public void test() {
	DynamicArray dynamicArray = new DynamicArray();
	dynamicArray.addLast(1);
	dynamicArray.addLast(2);
	dynamicArray.addLast(3);
	dynamicArray.addLast(4);

	assertArrayEquals(new int[] { 1, 2, 3, 4 }, dynamicArray.stream().toArray());
}
```

## 元素刪除
動態數組的有效元素範圍是 [0, size)，也就是索引 0 ~ size - 1。

刪除某個位置 index 的元素，本質上分成三步：

1. 先保存被刪除的元素（因為後續會覆蓋它）
2. 將後面的元素整段往前搬移一格，補上空缺
   - 要搬移的是區間 [index + 1, size - 1]
3. size 減 1，代表有效元素少了一個

```java
public class DynamicArray {
	private int size = 0;
	private int capacity = 8;
	private int[] array = new int[capacity];


	/**
	 * 从 [0 .. size) 范围删除元素
	 *
	 * @param index 索引位置
	 * @return 被删除元素
	 */
	public int remove(int index) { // [0..size)
		int removed = array[index];
		// 向前挪动
		System.arraycopy(array, index + 1, array, index, size - index - 1);
		size--;
		return removed;
	}
}
```

```java
@Test
@DisplayName("测试删除")
public void test() {
	DynamicArray dynamicArray = new DynamicArray();
	dynamicArray.addLast(1);
	dynamicArray.addLast(2);
	dynamicArray.addLast(3);
	dynamicArray.addLast(4);
	dynamicArray.addLast(5);

	int removed = dynamicArray.remove(2);
	assertEquals(3, removed);
}
```

### 特殊情況：刪除最後一個元素

假設元素為 `[1, 2, 3, 4, 5]`，刪除索引 4（也就是最後一個）：

* size = 5
* index = 4
* 需要搬移的元素數量：$size - index - 1 = 5 - 4 - 1 = 0$

這代表 後面沒有元素需要搬移，所以 `System.arraycopy(..., 0)` 其實什麼都不做，是完全合法的。

因此，不加判斷也能正確工作。

如果希望避免呼叫一次「拷貝 0 個元素」的 arraycopy（純粹是微優化、提升可讀性也行），可以加上判斷：

```java
public class DynamicArray{
	private int size = 0;
	private int capacity = 8;
	private int[] array = new int[capacity];

	/**
	 * 从 [0 .. size) 范围删除元素
	 *
	 * @param index 索引位置
	 * @return 被删除元素
	 */
	public int remove(int index) { // [0..size)
		int removed = array[index];
		if (index < size - 1) {
			// 向前挪动
			System.arraycopy(array, index + 1, array, index, size - index - 1);
		}
		size--;
		return removed;
	}
}
```

## 扩容

```java
public class DynamicArray {
	private int size = 0;
	private int capacity = 8;
	private int[] array = new int[capacity];

	/**
	 * 向最后位置 [size] 添加元素
	 *
	 * @param element 待添加元素
	 */
	public void addLast(int element) {
		add(size, element);
	}

	/**
	 * 向 [0 .. size] 位置添加元素
	 *
	 * @param index   索引位置
	 * @param element 待添加元素
	 */
	public void add(int index, int element) {
		checkAndGrow();

		// 添加逻辑
		if (index >= 0 && index < size) {
			// 向后挪动, 空出待插入位置
			System.arraycopy(array, index, array, index + 1, size - index);
		}
		array[index] = element;
		size++;
	}

    private void checkAndGrow() {
        // 容量檢查
        if (size == capacity) {
            // 进行扩容 1.5 倍
            capacity += capacity >> 1;
            int[] newArray = new int[capacity];
            System.arraycopy(array, 0, newArray, 0, size);
            array = newArray;
        }
    }
}
```

## 懒加载數組

```java
public class DynamicArray {
	private int size = 0;
	private int capacity = 8;
	private int[] array = {};

	/**
	 * 向最后位置 [size] 添加元素
	 *
	 * @param element 待添加元素
	 */
	public void addLast(int element) {
		add(size, element);
	}

	/**
	 * 向 [0 .. size] 位置添加元素
	 *
	 * @param index   索引位置
	 * @param element 待添加元素
	 */
	public void add(int index, int element) {
		checkAndGrow();

		// 添加逻辑
		if (index >= 0 && index < size) {
			// 向后挪动, 空出待插入位置
			System.arraycopy(array, index, array, index + 1, size - index);
		}
		array[index] = element;
		size++;
	}

    private void checkAndGrow() {
        // 容量检查
        if (size == 0) {
            array = new int[capacity];
        } else if (size == capacity) {
            // 进行扩容 1.5 倍
            capacity += capacity >> 1;
            int[] newArray = new int[capacity];
            System.arraycopy(array, 0,
                    newArray, 0, size);
            array = newArray;
        }
    }
}
```

## 查詢、插入、删除性能
### 1) 查詢（依索引存取）

* **依索引讀取**：`get(i)`

  * 底層是連續記憶體，可直接定位到第 `i` 個位置
  * 時間複雜度：**O(1)**

### 2) 插入（insert）

插入的關鍵成本在於：**為了維持連續索引，需要搬移插入點後面的元素**。

* **頭部插入**：`add(0, x)`

  * 需要把原本的元素整段往後移
  * 時間複雜度：**O(n)**

* **中間插入**：`add(index, x)`（0 < index < size）

  * 同樣需要搬移插入點後面的元素
  * 時間複雜度：**O(n)**

* **尾部追加**：`addLast(x)` / `add(size, x)`

  * 通常直接寫到 `array[size]`
  * **均攤（amortized）時間複雜度：O(1)**
  * 但如果容量不足，需要**擴容（新建陣列 + 複製舊資料）**，該次操作可能會變成 **O(n)**

### 3) 刪除（remove）

刪除的關鍵成本也在於：**刪除點後面的元素要往前補位**。

* **頭部刪除**：`remove(0)`

  * 後面元素需整段往前移
  * 時間複雜度：**O(n)**

* **中間刪除**：`remove(index)`（0 < index < size - 1）

  * 後面元素需搬移補上空缺
  * 時間複雜度：**O(n)**

* **尾部刪除**：`remove(size - 1)`

  * 不需要搬移元素，只是 `size--`
  * 時間複雜度：**O(1)**
