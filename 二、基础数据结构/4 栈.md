# 概述

在計算機科學中，**棧（Stack）** 是一種**線性資料結構**，它的特點是：

* 只能在**同一端**進行操作
* 這一端稱為 **棧頂**
* 另一端稱為 **棧底**
* 支援的基本操作是：

    * 從棧頂放入元素：**push**
    * 從棧頂取出元素：**pop**
    * 查看棧頂元素但不移除：**peek**

棧的行為遵循：

> **後進先出（LIFO, Last In First Out）**

也就是說，**最後放進去的元素，會最先被拿出來**。

這很像生活中一疊書：

* 放書：只能放在最上面
* 拿書：也只能從最上面拿

# 棧的設計

先提供一个栈接口：

```java
public interface Stack<E> {
    /**
     * 向栈顶压入元素
     * @param value 待压入值
     * @return 压入成功返回 true, 否则返回 false
     */
    boolean push(E value);

    /**
     * 从栈顶弹出元素
     * @return 栈非空返回栈顶元素, 栈为空返回 null
     */
    E pop();

    /**
     * 返回栈顶元素, 不弹出
     * @return 栈非空返回栈顶元素, 栈为空返回 null
     */
    E peek();

    /**
     * 判断栈是否为空
     * @return 空返回 true, 否则返回 false
     */
    boolean isEmpty();

    /**
     * 判断栈是否已满
     * @return 满返回 true, 否则返回 false
     */
    boolean isFull();
}
```

# 链表实现
## 為什麼鏈表實作棧只需要一個指針？

在學習棧之前，可以先回想一下 **鏈表實作隊列** 的情況。

### 隊列為什麼需要兩個指針？

隊列（Queue）是 **兩端操作** 的結構：

* 一端加入元素
* 另一端移除元素

所以在鏈表實作隊列時，通常需要：

* `sentinel`（哨兵節點）：方便處理頭部刪除
* `tail`（尾指針）：方便在尾部新增

例如下面這個隊列實作，就是使用：

* `sentinel` 管理頭部
* `tail` 管理尾部

```java
/**
 * 基於單向環形鏈表實作
 *
 * @param <E> 隊列中元素類型
 */
public class LinkedListQueue<E> implements Queue<E>, Iterable<E> {

    private static class Node<E> {
        E value;
        Node<E> next;

        Node(E value, Node<E> next) {
            this.value = value;
            this.next = next;
        }
    }

    // 哨兵節點，不存放真正資料
    private final Node<E> sentinel = new Node<>(null, null);

    // 空隊列時 tail == sentinel，否則 tail 指向最後一個有效節點
    private Node<E> tail = sentinel;

    // 當前元素個數
    private int size;

    // 隊列容量
    private int capacity = Integer.MAX_VALUE;

    {
        // 初始化成環
        sentinel.next = sentinel;
    }

    public LinkedListQueue() {
    }

    public LinkedListQueue(int capacity) {
        this.capacity = capacity;
    }

    @Override
    public boolean offer(E value) {
        if (isFull()) {
            return false;
        }
        Node<E> added = new Node<>(value, sentinel);
        tail.next = added;
        tail = added;
        size++;
        return true;
    }

    @Override
    public E poll() {
        if (isEmpty()) {
            return null;
        }

        Node<E> first = sentinel.next;
        sentinel.next = first.next;

        // 如果原本只有一個元素，刪除後要恢復空隊列狀態
        if (first == tail) {
            tail = sentinel;
        }

        size--;
        return first.value;
    }

    @Override
    public E peek() {
        if (isEmpty()) {
            return null;
        }
        return sentinel.next.value;
    }

    @Override
    public boolean isEmpty() {
        return tail == sentinel;
    }

    @Override
    public boolean isFull() {
        return size == capacity;
    }

    @Override
    public Iterator<E> iterator() {
        return new Iterator<E>() {
            Node<E> p = sentinel.next;

            @Override
            public boolean hasNext() {
                return p != sentinel;
            }

            @Override
            public E next() {
                E value = p.value;
                p = p.next;
                return value;
            }
        };
    }

    @Override
    public String toString() {
        StringJoiner sj = new StringJoiner(", ", "[", "]");
        for (E e : this) {
            sj.add(String.valueOf(e));
        }
        return sj.toString();
    }
}
```

### 棧為什麼只需要一個指針？

棧和隊列不同，**棧只操作一端**。

也就是說：

* `push`：只往棧頂加入
* `pop`：只從棧頂移除
* `peek`：只查看棧頂

所以我們不需要像隊列一樣同時管理頭尾兩端，
只要能夠操作**棧頂**即可。

因此，鏈表實作棧時，只需要一個：

* `head`（哨兵節點）

讓 `head.next` 永遠指向**棧頂元素**。

## 鏈表實作棧的核心思路

我們用 **單向鏈表** 來實作棧：

* `head` 是哨兵節點，不存放真實資料
* `head.next` 指向棧頂元素
* 每次 `push` 都把新節點插到最前面
* 每次 `pop` 都把最前面的節點移除

### 示意

假設依序執行：

```java
push(1)
push(2)
push(3)
```

鏈表會變成：

```text
head -> 3 -> 2 -> 1 -> null
```

此時：

* 棧頂是 `3`
* 棧底是 `1`

如果執行 `pop()`：

* 取出 `3`
* 鏈表變成：

```text
head -> 2 -> 1 -> null
```

## 代碼實作

```java
public class LinkedListStack<E> implements Stack<E>, Iterable<E> {

    private final int capacity;
    private int size;
    private final Node<E> head = new Node<>(null, null);

    public LinkedListStack(int capacity) {
        this.capacity = capacity;
    }

    @Override
    public boolean push(E value) {
        if (isFull()) {
            return false;
        }
        head.next = new Node<>(value, head.next);
        size++;
        return true;
    }

    @Override
    public E pop() {
        if (isEmpty()) {
            return null;
        }
        Node<E> first = head.next;
        head.next = first.next;
        size--;
        return first.value;
    }

    @Override
    public E peek() {
        if (isEmpty()) {
            return null;
        }
        return head.next.value;
    }

    @Override
    public boolean isEmpty() {
        // 方式一
        // return head.next == null;

        // 方式二
        return size == 0;
    }

    @Override
    public boolean isFull() {
        return size == capacity;
    }

    @Override
    public Iterator<E> iterator() {
        return new Iterator<E>() {
            Node<E> p = head.next;

            @Override
            public boolean hasNext() {
                return p != null;
            }

            @Override
            public E next() {
                E value = p.value;
                p = p.next;
                return value;
            }
        };
    }

    static class Node<E> {
        E value;
        Node<E> next;

        public Node(E value, Node<E> next) {
            this.value = value;
            this.next = next;
        }
    }
}
```

## 逐個方法理解

### push(E value)

```java
@Override
public boolean push(E value) {
    if (isFull()) {
        return false;
    }
    head.next = new Node<>(value, head.next);
    size++;
    return true;
}
```

#### 做了什麼？

這一行是關鍵：

```java
head.next = new Node<>(value, head.next);
```

意思是：

1. 建立一個新節點，值為 `value`
2. 新節點的 `next` 指向原本的第一個節點
3. 再讓 `head.next` 指向這個新節點

也就是把新元素插入到鏈表最前面，成為新的棧頂。

#### 例子

原本：

```text
head -> 2 -> 1 -> null
```

執行：

```java
push(3)
```

之後：

```text
head -> 3 -> 2 -> 1 -> null
```

### pop()

```java
@Override
public E pop() {
    if (isEmpty()) {
        return null;
    }
    Node<E> first = head.next;
    head.next = first.next;
    size--;
    return first.value;
}
```

#### 做了什麼？

1. 先取得棧頂節點 `first`
2. 讓 `head.next` 跳過它，直接指向下一個節點
3. 大小減一
4. 回傳被刪除節點的值

#### 例子

原本：

```text
head -> 3 -> 2 -> 1 -> null
```

執行：

```java
pop()
```

之後：

```text
head -> 2 -> 1 -> null
```

回傳值是：

```java
3
```

### peek()

```java
@Override
public E peek() {
    if (isEmpty()) {
        return null;
    }
    return head.next.value;
}
```

#### 做了什麼？

直接查看 `head.next` 的值，也就是棧頂元素，
**但不移除它**。

### isEmpty()

```java
@Override
public boolean isEmpty() {
    // 方式一
    // return head.next == null;

    // 方式二
    return size == 0;
}
```

#### 兩種判斷方式

都可以：

* `head.next == null`
* `size == 0`

這裡使用 `size == 0`，因為可讀性較高，也方便和 `isFull()` 保持一致。

### isFull()

```java
@Override
public boolean isFull() {
    return size == capacity;
}
```

如果目前元素數量等於容量上限，就表示棧已滿。

### 迭代器 iterator()

```java
@Override
public Iterator<E> iterator() {
    return new Iterator<E>() {
        Node<E> p = head.next;

        @Override
        public boolean hasNext() {
            return p != null;
        }

        @Override
        public E next() {
            E value = p.value;
            p = p.next;
            return value;
        }
    };
}
```

這樣就可以用 `for-each` 遍歷棧中的元素。

注意遍歷順序是：

> **從棧頂到棧底**

例如：

```text
head -> 3 -> 2 -> 1 -> null
```

遍歷得到：

```java
3, 2, 1
```

### push() 測試

```java

stack.push(2);

```

此時棧中元素為：

```text
3, 2, 1
```

因為後進先出，所以最後壓入的 `3` 位於最頂部。

再執行：

```java

```

由於容量是 `3`，所以壓入失敗，回傳：

棧中一開始是：


依次彈出：

* 第一次 `pop()` -> `3`
* 第二次 `pop()` -> `2`
* 第三次 `pop()` -> `1`
* 第四次 `pop()` -> `null`

這正好符合棧的 **LIFO** 特性。

# 数组实现

## 為什麼把數組右邊當作棧頂？

我們通常會把 **數組右邊** 當作棧頂。

示意圖如下：

```text
底              頂
0   1   2   3   4
t
```

這樣設計有一個很大的好處：

> **在數組尾端新增與刪除元素，效率通常更高，也更容易實作。**

因為：

* 往尾端放元素，不需要搬移其他元素
* 從尾端刪除元素，也不需要整體位移

所以數組實作棧時，很自然會選擇：

* 左邊當棧底
* 右邊當棧頂

## top 指針代表什麼？

這裡最重要的一點是：

> `top` 不是「棧頂元素的索引」，
> 而是「下一個可以放元素的位置」。

這句話非常重要。

### 一開始

```text
底              頂
0   1   2   3   4
t
```

此時：

* `top = 0`
* 表示目前棧是空的
* 下一個元素要放在索引 `0`

## 元素加入的過程

假設依序放入 `a`、`b`、`c`、`d`

### 放入 a

```text
底          頂
0   1   2   3
a
    t
```

說明：

* `a` 放在索引 `0`
* 放完後 `top = 1`
* 下一個元素要放在索引 `1`

### 放入 b

```text
底          頂
0   1   2   3
a   b
        t
```

說明：

* `b` 放在索引 `1`
* 放完後 `top = 2`

### 放入 c

```text
底          頂
0   1   2   3
a   b   c
            t
```

說明：

* `c` 放在索引 `2`
* 放完後 `top = 3`

### 放入 d

```text
底          頂
0   1   2   3
a   b   c   d
                t
```

說明：

* `d` 放在索引 `3`
* 放完後 `top = 4`

此時：

> `top == capacity`

代表棧已滿，不能再繼續放入元素了。

## 如何判斷棧是否為空、是否已滿？

### 棧為空

當：

```java
top == 0
```

表示目前沒有任何元素。

### 棧已滿

當：

```java
top == array.length
```

表示陣列所有位置都已經放滿。

## 程式碼實作

```java
public class ArrayStack<E> implements Stack<E>, Iterable<E> {

    private final E[] array;
    private int top; // 指向下一個可放入元素的位置

    @SuppressWarnings("all")
    public ArrayStack(int capacity) {
        this.array = (E[]) new Object[capacity];
    }

    @Override
    public boolean push(E value) {
        if (isFull()) {
            return false;
        }
        array[top++] = value;
        return true;
    }

    @Override
    public E pop() {
        if (isEmpty()) {
            return null;
        }
        E e = array[--top];
        array[top] = null; // 幫助 GC 回收
        return e;
    }

    @Override
    public E peek() {
        if (isEmpty()) {
            return null;
        }
        return array[top - 1];
    }

    @Override
    public boolean isEmpty() {
        return top == 0;
    }

    @Override
    public boolean isFull() {
        return top == array.length;
    }

    @Override
    public Iterator<E> iterator() {
        return new Iterator<E>() {
            int p = top;

            @Override
            public boolean hasNext() {
                return p > 0;
            }

            @Override
            public E next() {
                return array[--p];
            }
        };
    }
}
```

## 各方法詳細解析

### push(E value)

```java
@Override
public boolean push(E value) {
    if (isFull()) {
        return false;
    }
    array[top++] = value;
    return true;
}
```

#### 這行是關鍵

```java
array[top++] = value;
```

它等價於：

```java
array[top] = value;
top++;
```

也就是：

1. 先把元素放到 `top` 所指的位置
2. 再讓 `top` 往右移一格

#### 例子

假設：

```java
top = 2
```

表示目前前兩格已經有資料，下一個元素應該放在索引 `2`

執行：

```java
push("c")
```

會變成：

```java
array[2] = "c";
top = 3;
```

### pop()

```java
@Override
public E pop() {
    if (isEmpty()) {
        return null;
    }
    E e = array[--top];
    array[top] = null; // 幫助 GC 回收
    return e;
}
```

#### 這行是關鍵

```java
E e = array[--top];
```

它等價於：

```java
top--;
E e = array[top];
```

也就是：

1. 先把 `top` 往左退一格
2. 再取出該位置的元素

因為 `top` 原本指向的是「下一個可放入的位置」，
所以要先退一格，才會回到目前真正的棧頂元素。

#### 例子

假設目前：

```text
索引:   0   1   2
值:     a   b   c
top = 3
```

表示棧頂是 `c`

執行 `pop()`：

1. `top--`，變成 `2`
2. 取出 `array[2]`，也就是 `c`

取出後再做：

```java
array[top] = null;
```

把原本的位置清空。

#### 為什麼要 array[top] = null？

這是為了：

> **幫助垃圾回收器（GC）更早回收不再使用的物件**

如果不把它設為 `null`，陣列還是會持有該物件的引用，
某些情況下可能造成物件無法及時回收。

### peek()

```java
@Override
public E peek() {
    if (isEmpty()) {
        return null;
    }
    return array[top - 1];
}
```

`peek()` 的功能是：

> 查看棧頂元素，但不移除

因為：

* `top` 指向下一個可放入的位置
* 所以真正的棧頂元素在 `top - 1`

#### 例子

假設：

```text
索引:   0   1   2
值:     a   b   c
top = 3
```

那麼：

```java
peek()
```

回傳的就是：

```java
array[2]
```

也就是 `c`

### isEmpty()

```java
@Override
public boolean isEmpty() {
    return top == 0;
}
```

當 `top == 0` 時，代表還沒有放入任何元素，所以棧是空的。

### isFull()

```java
@Override
public boolean isFull() {
    return top == array.length;
}
```

當 `top` 走到陣列長度時，代表所有位置都已經用完，棧已滿。

### iterator()

```java
@Override
public Iterator<E> iterator() {
    return new Iterator<E>() {
        int p = top;

        @Override
        public boolean hasNext() {
            return p > 0;
        }

        @Override
        public E next() {
            return array[--p];
        }
    };
}
```

這個迭代器的設計，是讓我們可以從**棧頂往棧底**遍歷。

#### 為什麼 `p = top`？

因為 `top` 指向的是下一個可放入的位置，
所以第一個要取出的元素應該在 `top - 1`，因此每次都先 `--p` 再取值。

#### 例子

假設陣列內容如下：

```text
索引:   0   1   2
值:     a   b   c
top = 3
```

遍歷順序會是：

```text
c -> b -> a
```

這正好符合棧的順序：**從棧頂到棧底**

## 用一個完整例子理解

假設建立一個容量為 `4` 的棧：

```java
ArrayStack<String> stack = new ArrayStack<>(4);
```

一開始：

```text
array = [null, null, null, null]
top = 0
```

### 執行 push("a")

```text
array = [a, null, null, null]
top = 1
```

### 執行 push("b")

```text
array = [a, b, null, null]
top = 2
```

### 執行 push("c")

```text
array = [a, b, c, null]
top = 3
```

### 執行 peek()

回傳：

```text
c
```

因為棧頂在 `top - 1 = 2`

### 執行 pop()

取出 `c` 後：

```text
array = [a, b, null, null]
top = 2
```

### 再執行 pop()

取出 `b` 後：

```text
array = [a, null, null, null]
top = 1
```

# 20. 有效的括号
> [20. 有效的括号](https://leetcode.cn/problems/valid-parentheses/description/)

## 解題思路

這題的核心是：**括號必須成對，而且關閉順序也要正確。**

例如：

* `()` 是合法的
* `([])` 是合法的
* `(]` 是不合法的
* `([)]` 也是不合法的

### 為什麼想到用棧？

因為括號的匹配規則是 **後進先出**。

例如：

* 先遇到 `(`，之後要等到對應的 `)` 來關閉
* 如果又遇到 `[`，那就要先關閉 `[`，再回頭關閉 `(`

這種「**最後進來的左括號，要最先被匹配**」的特性，很適合用棧處理。

### 具體做法

我們從左到右遍歷字串中的每一個字元。

#### 1. 遇到左括號時

如果目前字元是左括號：

* `(` → 就把對應的右括號 `)` 放進棧中
* `[` → 就把對應的右括號 `]` 放進棧中
* `{` → 就把對應的右括號 `}` 放進棧中

這樣做的好處是：

* 棧頂永遠存放「**接下來最應該出現的右括號**」

#### 2. 遇到右括號時

如果目前字元是右括號，就需要檢查兩件事：

* 棧是否為空
  如果棧是空的，表示前面根本沒有可以和它配對的左括號，直接返回 `false`

* 當前右括號是否等於棧頂元素
  如果相等，說明成功匹配，彈出棧頂
  如果不相等，說明括號類型或順序錯了，直接返回 `false`

#### 3. 遍歷結束後

如果最後棧是空的，表示所有括號都成功匹配，返回 `true`

如果棧不為空，表示還有左括號沒有被關閉，返回 `false`

## 舉例說明

以字串 `([])` 為例：

1. 遇到 `(` → 把 `)` 放入棧
   棧：`)`

2. 遇到 `[` → 把 `]` 放入棧
   棧：`) , ]`

3. 遇到 `]` → 和棧頂 `]` 比較，相等，彈出
   棧：`)`

4. 遇到 `)` → 和棧頂 `)` 比較，相等，彈出
   棧：空

最後棧為空，所以是有效括號。

# 150. 逆波兰表达式求值
> (150. 逆波兰表达式求值)[https://leetcode.cn/problems/evaluate-reverse-polish-notation/description/]

## 什麼是中綴表達式與後綴表達式？

我們平常寫的運算式，大多是**中綴表達式**。

例如：

```text
a + b
a + b * c
(a + b) * c
```

所謂中綴，就是**運算符寫在兩個操作數中間**。

而**後綴表達式**則是把運算符放到最後面，例如：

```text
a + b      -> ab+
a + b * c  -> abc*+
(a + b) * c -> ab+c*
```

後綴表達式的好處是：**不需要額外用括號，就能明確表達運算順序**。

## 為什麼這題適合用棧？

因為後綴表達式的運算規則是：

* **遇到數字，先存起來**
* **遇到運算符，再把前面最近的兩個數字拿出來計算**

這個「**先存、後取，而且後放進去的數字會先被取出**」的特性，正好就是棧的使用場景。

## 解題思路

我們從左到右遍歷 `tokens` 陣列中的每一個元素。

### 1. 遇到數字

如果當前元素是數字，就直接壓入棧中。

### 2. 遇到運算符

如果當前元素是 `+`、`-`、`*`、`/`，就從棧中彈出兩個數字進行計算。

注意順序：

```text
先彈出的數字是右操作數
後彈出的數字是左操作數
```

也就是說：

```java
int b = stack.pop();
int a = stack.pop();
```

真正的計算應該是：

```java
a op b
```

不能寫反，尤其是減法和除法特別重要。

例如：

* `4 2 -` 應該是 `4 - 2 = 2`
* `4 2 /` 應該是 `4 / 2 = 2`

如果順序反了，就會變成錯誤答案。

### 3. 計算後再壓回棧中

把運算結果重新壓入棧中，讓後面的運算繼續使用。

### 4. 遍歷結束

當所有元素都處理完後，棧中最後剩下的那個數字，就是整個逆波蘭表達式的結果。

## 例子說明

以：

```java
tokens = ["2","1","+","3","*"]
```

為例。

它對應的中綴表達式其實就是：

```text
(2 + 1) * 3
```

### 過程 1：讀到 "2"

是數字，壓入棧。

```text
棧： [2]
```

### 過程 2：讀到 "1"

是數字，壓入棧。

```text
棧： [2, 1]
```

### 過程 3：讀到 "+"

是運算符，從棧中彈出兩個數字：

* b = 1
* a = 2

計算：

```text
2 + 1 = 3
```

把結果 3 壓回棧中。

```text
棧： [3]
```

### 過程 4：讀到 "3"

是數字，壓入棧。

```text
棧： [3, 3]
```

### 過程 5：讀到 "*"

是運算符，從棧中彈出兩個數字：

* b = 3
* a = 3

計算：

```text
3 * 3 = 9
```

把結果 9 壓回棧中。

```text
棧： [9]
```

### 最後結果

遍歷結束，棧中只剩下一個元素：

```text
9
```

所以答案就是：

```java
9
```

## 程式碼

```java
/**
 * 后缀表达式求值
 */
public class Leetcode150 {
    public int evalRPN(String[] tokens) {
        LinkedList<Integer> stack = new LinkedList<>();
        for (String t : tokens) {
            switch (t) {
                case "+" -> {
                    Integer b = stack.pop();
                    Integer a = stack.pop();
                    stack.push(a + b);
                }
                case "-" -> {
                    Integer b = stack.pop();
                    Integer a = stack.pop();
                    stack.push(a - b);
                }
                case "*" -> {
                    Integer b = stack.pop();
                    Integer a = stack.pop();
                    stack.push(a * b);
                }
                case "/" -> {
                    Integer b = stack.pop();
                    Integer a = stack.pop();
                    stack.push(a / b);
                }
                default -> { // 数字
                    stack.push(Integer.parseInt(t));
                }
            }
        }
        return stack.pop();
    }

    public static void main(String[] args) {
        String[] tokens = {"10", "6", "9", "3", "+", "-11", "*", "/", "*", "17", "+", "5", "+"};
        System.out.println(new Leetcode150().evalRPN(tokens));

        tokens = new String[]{"2", "1", "+", "3", "*"};
        System.out.println(new Leetcode150().evalRPN(tokens));
    }
}
```

## 中綴表達式轉換後綴表達式

### 為什麼要學後綴表達式？

因為電腦在做運算時，常常不是直接照我們平常寫的 `a + b` 去處理，而是會轉成更適合機器執行的形式。

尤其 JVM 的 bytecode 本質上就是**基於棧（stack）的運算模型**，它在做加法時，會先把操作數放上棧，再執行運算。這種執行方式和後綴表達式的思想非常接近。

### 從 Java bytecode 理解後綴表達式

先看一段 Java 程式：

```java
public class InfixToSuffix {
    public static void test() {
        int a = 10;
        int b = 20;
        int c = a + b;
    }

    public static void main(String[] args) {
        test();
    }
}
```

編譯後，再用下面指令反編譯：

```shell
javap -c InfixToSuffix.class
```

會看到類似輸出：

```shell
Compiled from "InfixToSuffix.java"
public class com.practice.dsa.leetcode.stack.InfixToSuffix {
  public com.practice.dsa.leetcode.stack.InfixToSuffix();
    Code:
       0: aload_0
       1: invokespecial #1                  // Method java/lang/Object."<init>":()V
       4: return

  public static void test();
    Code:
       0: bipush        10
       2: istore_0
       3: bipush        20
       5: istore_1
       6: iload_0
       7: iload_1
       8: iadd
       9: istore_2
      10: return

  public static void main(java.lang.String[]);
    Code:
       0: invokestatic  #7                  // Method test:()V
       3: return
}
```

#### 1. int a = 10;

對應 bytecode：

```shell
0: bipush 10
2: istore_0
```

意思是：

* 把常數 `10` 放進來
* 存到區域變數 `0`，也就是 `a`

#### 2. int b = 20;

對應 bytecode：

```shell
3: bipush 20
5: istore_1
```

意思是：

* 把常數 `20` 放進來
* 存到區域變數 `1`，也就是 `b`

#### 3. int c = a + b;

對應 bytecode：

```shell
6: iload_0
7: iload_1
8: iadd
9: istore_2
```

這裡最值得注意。

`a + b` 不會直接變成「看到 `+` 就馬上算」，而是：

1. 先把 `a` 載入
2. 再把 `b` 載入
3. 最後執行 `iadd`

也就是：

```text
a b +
```

這其實就是**後綴表達式**的思維。

所以可以把 JVM 的這段執行流程理解成：

```text
iload_0   -> 取出 a
iload_1   -> 取出 b
iadd      -> 將前兩個值相加
```

這就是為什麼學「中綴轉後綴」很重要：它可以幫助我們理解**編譯器與棧如何處理運算式**。

### 中綴轉後綴的核心思路

我們的目標是：

> **把原本寫在中間的運算符，移到適當的位置，最後得到正確的後綴表達式。**

這個過程需要一個**棧**來暫存運算符。

基本想法很簡單：

* 遇到**操作數**（例如 `a`、`b`、`c`）就直接輸出
* 遇到**運算符**（例如 `+`、`-`、`*`、`/`）就看情況放入棧中
* 如果棧頂運算符優先級比較高或相同，就先把它彈出來
* 最後把棧裡剩下的運算符全部輸出

### 先用最簡單的例子理解

#### 例子 1：a + b

轉換過程：

1. 讀到 `a`，它不是運算符，直接輸出
2. 讀到 `+`，放進棧中
3. 讀到 `b`，直接輸出
4. 掃描結束，把棧中的 `+` 彈出

結果：

```text
ab+
```

#### 例子 2：a + b - c

先看原式：

```text
a + b - c
```

`+` 和 `-` 的優先級相同，而且是**左結合**，所以要從左到右算，也就是：

```text
(a + b) - c
```

轉換過程：

1. `a` 輸出 → `a`
2. `+` 入棧
3. `b` 輸出 → `ab`
4. 遇到 `-`

    * 棧頂是 `+`
    * `+` 和 `-` 優先級相同
    * 因為要先算左邊的 `+`，所以先把 `+` 彈出
5. `-` 再入棧
6. `c` 輸出 → `ab+c`
7. 最後把 `-` 彈出

結果：

```text
ab+c-
```

#### 例子 3：a * b + c

這裡 `*` 的優先級比 `+` 高，所以要先做乘法。

也就是：

```text
(a * b) + c
```

轉換過程：

1. `a` 輸出 → `a`
2. `*` 入棧
3. `b` 輸出 → `ab`
4. 遇到 `+`

    * 棧頂是 `*`
    * `*` 優先級比 `+` 高
    * 所以先把 `*` 彈出
5. `+` 入棧
6. `c` 輸出 → `ab*c`
7. 最後把 `+` 彈出

結果：

```text
ab*c+
```

#### 例子 4：a + b * c

這裡 `*` 比 `+` 優先級高，所以不能太早把 `+` 彈出。

也就是：

```text
a + (b * c)
```

轉換過程：

1. `a` 輸出 → `a`
2. `+` 入棧
3. `b` 輸出 → `ab`
4. 遇到 `*`

    * 棧頂是 `+`
    * `*` 優先級比 `+` 高
    * 所以 `*` 直接入棧
5. `c` 輸出 → `abc`
6. 掃描結束，依序彈出棧內運算符：先 `*`，再 `+`

結果：

```text
abc*+
```

#### 例子 5：a + b * c - d

先看運算順序：

```text
a + (b * c) - d
```

再根據左結合可理解為：

```text
(a + (b * c)) - d
```

轉換過程：

1. `a` 輸出 → `a`
2. `+` 入棧
3. `b` 輸出 → `ab`
4. `*` 入棧（因為比 `+` 優先級高）
5. `c` 輸出 → `abc`
6. 遇到 `-`

    * 棧頂是 `*`，先彈出 → `abc*`
    * 再看棧頂是 `+`，與 `-` 同級，先彈出 → `abc*+`
    * 最後 `-` 入棧
7. `d` 輸出 → `abc*+d`
8. 結束後彈出 `-`

結果：

```text
abc*+d-
```

### 總結規則

中綴轉後綴時，可以記住這幾條規則：

#### 1. 遇到操作數，直接輸出

例如：

```text
a、b、c、1、2、3
```

都直接放到結果中。

#### 遇到運算符時，和棧頂比較優先級

##### 如果目前運算符優先級比較高

直接入棧。

例如：

```text
棧頂是 +
目前是 *
```

因為 `*` 比 `+` 高，所以 `*` 可以直接進棧。

##### 如果目前運算符優先級比較低或相同

就要先把棧中優先級大於等於它的運算符彈出，然後再把自己放進去。

例如：

```text
棧裡有 + *
目前是 -
```

那就要先把 `*`、`+` 依序彈出，再把 `-` 放進去。

#### 3. 掃描完成後，把棧內剩餘運算符全部彈出

### 程式碼

```java
public class InfixToSuffix {
    public static void main(String[] args) {
        System.out.println(infixToSuffix("a+b"));
        System.out.println(infixToSuffix("a+b-c"));
        System.out.println(infixToSuffix("a+b*c"));
        System.out.println(infixToSuffix("a*b-c"));
    }

    static String infixToSuffix(String exp) {
        LinkedList<Character> stack = new LinkedList<>();
        StringBuilder sb = new StringBuilder(exp.length());
        for (int i = 0; i < exp.length(); i++) {
            char c = exp.charAt(i);
            switch (c) {
                case '*', '/', '+', '-' -> {
                   if (stack.isEmpty()) {
                       stack.push(c);
                   } else {
                       if (priority(c) > priority(stack.peek())) {
                           stack.push(c);
                       } else {
                           while (!stack.isEmpty() && priority(stack.peek()) >= priority(c)) {
                               sb.append(stack.pop());
                           }
                           stack.push(c);
                       }
                   }
                }
                default -> {
                    sb.append(c);
                }
            }
        }
        while (!stack.isEmpty()) {
            sb.append(stack.pop());
        }
        return sb.toString();
    }

    static int priority(char c) {
        return switch (c) {
            case '*', '/' -> 2;
            case '+', '-' -> 1;
            default -> throw new IllegalArgumentException("不合法的运算符:" + c);
        };
    }
}
```

## 中綴表達式轉換後綴表達式（支援括號）

前面我們討論過不含括號的情況，但真實的運算式常常會有括號，例如：

```text
(a + b) * c
a * (b + c)
(a + b * c - d) * e
```

括號的存在，會改變原本的運算順序，所以在「中綴轉後綴」時，也必須額外處理。

### 先理解：為什麼括號需要特別處理？

先看這個例子：

```text
(a + b) * c
```

如果沒有括號，`*` 的優先級比 `+` 高，原本可能會先做乘法。
但因為有了 `(a + b)`，所以括號中的內容必須先算。

也就是說：

```text
(a + b) * c
```

真正的運算順序是：

```text
先算 a + b，再算 * c
```

因此在轉換成後綴表達式時，括號不能直接輸出，而是要幫助我們決定：

* 哪些運算要先做
* 哪些運算要先留在棧中
* 遇到右括號時，要把括號內的運算一次處理完

### 處理括號的核心觀念

在不含括號時，我們的規則是：

1. 遇到操作數，直接輸出
2. 遇到運算符，根據優先級決定是否入棧或彈出
3. 掃描完成後，把棧中剩餘運算符全部彈出

加入括號後，只要再補上兩條規則：

#### 1. 左括號 `(` 直接入棧

左括號可以看成一個**特殊標記**，用來表示「從這裡開始是一個新的區域」。

它的作用不是參與運算，而是幫助我們界定範圍。

#### 2. 遇到右括號 `)` 時

就一直把棧中的運算符彈出，直到遇到左括號 `(` 為止。

注意：

* 彈出的運算符要加入結果
* 左括號只需要丟掉，不需要輸出

因為括號只是改變運算順序，本身不是運算符。

### 為什麼左括號的優先級可以設成最低？

這裡很容易讓初學者困惑。

平常我們會說：

> 括號優先級最高

但在程式實作時，我們常常把左括號 `(` 的優先級設定成最低，例如 `0`。

這麼做**不是在說括號真的比 `+`、`*` 低**，而是為了讓演算法更容易寫。

因為左括號的任務不是參與比較誰先算，而是：

* 阻止外面的運算符影響括號內部
* 幫助右括號找到配對位置

所以可以把 `(` 理解成：

> 不是一般的運算符，而是一個「分界符」

### 例子一：(a + b) * c

我們一步一步看。

#### 初始狀態

* 輸入：`(a+b)*c`
* 棧：空
* 結果：空

#### 1. 讀到 (

左括號直接入棧。

```text
棧:   (
結果:
```

#### 2. 讀到 a

不是運算符，直接輸出。

```text
棧:   (
結果: a
```

#### 3. 讀到 +

棧頂是 `(`，左括號不應該被彈出，所以 `+` 直接入棧。

```text
棧:   ( +
結果: a
```

#### 4. 讀到 b

直接輸出。

```text
棧:   ( +
結果: ab
```

#### 5. 讀到 )

右括號表示：括號內的運算要結束了。

因此要一直彈出棧中運算符，直到遇到 `(` 為止。

先彈出 `+`：

```text
棧:   (
結果: ab+
```

再把 `(` 移除，但不輸出。

```text
棧:
結果: ab+
```

#### 6. 讀到 *

直接入棧。

```text
棧:   *
結果: ab+
```

直接輸出。

把棧中剩餘運算符彈出。


### 例子二：(a + b * c - d) * e

這個例子比較完整，因為同時包含：

* 括號
* 不同優先級
* 同優先級運算符

原式：


#### 1. 讀到 (

左括號入棧。

```text
棧:   (
結果:
```

#### 2. 讀到 a

直接輸出。

```text
棧:   (
結果: a
```

#### 3. 讀到 +

入棧。

```text
棧:   ( +
結果: a
```

#### 4. 讀到 b

直接輸出。

```text
棧:   ( +
結果: ab
```

#### 5. 讀到 *

`*` 的優先級比 `+` 高，所以直接入棧。

```text
棧:   ( + *
結果: ab
```

#### 6. 讀到 c

直接輸出。

* 接著棧頂變成 `+`
* `+` 和 `-` 優先級相同，且運算是左結合，所以 `+` 也要先彈出
* 最後再把 `-` 入棧

結果變成：

一直彈出運算符直到遇到 `(`。

先彈出 `-`：

```text
結果: abc*+d-
```

再移除 `(`，但不輸出。

```text
棧:
結果: abc*+d-
```

#### 10. 讀到 *

入棧。

```text

結果: abc*+d-
```

#### 11. 讀到 e

直接輸出。

```text
棧:   *
結果: abc*+d-e
```

#### 12. 掃描結束

把棧中 `*` 彈出。

```text
棧:
結果: abc*+d-e*
```

#### 最終結果

```text
(a+b*c-d)*e  ->  abc*+d-e*
```

這個例子可以幫助我們理解：

> 左括號進棧時，不需要和棧頂運算符比較優先級。

### 總結規則

支援括號後，中綴轉後綴的規則可以整理成下面幾點：

#### 1. 遇到操作數，直接輸出

例如：

```text
a、b、c、1、2、3
```

直接拼到結果字串。

#### 2. 遇到一般運算符 + - * /

* 如果棧為空，直接入棧
* 如果目前運算符優先級 **高於** 棧頂運算符，直接入棧
* 否則，把棧中優先級 **大於等於** 它的運算符都彈出，再把目前運算符入棧

#### 3. 遇到左括號 (

直接入棧。

#### 4. 遇到右括號 )

一直彈出棧中運算符，直到遇到左括號 `(` 為止。
左括號只移除，不輸出。

#### 5. 掃描完成後

把棧中剩餘運算符全部彈出。

### 完整程式碼

下面這份程式碼就是支援括號的版本：

```java
package com.practice.dsa.leetcode.stack;

import java.util.LinkedList;

/**
 * 中綴表達式轉後綴表達式
 */
public class InfixToSuffix {
    public static void main(String[] args) {
        System.out.println(infixToSuffix("(a+b)*c"));       // ab+c*
        System.out.println(infixToSuffix("(a+b*c-d)*e"));   // abc*+d-e*
        System.out.println(infixToSuffix("a*(b+c)"));       // abc+*
    }

    static int priority(char c) {
        return switch (c) {
            case '*', '/' -> 2;
            case '+', '-' -> 1;
            case '(' -> 0;
            default -> throw new IllegalArgumentException("不合法的運算符: " + c);
        };
    }

    static String infixToSuffix(String exp) {
        LinkedList<Character> stack = new LinkedList<>();
        StringBuilder sb = new StringBuilder(exp.length());

        for (int i = 0; i < exp.length(); i++) {
            char c = exp.charAt(i);

            switch (c) {
                case '*', '/', '+', '-' -> {
                    if (stack.isEmpty()) {
                        stack.push(c);
                    } else {
                        if (priority(c) > priority(stack.peek())) {
                            stack.push(c);
                        } else {
                            while (!stack.isEmpty() && priority(stack.peek()) >= priority(c)) {
                                sb.append(stack.pop());
                            }
                            stack.push(c);
                        }
                    }
                }
                case '(' -> {
                    // 左括號直接入棧
                    stack.push(c);
                }
                case ')' -> {
                    // 右括號：彈出直到遇到左括號
                    while (!stack.isEmpty() && stack.peek() != '(') {
                        sb.append(stack.pop());
                    }
                    stack.pop(); // 移除左括號，但不輸出
                }
                default -> {
                    // 操作數直接輸出
                    sb.append(c);
                }
            }
        }

        // 將棧中剩餘運算符全部輸出
        while (!stack.isEmpty()) {
            sb.append(stack.pop());
        }

        return sb.toString();
    }
}
```

# 232. 用栈实现队列

> 題目連結：
> [232. 用栈实现队列](https://leetcode.cn/problems/implement-queue-using-stacks/description/)

## 題目要求

只能使用**棧（Stack）**來實作**隊列（Queue）**。

而隊列的特性是：

* **先進先出**（FIFO, First In First Out）

但棧的特性是：

* **後進先出**（LIFO, Last In First Out）

所以這題的重點就是：
**如何用兩個後進先出的結構，模擬出先進先出的效果。**

## 核心思路

使用兩個棧：

* `s2`：專門負責**入隊**
* `s1`：專門負責**出隊**

可以這樣理解：

```text
队列头                队列尾

顶   底              底   顶
s1                       s2
```

也就是說：

* 新元素進來時，先放進 `s2`
* 要從隊列頭取元素時，從 `s1` 取
* 如果 `s1` 是空的，就把 `s2` 的所有元素倒進 `s1`

---

## 為什麼要把 `s2` 倒進 `s1`？

因為棧只能從頂部操作。

假設依序加入兩個元素：

```text
push(a)
push(b)
```

此時資料都在 `s2` 中：

```text
队列头                队列尾
                     a   b
顶   底              底   顶
s1                   s2
```

注意：

* `a` 是最早加入的，應該最先出隊
* 但 `s2` 的頂部是 `b`
* 如果直接從 `s2` 彈出，就會變成 `b` 先出去，這不符合隊列規則

所以我們要把 `s2` 的元素一個一個彈出，再壓入 `s1`。

## 搬運過程示例

### 第一步：b 從 s2 彈出，再壓入 s1

```text
队列头                队列尾
b                         a
顶   底              底   顶
s1                       s2
```

### 第二步：a 從 s2 彈出，再壓入 s1

```text
队列头                队列尾
a    b
顶   底              底   顶
s1                       s2
```

這時候：

* `s1` 的頂部就是 `a`
* 也就是最早進入隊列的元素
* 因此可以正確模擬隊列的「先進先出」

## 操作規則整理

### 1. push(x)：加入隊列尾部

直接把元素放進 `s2`

```java
s2.push(x);
```

### 2. pop()：移除隊列頭部元素

* 如果 `s1` 不為空，直接從 `s1` 彈出
* 如果 `s1` 是空的，就先把 `s2` 所有元素搬到 `s1`
* 然後再從 `s1` 彈出

### 3. peek()：查看隊列頭部元素

邏輯和 `pop()` 類似：

* 如果 `s1` 有元素，直接看 `s1` 的棧頂
* 如果 `s1` 是空的，就先把 `s2` 搬到 `s1`
* 再查看 `s1` 的棧頂

### 4. empty()：判斷隊列是否為空

當 `s1` 和 `s2` 都為空時，隊列才為空。

## 一個重要細節

### 為什麼不是每次 push 後都立刻搬運？

因為這樣效率比較差。

更好的做法是：

* **入隊時只管放進 `s2`**
* **只有在 `s1` 為空且需要出隊 / 取隊頭時，才把 `s2` 搬到 `s1`**

這樣可以避免重複搬運元素。

## 程式碼

```java
/**
 * 双栈模拟队列
 *
 * <ul>
 *     <li>调用 push、pop 等方法的次数最多 100</li>
 * </ul>
 */
public class Leetcode232 {

    ArrayStack<Integer> s1 = new ArrayStack<>(100);
    ArrayStack<Integer> s2 = new ArrayStack<>(100);

    public void push(int x) { // 向队列尾添加
        s2.push(x);
    }

    public int pop() { // 从队列头移除
        if (s1.isEmpty()) {
            while (!s2.isEmpty()) {
                s1.push(s2.pop());
            }
        }
        return s1.pop();
    }

    public int peek() { // 从队列头获取
        if (s1.isEmpty()) {
            while (!s2.isEmpty()) {
                s1.push(s2.pop());
            }
        }
        return s1.peek();
    }

    public boolean empty() {
        return s1.isEmpty() && s2.isEmpty();
    }
}
```

# 225. 用队列实现栈

> 題目連結：
> [225. 用队列实现栈](https://leetcode.cn/problems/implement-stack-using-queues/description/)

## 題目要求

只能使用**隊列（Queue）**來實作**棧（Stack）**。

但這兩種資料結構的特性剛好相反：

* **棧**：後進先出（LIFO, Last In First Out）
* **隊列**：先進先出（FIFO, First In First Out）

所以這題的關鍵在於：

> **如何用先進先出的隊列，模擬出後進先出的棧。**

## 核心思路

我們只用**一個隊列**來模擬棧。

棧最重要的特性是：

* 新加入的元素，要最先被取出
* 也就是說，**最新加入的元素必須在最前面**

但隊列的新增操作 `offer()` 只能把元素放到**隊列尾部**。
因此，每次加入新元素後，我們都要想辦法把它「轉到最前面」。

## 為什麼要旋轉隊列？

假設現在依序加入元素 `a`、`b`。

### 先加入 a

```text
栈顶      栈底
         a
队列头    队列尾

queue.offer(a)
```

這時只有一個元素，沒有問題。

### 再加入 b

```text
栈顶      栈底
b        a
队列头    队列尾

queue.offer(b)
```

實際上，`b` 會先被放到隊列尾部，所以隊列中的順序原本是：

```text
a -> b
```

但如果要表現出「棧」的行為，我們希望：

* `b` 是最新加入的
* 所以 `b` 應該在**棧頂**
* 也就是應該排在 `a` 前面

因此，我們要把 `b` 前面的元素重新移到後面，讓 `b` 轉到隊列頭部。

## 操作示例

### 加入 a

```text
queue.offer(a)
```

隊列內容：

```text
a
```

### 加入 b

先放進隊列尾部：

```text
a -> b
```

接著把 `b` 前面的元素 `a` 從隊列頭取出，再放回隊列尾：

```text
b -> a
```

這樣 `b` 就跑到最前面了，也就模擬出了棧頂的效果。

### 加入 c

先放進隊列尾部：

```text
b -> a -> c
```

但我們希望 `c` 成為新的棧頂，所以要把它前面的元素都移到後面。

先移動 `b`：

```text
a -> c -> b
```

再移動 `a`：

```text
c -> b -> a
```

最後結果：

```text
栈顶      栈底
c        b   a
队列头    队列尾
```

這就符合棧的順序了。

## 規則整理

### push(x)：加入元素

做兩件事：

1. 先把新元素加入隊列尾部
2. 再把它前面的所有元素，依序從隊列頭移到隊列尾

這樣新元素就會被轉到最前面，也就是棧頂的位置。

### pop()：移除棧頂元素

因為我們在 `push()` 時，已經把最新元素放到隊列頭部了，
所以 `pop()` 時只要直接移除隊列頭即可。

### top()：查看棧頂元素

直接查看隊列頭元素即可。

### empty()：判斷是否為空

直接判斷隊列是否為空即可。

## 程式碼

```java
/**
 * 单队列模拟栈
 * <ol>
 *     <li>调用 push、pop 等方法的次数最多 100</li>
 *     <li>每次调用 pop 和 top 都能保证栈不为空</li>
 * </ol>
 */
public class Leetcode225 {

    ArrayQueue3<Integer> queue = new ArrayQueue3<>(100);
    private int size = 0;

    public void push(int x) {
        queue.offer(x);
        for (int i = 0; i < size; i++) {
            queue.offer(queue.poll());
        }
        size++;
    }

    public int pop() {
        size--;
        return queue.poll();
    }

    public int top() {
        return queue.peek();
    }

    public boolean empty() {
        return queue.isEmpty();
    }
}
```
