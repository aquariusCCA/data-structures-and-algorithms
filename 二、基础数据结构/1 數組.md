# 概述

## 定義

> In computer science, an array is a data structure consisting of a collection of elements (values or variables), each identified by at least one array index or key
>
> - **定义**：在计算机科学中，数组是由一组元素（值或变量）组成的数据结构，每个元素有至少一个索引或键来标识。

因为数组内的元素是**连续存储**的，所以数组中元素的地址，可以通过其索引计算出来，例如：

```java
int[] array = {1, 2, 3, 4, 5};
```

知道了数组的 **数据** 起始地址 $BaseAddress$，就可以由公式 $BaseAddress + i \times size$ 计算出索引 $i$ 元素的地址

- $i$ 即索引，在 Java、C 等语言都是从 0 开始
- $size$ 是每个元素占用字节，例如 int 占 4，double 占 8

**小测试**

```java
byte[] array = {1, 2, 3, 4, 5};
```

已知 array 的 **数据** 的起始地址是 `0x7138f94c8`，那么元素 3 的地址是什么 ？

> 答：$0x7138f94c8 + 2 \times 1 = 0x7138f94ca$

# 动态数组

Java 的數組（array）在建立時長度就固定，之後無法直接改變長度。
因此若要在中間插入或刪除元素，通常需要把後面的元素依序往後或往前移動，才能維持連續的索引結構。
ArrayList 則是以數組作為底層儲存結構的可變長度容器：當容量不足時，會建立更大的新數組，並把原本的元素複製過去，從而達到「看起來可以變長」的效果；但在中間插入或刪除元素時，仍然通常需要移動其他元素。

思考一下，DynamicArray 這個類裡面需要哪些屬性？

```java
    private int size = 0; // 逻辑大小，表示動態數組裡有效的元素個數
    private int capacity = 8; // capacity：底层数组的容量
    private int[] array = new int[capacity]; // 存放元素的数组
}
```

## 插入功能實現 

```java
/**
 * 动态数组
 */
public class DynamicArray {
    
	private int size = 0;
	private int capacity = 8; 
	private int[] array = new int[capacity]; 

	/**
	 * 向最后位置 [size] 添加元素
	 *
	 * @param element 待添加元素
	 */
	public void addLast(int element) {
        // size：当前已存放的元素个数，也是「下一个可写入的位置索引」
        // 例如：已有 0 ~ size-1 共 size 个元素，那么尾插 addLast() 就应该写到 array[size]
		array[size] = element;
		size++;
	}

    /**
     * 向 [0 .. size] 位置添加元素
     *
     * @param index   索引位置
     * @param element 待添加元素
     */
    public void add(int index, int element) {
        // 添加逻辑
        if (index >= 0 && index < size) {
            // 向后挪动, 空出待插入位置
            System.arraycopy(array, index, array, index + 1, size - index);
            array[index] = element;
            size++;
        }
    }
}
```

### 插入代碼改進
**方法复用**：将 `addLast` 的逻辑改为调用 `add(size, element)`，使得所有插入逻辑都在 `add` 方法中处理。这样可以减少重复代码，并简化 `addLast` 的实现。

```java
public class DynamicArray {
	private int size = 0; 
	private int capacity = 8; 
	private int[] array = new int[capacity];

	public void addLast(int element) {
		add(size, element);
	}

	public void add(int index, int element) {
		// 添加逻辑
		if (index >= 0 && index < size) {
			// 向后挪动, 空出待插入位置
			System.arraycopy(array, index, array, index + 1, size - index);
			array[index] = element;
			size++;
		} else if (index == size) {
			array[index] = element;
			size++;
		}
	}
}
```

在原版的 `add` 方法中，有两种情况：

1. **插入位置在 `0` 到 `size-1` 之间**：当 `index` 在 `0` 到 `size-1` 范围时，调用 `System.arraycopy` 将 `index` 及之后的元素向后移动一位，以腾出插入位置，然后将 `element` 插入到 `array[index]`，并增加 `size`。
2. **插入位置为 `size`**：如果 `index == size`，意味着在末尾插入一个元素。此时，不需要移动任何元素，直接将 `element` 插入到末尾 `array[size]`，并增加 `size`。

在改进版中，对上述逻辑进行了简化：

1. 首先检查 `index` 是否在 `0` 到 `size` 范围内。
2. 若 `index` 在 `0` 到 `size-1` 之间，则调用 `System.arraycopy` 将当前位置及之后的元素向后移动一位，腾出插入位置。
3. 无论插入位置在哪里，都直接将 `element` 插入到 `array[index]`，并增加 `size`。

```java
public void addLast(int element) {
    add(size, element);
}

public void add(int index, int element) {
    // 添加逻辑
    if (index >= 0 && index < size) {
        // 向后挪动, 空出待插入位置
        System.arraycopy(array, index, array, index + 1, size - index);
    }
    array[index] = element;
    size++;
}
```

## 元素遍歷

### foreach 遍历

```java
public class DynamicArray {
	private int size = 0; 
	private int capacity = 8; 
	private int[] array = new int[capacity]; 
    
	public void foreach() {
		for (int i = 0; i < size; i++) {
            System.out.println(array[i]);
		}
	}
}
```

```java
@Test
@DisplayName("测试遍历")
public void test() {
  DynamicArray dynamicArray = new DynamicArray();
  dynamicArray.addLast(1);
  dynamicArray.addLast(2);
  dynamicArray.addLast(3);
  dynamicArray.addLast(4);

  dynamicArray.foreach();
}
```

思考一下這樣寫好不好，這樣寫相當於這個 foreach 方法只能做一件事情，就是打印每個元素。

如果外界想要拿到每一個元素，做一些別的操作，比如說：把元素插入到數據庫等，那這個 foreach 方法就不能使用了。

那能不能把它要做的事情不要寫死在方法裡面，而是透過方法的參數傳遞進來呢？

如果在 `foreach()` 方法裡直接寫死 `System.out.println`，那這個方法就只剩下一個用途：列印每個元素。
遍歷邏輯（走訪每個元素）和業務行為（要對元素做什麼）被綁在一起，導致可重用性很差。

一旦呼叫端想對元素做其他事情，例如：

* 寫入資料庫
* 累加統計、計算總和
* 過濾、轉換、收集到另一個容器

就必須重新寫一份遍歷程式碼，原本的 `foreach()` 幾乎派不上用場。

更好的設計是：遍歷只負責提供元素，行為交由呼叫端決定。

做法是把「對每個元素要執行的操作」抽成參數（例如 Consumer），由外部傳入 callback，讓遍歷方法保持通用。

```java
public class DynamicArray {
	private int size = 0; 
	private int capacity = 8; 
	private int[] array = new int[capacity];

  /**
   * 遍历有效元素（索引范围 [0, size)），并把每个元素交给调用方处理。
   *
   * 说明：
   * - 遍历本身只负责「按顺序提供元素」
   * - 具体要对元素做什么（打印、入库、统计等）由参数 consumer 决定
   *
   * @param consumer 对每个元素要执行的操作
   */
	public void foreach(Consumer<Integer> consumer) {
		for (int i = 0; i < size; i++) {
			// 提供 array[i]
			// 返回 void
			consumer.accept(array[i]);
		}
	}
}
```

```java
@Test
@DisplayName("测试 foreach 遍历")
public void test() {
  DynamicArray dynamicArray = new DynamicArray();
  dynamicArray.addLast(1);
  dynamicArray.addLast(2);
  dynamicArray.addLast(3);
  dynamicArray.addLast(4);

  dynamicArray.foreach((element) -> {
      System.out.println(element);
  });
}
```

### 迭代器遍历

```java
public class DynamicArray implements Iterable<Integer> {
	private int size = 0; 
	private int capacity = 8; 
	private int[] array = new int[capacity]; 

	/**
	 * 遍历方法 - 迭代器遍历
	 */
	@Override
	public Iterator<Integer> iterator() {
		return new Iterator<Integer>() {
			int i = 0;

			@Override
			public boolean hasNext() { // 有没有下一个元素
				return i < size;
			}

			@Override
			public Integer next() { // 返回当前元素,并移动到下一个元素
				return array[i++];
			}
		};
	}
}
```

## 元素刪除
動態數組的有效元素範圍是 [0, size)，也就是索引 0 ~ size - 1。

刪除某個位置 index 的元素，本質上分成三步：

1. 先保存被刪除的元素（因為後續會覆蓋它）
2. 將後面的元素整段往前搬移一格，補上空缺
   - 要搬移的是區間 [index + 1, size - 1]
3. size 減 1，代表有效元素少了一個

```java
public class DynamicArray {
	private int size = 0;
	private int capacity = 8; 
	private int[] array = new int[capacity];

	
	/**
	 * 从 [0 .. size) 范围删除元素
	 *
	 * @param index 索引位置
	 * @return 被删除元素
	 */
	public int remove(int index) { // [0..size)
		int removed = array[index];
		// 向前挪动
		System.arraycopy(array, index + 1, array, index, size - index - 1);
		size--;
		return removed;	
	}
}
```

```java
@Test
@DisplayName("测试删除")
public void test() {
	DynamicArray dynamicArray = new DynamicArray();
	dynamicArray.addLast(1);
	dynamicArray.addLast(2);
	dynamicArray.addLast(3);
	dynamicArray.addLast(4);
	dynamicArray.addLast(5);

	int removed = dynamicArray.remove(2);
	assertEquals(3, removed);
}
```

### 特殊情況：刪除最後一個元素

假設元素為 `[1, 2, 3, 4, 5]`，刪除索引 4（也就是最後一個）：

* size = 5
* index = 4
* 需要搬移的元素數量：$size - index - 1 = 5 - 4 - 1 = 0$

這代表 後面沒有元素需要搬移，所以 `System.arraycopy(..., 0)` 其實什麼都不做，是完全合法的。

因此，不加判斷也能正確工作。

如果希望避免呼叫一次「拷貝 0 個元素」的 arraycopy（純粹是微優化、提升可讀性也行），可以加上判斷：

```java
public class DynamicArray{
	private int size = 0; 
	private int capacity = 8; 
	private int[] array = new int[capacity]; 

	/**
	 * 从 [0 .. size) 范围删除元素
	 *
	 * @param index 索引位置
	 * @return 被删除元素
	 */
	public int remove(int index) { // [0..size)
		int removed = array[index];
		if (index < size - 1) {
			// 向前挪动
			System.arraycopy(array, index + 1, array, index, size - index - 1);
		}
		size--;
		return removed;
	}
}
```

## 扩容

```java
public class DynamicArray {
	private int size = 0; 
	private int capacity = 8; 
	private int[] array = new int[capacity]; 

	/**
	 * 向最后位置 [size] 添加元素
	 *
	 * @param element 待添加元素
	 */
	public void addLast(int element) {
		add(size, element);
	}

	/**
	 * 向 [0 .. size] 位置添加元素
	 *
	 * @param index   索引位置
	 * @param element 待添加元素
	 */
	public void add(int index, int element) {
		checkAndGrow();
        
		// 添加逻辑
		if (index >= 0 && index < size) {
			// 向后挪动, 空出待插入位置
			System.arraycopy(array, index, array, index + 1, size - index);
		}
		array[index] = element;
		size++;
	}

    private void checkAndGrow() {
        // 容量檢查
        if (size == capacity) {
            // 进行扩容 1.5 倍
            capacity += capacity >> 1;
            int[] newArray = new int[capacity];
            System.arraycopy(array, 0, newArray, 0, size);
            array = newArray;
        }
    }
}
```

## 懒加载數組

```java
public class DynamicArray {
	private int size = 0; 
	private int capacity = 8; 
	private int[] array = {};

	/**
	 * 向最后位置 [size] 添加元素
	 *
	 * @param element 待添加元素
	 */
	public void addLast(int element) {
		add(size, element);
	}

	/**
	 * 向 [0 .. size] 位置添加元素
	 *
	 * @param index   索引位置
	 * @param element 待添加元素
	 */
	public void add(int index, int element) {
		checkAndGrow();

		// 添加逻辑
		if (index >= 0 && index < size) {
			// 向后挪动, 空出待插入位置
			System.arraycopy(array, index, array, index + 1, size - index);
		}
		array[index] = element;
		size++;
	}

    private void checkAndGrow() {
        // 容量检查
        if (size == 0) {
            array = new int[capacity];
        } else if (size == capacity) {
            // 进行扩容 1.5 倍
            capacity += capacity >> 1;
            int[] newArray = new int[capacity];
            System.arraycopy(array, 0,
                    newArray, 0, size);
            array = newArray;
        }
    }
}
```

## 查詢、插入、删除性能
### 1) 查詢（依索引存取）

* **依索引讀取**：`get(i)`

  * 底層是連續記憶體，可直接定位到第 `i` 個位置
  * 時間複雜度：**O(1)**

### 2) 插入（insert）

插入的關鍵成本在於：**為了維持連續索引，需要搬移插入點後面的元素**。

* **頭部插入**：`add(0, x)`

  * 需要把原本的元素整段往後移
  * 時間複雜度：**O(n)**

* **中間插入**：`add(index, x)`（0 < index < size）

  * 同樣需要搬移插入點後面的元素
  * 時間複雜度：**O(n)**

* **尾部追加**：`addLast(x)` / `add(size, x)`

  * 通常直接寫到 `array[size]`
  * **均攤（amortized）時間複雜度：O(1)**
  * 但如果容量不足，需要**擴容（新建陣列 + 複製舊資料）**，該次操作可能會變成 **O(n)**

### 3) 刪除（remove）

刪除的關鍵成本也在於：**刪除點後面的元素要往前補位**。

* **頭部刪除**：`remove(0)`

  * 後面元素需整段往前移
  * 時間複雜度：**O(n)**

* **中間刪除**：`remove(index)`（0 < index < size - 1）

  * 後面元素需搬移補上空缺
  * 時間複雜度：**O(n)**

* **尾部刪除**：`remove(size - 1)`

  * 不需要搬移元素，只是 `size--`
  * 時間複雜度：**O(1)**

> 👉 **灰色的部分就是之前提到的對齊字節  ⇒  java 中所有对象大小都是 8 字节的整数倍，不足的要用对齐字节补足。**

- 二维数组占 32 个字节，其中 `array[0]`，`array[1]`，`array[2]` 三个元素分别保存了指向三个一维数组的引用
- 三个一维数组各占 40 个字节
- 它们在内层布局上是 **连续** 的

更一般的，对一个二维数组 `Array[m][n]`

- m 是外层数组的长度，可以看作 row 行
- n 是内层数组的长度，可以看作 column 列
- 当访问 `Array[i][j]`，$0\leq i \lt m, 0\leq j \lt n$ 时，就相当于
  - 先找到第 i 个内层数组（行）
  - 再找到此内层数组中第 j 个元素（列）

**小测试**

Java 环境下（不考虑类指针和引用压缩，此为默认情况），有下面的二维数组

```java
byte[][] array = {
    {11, 12, 13, 14, 15},
    {21, 22, 23, 24, 25},
    {31, 32, 33, 34, 35},
};
```

已知 array **对象**起始地址是 `0x1000`，那么 23 这个元素的地址是什么？

> 答：
>
> - 起始地址 `0x1000`
> - 外层数组大小：$16 字节对象头 + 3元素 \times 每个引用4字节 + 4 对齐字节 = 32 = 0x20$
> - 第一个内层数组大小：$16字节对象头 + 5元素 \times 每个byte1字节 + 3 对齐字节 = 24 = 0x18$
> - 第二个内层数组：$16字节对象头 = 0x10$，待查找元素索引为 2
> - 最后结果：$0x1000 + 0x20 + 0x18 + 0x10 + 2 \times 1 = 0x104a$

# 局部性原理

这里只讨论空间局部性

- cpu 读取内存（速度慢）数据后，会将其放入高速缓存（速度快）当中，如果后来的计算再用到此数据，在缓存中能读到的话，就不必读内存了
- 缓存的最小存储单位是缓存行（cache line），一般是 64 bytes，一次读的数据少了不划算啊，因此最少读 64 bytes 填满一个缓存行，因此读入某个数据时也会读取其**临近的数据**，这就是所谓**空间局部性。**

## 对效率的影响

比较下面 `ij` 和 `ji` 两个方法的执行效率

- 兩者遍歷二維數組的順序不一樣
- `ij`：先行再列
- `ji`：先列再行

```java
public class TestCacheLine {
	public static void main(String[] args) throws Exception {
		int rows = 1000000;
		int columns = 14;
		int[][] a = new int[rows][columns];

		StopWatch sw = new StopWatch();
		sw.start("ij");
		ij(a, rows, columns);
		sw.stop();
		sw.start("ji");
		ji(a, rows, columns);
		sw.stop();
		System.out.println(sw.prettyPrint());
	}

	public static void ij(int[][] a, int rows, int columns) {

		}
		System.out.println(sum);
	}
}
```

```java
import java.util.ArrayList;
import java.util.List;

public class StopWatch {
	private static class Task {
		String name;
		long startTime;
		long duration;

		Task(String name, long startTime) {
			this.name = name;
			this.startTime = startTime;
		}
	}

	private final List<Task> tasks = new ArrayList<>();
	private long startTime;
	private boolean running = false;
	private long totalTime = 0;

	// 開始計時
	public void start(String taskName) {
		if (running) {
			throw new IllegalStateException("StopWatch is already running");
		}
		startTime = System.nanoTime();
		tasks.add(new Task(taskName, startTime));
		running = true;
	}

	// 停止計時
	public void stop() {
		if (!running) {
			throw new IllegalStateException("StopWatch is not running");
		}
		long endTime = System.nanoTime();
		Task currentTask = tasks.get(tasks.size() - 1);
		currentTask.duration = endTime - currentTask.startTime;
		totalTime += currentTask.duration;
		running = false;
	}

	// 取得格式化的報告
	public String prettyPrint() {
		StringBuilder sb = new StringBuilder();
		sb.append("StopWatch '': running time = ").append(totalTime).append(" ns\n");
		sb.append("---------------------------------------------\n");
		sb.append("ns         %     Task name\n");
		sb.append("---------------------------------------------\n");

		for (Task task : tasks) {
			long percent = (task.duration * 100) / totalTime;
			sb.append(String.format("%09d  %03d%%  %s\n", task.duration, percent, task.name));
		}
		return sb.toString();
	}
}
```

# System.arraycopy 的使用

`System.arraycopy` 是 Java 提供的一种高效数组复制方法，用于在数组之间快速复制元素。

该方法是静态的，属于 `System` 类，可以比手动循环复制数组元素更快，因为它直接利用底层的内存操作。

语法：

```java
System.arraycopy(Object src, int srcPos, Object dest, int destPos, int length);
```

参数说明：

- `src`：源数组，表示从哪个数组中复制数据。
- `srcPos`：源数组的起始位置，表示从源数组的哪个索引位置开始复制。
- `dest`：目标数组，表示数据复制到哪个数组。
- `destPos`：目标数组的起始位置，表示从目标数组的哪个索引位置开始写入。
- `length`：复制的元素数量。

以下是一个简单的示例，展示如何使用 `System.arraycopy` 将一个数组的一部分复制到另一个数组的特定位置。

```java
public class ArrayCopyExample {
    public static void main(String[] args) {
        // 源数组
        int[] sourceArray = {1, 2, 3, 4, 5};

        // 目标数组
        int[] destinationArray = new int[10];

        // 从源数组的索引 0 开始，复制 5 个元素到目标数组的索引 3 开始的位置
        System.arraycopy(sourceArray, 0, destinationArray, 3, 5);

        // 输出目标数组
        for (int i : destinationArray) {
            System.out.print(i + " ");
        }
    }
}
```

输出结果：

```
0 0 0 1 2 3 4 5 0 0
```

在这个例子中，`System.arraycopy` 将 `sourceArray` 的前 5 个元素复制到 `destinationArray` 从索引 `3` 开始的位置。未复制到的元素位置会保持目标数组的默认值（例如 `0`）。

# 合并两个有序数组 ( Leetcode 88 题改)

- [88. 合并两个有序数组](https://leetcode.cn/problems/merge-sorted-array/description/)

> 這題是從 **Leetcode 88. Merge Sorted Array** 延伸出的變形。
> * 和原題不同的是：
> * **不是兩個獨立陣列**，而是**同一個陣列中有兩段已排序區間**，現在要把它們合併起來。

## 題目概念

將數組兩個區間內的有序元素合併

例：

```java
[1, 5, 6, 2, 4, 10, 11]
```

它其實可以看成兩個已經排好序的區間：

```java
[1, 5, 6] 和 [2, 4, 10, 11]
```

目標是把這兩段有序區間合併成一個完整有序序列：

```java
[1, 2, 4, 5, 6, 10, 11]
```

## 方法一：遞迴版

### 核心思路

這題本質上就是：

> **合併兩個有序序列**

做法是同時用兩個指標：

* `i`：指向第一段有序區間的目前元素
* `j`：指向第二段有序區間的目前元素
* `k`：指向結果陣列 `a2` 要放入元素的位置

每次比較 `a1[i]` 和 `a1[j]`：

* 較小者放入 `a2[k]`
* 對應的指標往後移動
* `k` 也往後移動

直到某一段用完，再把另一段剩餘元素整段複製過去。

### 遞迴過程示意

偽碼如下：

```java
merge([1,5,6] [2,4,10,11], a2=[]) {
    merge([5,6] [2,4,10,11], a2=[1]) {
        merge([5,6] [4,10,11], a2=[1,2]) {
            merge([5,6] [10,11], a2=[1,2,4]) {
                merge([6] [10,11], a2=[1,2,4,5]) {
                    merge([] [10,11], a2=[1,2,4,5,6]) {
                         merge([], [], a2=[1,2,4,5,6,10,11])
                    }
                }
            }
        }
    }
}
```

> **遞迴的本質：每一層遞迴只做一件事**
> 1. 比較兩個區間目前的起點元素
> 2. 取較小的那個放進 `a2`
> 3. 縮小對應區間
> 4. 繼續合併剩下的部分

### 終止條件

遞迴停止有兩種情況：

##### 1. 第一段區間已經用完

```java
if (i > iEnd)
```

代表第一段沒元素了，直接把第二段剩餘元素整段複製到 `a2`。


##### 2. 第二段區間已經用完

```java
if (j > jEnd)
```

代表第二段沒元素了，直接把第一段剩餘元素整段複製到 `a2`。

### 程式碼

```java
public class LeetCode88 {

  public static void main(String[] args) {
    int[] a1 = {1, 5, 6, 2, 4, 10, 11};
    int[] a2 = new int[a1.length];

    // 合併 a1[0..2] 和 a1[3..6]
    merge(a1, 0, 2, 3, 6, a2, 0);

    System.out.println("合併後的暫存陣列 a2: " + Arrays.toString(a2));

    // 將結果複製回原陣列
    System.arraycopy(a2, 0, a1, 0, a2.length);
    System.out.println("複製回原陣列 a1: " + Arrays.toString(a1));
  }

  /**
   * 將 a1 中兩個有序區間合併到 a2
   *
   * @param a1   原始陣列
   * @param i    第一個有序區間起點
   * @param iEnd 第一個有序區間終點
   * @param j    第二個有序區間起點
   * @param jEnd 第二個有序區間終點
   * @param a2   結果陣列
   * @param k    a2 的目前填入位置
   */
  public static void merge(int[] a1, int i, int iEnd,
                           int j, int jEnd, int[] a2, int k) {

    // 情況1：第一段已合併完，直接複製第二段剩餘元素
    if (i > iEnd) {
      System.arraycopy(a1, j, a2, k, jEnd - j + 1);
      return;
    }

    // 情況2：第二段已合併完，直接複製第一段剩餘元素
    if (j > jEnd) {
      System.arraycopy(a1, i, a2, k, iEnd - i + 1);
      return;
    }

    // 比較兩段目前元素，較小者放入 a2
    if (a1[i] < a1[j]) {
      a2[k] = a1[i];
      merge(a1, i + 1, iEnd, j, jEnd, a2, k + 1);
    } else {
      a2[k] = a1[j];
      merge(a1, i, iEnd, j + 1, jEnd, a2, k + 1);
    }
  }
}
```

## 方法二：迴圈版

這一題一樣是把同一個數組中的兩段有序區間合併起來。

例如：

```java
[1, 5, 6, 2, 4, 10, 11]
```

可以拆成兩個有序區間：

```java
[1, 5, 6] 和 [2, 4, 10, 11]
```

合併後得到：

```java
[1, 2, 4, 5, 6, 10, 11]
```

### 核心思路

方法二不用遞迴，而是改用 **while 迴圈** 來完成合併。

準備三個指標：

* `i`：指向第一個有序區間的目前元素
* `j`：指向第二個有序區間的目前元素
* `k`：指向結果陣列 `a2` 目前要填入的位置

每次比較 `a1[i]` 和 `a1[j]`：

* 誰比較小，就放進 `a2[k]`
* 對應的指標往後移動
* `k` 也往後移動

當其中一段區間走完後，再把另一段剩餘元素直接複製到 `a2`。

```java
public class LeetCode88 {
    public static void main(String[] args) {
        int[] a1 = {1, 5, 6, 2, 4, 10, 11};
        int[] a2 = new int[a1.length];

        // 合併 a1[0..2] 和 a1[3..6]
        merge(a1, 0, 2, 3, 6, a2);

        System.out.println("合併後的暫存陣列 a2: " + Arrays.toString(a2));

        // 將結果複製回原陣列
        System.arraycopy(a2, 0, a1, 0, a2.length);
        System.out.println("複製回原陣列 a1: " + Arrays.toString(a1));
    }

    /**
     * 將 a1 中兩個有序區間合併到 a2
     *
     * @param a1   原始陣列
     * @param i    第一個有序區間起點
     * @param iEnd 第一個有序區間終點
     * @param j    第二個有序區間起點
     * @param jEnd 第二個有序區間終點
     * @param a2   暫存結果陣列
     */
    public static void merge(int[] a1, int i, int iEnd, int j, int jEnd, int[] a2) {
        int k = i;

        // 只要兩段都還沒走完，就持續比較
        while (i <= iEnd && j <= jEnd) {
            if (a1[i] < a1[j]) {
                a2[k] = a1[i];
                i++;
            } else {
                a2[k] = a1[j];
                j++;
            }
            k++;
        }

        // 如果第一段走完，複製第二段剩餘元素
        if (i > iEnd) {
            System.arraycopy(a1, j, a2, k, jEnd - j + 1);
        }

        // 如果第二段走完，複製第一段剩餘元素
        if (j > jEnd) {
            System.arraycopy(a1, i, a2, k, iEnd - i + 1);
        }
    }
}
```
